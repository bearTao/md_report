## 修正内容

### 1. `parameters` 字段必要性说明

#### 修正前
- 标记为"是（必填）"，但未说明具体条件

#### 修正后
- **条件必填**：SQL 中使用了 `:param_name` 占位符时必须声明
- **可选情况**：SQL 中只使用 `{{variable}}` 插值或完全静态查询时可以为空数组 `[]`

#### 新增说明
```yaml
# 场景1：只用参数化（最常见）
query: "SELECT * FROM users WHERE id = :user_id"
dependencies: ["user_id"]  # 依赖 user_id
parameters: ["user_id"]    # 参数化 user_id
# 两者相同

# 场景2：混合使用
query: "SELECT {{columns}} FROM {{table}} WHERE id = :user_id"
dependencies: ["columns", "table", "user_id"]  # 依赖3个变量
parameters: ["user_id"]                        # 只参数化1个
# parameters 是 dependencies 的子集

# 场景3：只用插值（不推荐，有注入风险）
query: "SELECT * FROM {{table_name}}"
dependencies: ["table_name"]  # 依赖 table_name
parameters: []                # 无参数化
```

### 2. `dependencies` vs `parameters` 关系说明

#### 新增内容
明确两者的区别和关系：

| 字段 | 所属 | 作用域 | 用途 |
|------|------|--------|------|
| `dependencies` | 变量级别 | 调度器层面 | 声明执行顺序依赖，包括所有使用到的变量 |
| `parameters` | SQL 级别 | SQL 执行层面 | 声明参数化查询的参数，只包括 `:param_name` 的变量 |

**关系**：`parameters` 必须是 `dependencies` 的子集

### 3. `schema` 字段使用说明

#### 修正前
- 标记为"否（可选）"，但未说明哪些变量类型使用

#### 修正后
明确 `schema` 字段的实际使用情况：

**仅用于 AI 生成变量和 Vision AI 变量：**
1. **指导 AI 生成**：告诉 AI 应该返回什么结构的数据
2. **验证 AI 输出**：检查 AI 返回的数据是否符合预期结构

**对于其他变量类型：**
- ✅ **AI/Vision AI 变量**：**必须配置** `schema`，确保输出结构正确
- ⚠️ **SQL 变量**：`schema` 字段**不会被使用**，可以省略（仅作文档用途）
- ⚠️ **API 变量**：`schema` 字段**不会被使用**，可以省略（仅作文档用途）
- ⚠️ **用户输入变量**：`schema` 字段**不会被使用**，可以省略

### 4. SQL 示例修正

#### 修正内容
1. **移除不必要的 schema 字段**：SQL 变量示例中删除 `schema` 配置
2. **修正 SQL 语法**：将 `{{variable}}` 改为 `:variable` 参数化查询
3. **添加注释说明**：标注 `parameters` 字段的必要性

#### 修正示例

**修正前：**
```yaml
user_profile:
  sql_config:
    query: "SELECT name FROM users WHERE id = {{user_id}}"
    parameters: ["user_id"]
  schema:  # 不必要
    type: "object"
    properties:
      name: {type: "string"}
```

**修正后：**
```yaml
user_profile:
  dependencies: ["user_id"]  # 声明依赖：用于执行顺序
  sql_config:
    query: "SELECT name FROM users WHERE id = :user_id"  # 使用 :user_id 参数化
    parameters: ["user_id"]  # 必须声明：因为 SQL 中使用了 :user_id
  # schema 字段可以省略：SQL 变量不使用 schema，仅作文档用途
```

### 5. 新增静态查询示例

添加了无参数的静态查询示例：

```yaml
user_ids:
  type: "array"
  source: "sql"
  dependencies: []  # 无依赖
  sql_config:
    query: "SELECT id FROM users WHERE status = 'active'"  # 静态查询，无参数
    parameters: []  # 空数组：因为 SQL 中没有使用 :param
    result_mode: "first_column"
```
