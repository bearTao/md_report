# 日志查看和调试指南

## 📊 日志系统概述

本系统使用分级日志系统，支持详细的调试和问题排查。

### 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG | 详细的调试信息 | 变量值、中间结果、API调用详情 |
| INFO | 重要的操作步骤 | 任务开始、完成、状态变更 |
| WARNING | 警告信息 | 非致命错误、性能问题 |
| ERROR | 错误信息 | 失败的操作、异常 |
| CRITICAL | 严重错误 | 系统崩溃级别的问题 |

### 日志输出位置

- **控制台**: 显示 INFO 及以上级别（彩色输出）
- **文件**: `/data/tao/code/xuqiu/backend/logs/app.log` 显示 DEBUG 及以上所有级别

## 🚀 快速开始

### 方法1：查看实时日志（推荐）

```bash
# 在一个终端窗口
tail -f /data/tao/code/xuqiu/backend/logs/app.log

# 或者只看ERROR和WARNING
tail -f /data/tao/code/xuqiu/backend/logs/app.log | grep -E "ERROR|WARNING"

# 或者只看AI相关的日志
tail -f /data/tao/code/xuqiu/backend/logs/app.log | grep "app.executors.ai"
```

### 方法2：搜索特定内容

```bash
# 搜索某个任务ID的所有日志
grep "task_abc123" /data/tao/code/xuqiu/backend/logs/app.log

# 搜索AI相关的错误
grep "app.executors.ai.*ERROR" /data/tao/code/xuqiu/backend/logs/app.log

# 查看最近的错误
grep "ERROR" /data/tao/code/xuqiu/backend/logs/app.log | tail -20
```

### 方法3：使用任务监控脚本

```bash
# 实时监控任务进度
python3 /data/tao/code/xuqiu/monitor_task.py <task_id>
```

## 🔍 AI生成调试详解

### DEBUG级别日志内容

当AI变量执行时，DEBUG日志会记录：

```
1. 🤖 开始执行AI变量
   - 变量名
   - AI配置（模型、温度等）

2. 🔄 插值提示词模板
   - 原始模板
   - 依赖变量列表
   - 每个依赖变量的实际值（前200字符）
   - 最终提示词（前500字符）

3. 🔧 创建LLM实例
   - LLM配置参数
   - 自定义API base（如有）

4. 🚀 调用AI模型
   - 调用开始时间
   - 等待提示

5. ✅ AI响应
   - 响应完成时间和耗时
   - 响应对象类型
   - 完整响应内容

6. 🔍 解析JSON
   - 原始内容长度
   - 是否检测到Markdown代码块
   - 解析尝试过程
   - 解析失败的具体位置（行号、列号）

7. ✅ Schema验证
   - 验证结果
```

## ❌ 常见错误及解决方案

### 1. OpenAI API Key未配置

**日志**:
```
ERROR - app.executors.ai - ❌ OpenAI API key 未配置
```

**解决**: 
- 访问 http://localhost:5174/settings/ai
- 配置OpenAI API Key

### 2. 提示词插值失败

**日志**:
```
ERROR - app.executors.ai - ❌ 提示词插值失败: Variable 'xxx' not found
```

**解决**: 
- 检查依赖变量是否存在
- 检查变量名拼写是否正确
- 查看DEBUG日志确认依赖变量的值

### 3. AI API调用失败

**日志**:
```
ERROR - app.executors.ai - ❌ AI API调用失败，耗时: 5.00秒
ERROR - app.executors.ai - 错误类型: AuthenticationError
ERROR - app.executors.ai - 错误详情: Invalid API key
```

**解决**:
- 检查API Key是否正确
- 检查API Key是否有余额
- 检查网络连接

### 4. JSON解析失败

**日志**:
```
WARNING - app.executors.ai - ⚠️  初次JSON解析失败: Expecting ',' delimiter
DEBUG - app.executors.ai - 解析失败位置: 行3 列15
```

**解决**:
- 查看DEBUG日志中的完整AI响应
- 检查AI返回的JSON格式
- 可能需要调整提示词，明确要求返回纯JSON

### 5. Schema验证失败

**日志**:
```
ERROR - app.executors.ai - ❌ AI生成失败: Required field 'xxx' missing
```

**解决**:
- 查看schema定义
- 检查AI返回的JSON是否包含所有必需字段
- 调整提示词，明确说明需要的字段

## 📁 日志文件管理

### 日志轮转

- 单个日志文件最大: 10MB
- 保留备份数量: 5个
- 文件命名: app.log, app.log.1, app.log.2, ..., app.log.5

### 清理旧日志

```bash
# 清理30天前的日志
find /data/tao/code/xuqiu/backend/logs -name "*.log*" -mtime +30 -delete

# 或者手动删除
rm /data/tao/code/xuqiu/backend/logs/app.log.*
```

## 🎯 最佳实践

1. **开发阶段**: 使用DEBUG级别查看详细信息
2. **生产环境**: 使用INFO级别，减少日志量
3. **排查问题**: 先看ERROR日志，再看对应的DEBUG日志
4. **性能分析**: 关注"耗时"相关的日志
5. **定期清理**: 避免日志文件占用过多磁盘空间

## 📞 需要帮助？

如果日志中出现无法理解的错误：

1. 复制完整的错误日志（包括堆栈跟踪）
2. 查看该错误前后的DEBUG日志
3. 检查相关的配置和输入数据
4. 使用monitor_task.py实时查看任务进度
