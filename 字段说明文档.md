# 变量元数据字段说明文档

## 基础字段说明

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| type | string | 是 | 变量数据类型 | "string", "number", "boolean", "array", "object" |
| source | string | 是 | 数据来源类型 | "user_input", "sql", "api", "ai_generation", "constant", "system", "image", "vision_ai" |
| required | boolean | 是 | 是否必须提供值 | true, false |
| description | string | 是 | 变量功能描述 | "报告标题", "用户档案信息" |
| default | any | 否 | 默认值（为空时使用） | "未命名报告", [], {}, 0 |
| dependencies | array | 否 | 依赖的其他变量名 | ["user_id"], ["project_id", "time_period"] |
| schema | object | 否 | 数据结构定义 | 见下方schema详细说明 |

|--------|------|------|------|---------|
| ui_config | object | 是 | 用户界面配置 |  |

### SQL数据库变量 (source: "sql")

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| sql_config | object | 是 | SQL查询配置 |  |
| sql_config.connection | string | 是 | 数据库连接名称 | "user_db", "project_db" |
| sql_config.query | string | 是 | SQL查询语句 | "SELECT name FROM users WHERE id = {{user_id}}" |
| sql_config.parameters | array | 否 | 查询参数变量名 | ["user_id"], ["project_id", "start_date"] |
| sql_config.timeout | number | 否 | 查询超时时间(秒) | 10, 30 |
| sql_config.result_mode | string | 否 | 结果返回模式 | "first_row", "all_rows", "first_value", "first_column", "auto" (默认) |

**result_mode 说明：**
- `first_row`: 返回第一行作为对象 `{col1: val1, col2: val2}`
- `all_rows`: 返回所有行作为数组 `[{row1}, {row2}, ...]`
- `first_value`: 返回第一行第一列的值（标量）
- `first_column`: 返回第一列的所有值 `[val1, val2, ...]`
- `auto`: 根据 `type` 自动判断（默认行为）
  - `type="object"`: 单行返回对象，多行返回数组
  - `type="array"`: 返回所有行
  - `type="string/number/boolean"`: 返回第一行第一列

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|

**类型保持说明**（2025-10-29 新增）：

从 2025-10-29 版本开始，`parameters` 和 `body` 中的变量插值会智能保持原始类型：

**纯变量引用** - 保持原始类型：
```yaml
api_config:
  body:
    count: "{{total_count}}"      # 如果 total_count=15 → 发送 15 (number)
    active: "{{is_active}}"       # 如果 is_active=true → 发送 true (boolean)
    user_id: "{{user_id}}"        # 如果 user_id=123 → 发送 123 (number)
```

**模板字符串** - 返回字符串：
```yaml
api_config:
  body:
    message: "共{{count}}条"      # → "共15条" (string)
    range: "{{min}}-{{max}}"      # → "100-500" (string)
```

**判断规则**：
- 整个字符串只有 `{{var}}` → 保持原始类型
- 包含其他文本或多个变量 → 转换为字符串
- 使用过滤器 `{{var|filter}}` → 转换为字符串

**实际 HTTP 请求示例**：
```yaml
# 配置
api_config:
  body:
    department: "{{dept_name}}"        # dept_name = "技术部"
    min_salary: "{{min_salary}}"       # min_salary = 15000
    active: "{{is_active}}"            # is_active = true
    info: "最低{{min_salary}}"          # 模板字符串

# 实际发送的 JSON
{
  "department": "技术部",               // string
  "min_salary": 15000,                 // number ✅
  "active": true,                      // boolean ✅
  "info": "最低15000"                  // string
}
```

**向后兼容**：
- 字符串变量行为不变
- 仅影响数字和布尔类型的纯变量引用
- 默认启用，无需额外配置

#### response_mapping 使用说明

`response_mapping` 支持三种模式，根据值的类型自动判断：

**模式1：完整响应（不映射）**
```yaml
# 方式1：不写 response_mapping 字段
api_config:
  endpoint: "http://api.example.com/data"
  
# 方式2：设为 null
api_config:
  endpoint: "http://api.example.com/data"
  response_mapping: null
```
返回 API 的完整原始响应。

**模式2：单路径提取（字符串）**
```yaml
# 提取嵌套对象
api_config:
  endpoint: "http://api.example.com/weather"
  response_mapping: "data.current"  # 字符串

# 提取数组
api_config:
  endpoint: "http://api.example.com/products"
  response_mapping: "data.items"  # 返回纯数组

# 提取基础类型
api_config:
  endpoint: "http://api.example.com/version"
  response_mapping: "version"  # 返回字符串 "2.5.1"
```
使用 JMESPath 语法提取单个路径，返回值可以是任意类型（对象/数组/字符串/数字）。

**模式3：多字段映射（对象）**
```yaml
api_config:
  endpoint: "http://api.example.com/weather"
  response_mapping:  # 对象：映射多个字段
    temperature: "data.current.temp"
    condition: "data.current.condition"
    latitude: "coord.lat"
```
从不同路径提取多个字段并重组为新对象。

#### JMESPath 语法支持

系统使用 JMESPath 进行强大的路径提取，支持：

- **简单路径**: `"data.items"`
- **数组索引**: `"items[0]"`, `"data.products[0].name"`
- **数组切片**: `"items[:3]"`, `"items[1:5]"`
- **投影（提取所有）**: `"items[*].name"` - 提取所有name字段
- **条件过滤**: `"items[?price > \`100\`]"` - 筛选价格>100的项
- **函数**: `"length(items)"`, `"sum(items[*].price)"`

详见 [JMESPath 文档](https://jmespath.org/)

#### 重试机制说明

当配置 `retry_count > 0` 时，系统会在以下情况自动重试：
- HTTP 状态码在 `retry_status_codes` 列表中
- 请求超时
- 重试间隔采用线性退避：第n次重试等待 `retry_backoff * n` 秒

示例：
```yaml
api_config:
  endpoint: "http://api.unstable.com/data"
  retry_count: 3  # 最多重试3次
  retry_status_codes: [429, 500, 503]  # 这些状态码会重试
  retry_backoff: 2.0  # 第1次等2秒，第2次等4秒，第3次等6秒
```

### 图片变量 (source: "image")

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| image_config | object | 是 | 图片获取配置 |  |
| image_config.endpoint | string | 是 | 图片URL或API端点 | "https://api.example.com/image.jpg" |
| image_config.method | string | 否 | HTTP方法 | "GET" (默认) |
| image_config.headers | object | 否 | 请求头 | {"Authorization": "Bearer {{token}}"} |
| image_config.parameters | object | 否 | URL参数 | {"size": "large", "quality": "80"} |

```json
  "markdown": "![图片](https://...)"
}
```

**返回数据格式（多张图片）：**
```json
[
  {"data": "...", "url": "...", "mime_type": "...", "size": ..., "markdown": "..."},
  {"data": "...", "url": "...", "mime_type": "...", "size": ..., "markdown": "..."}
]
```

### 常量变量 (source: "constant")

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| value | any | 是 | 常量的固定值 | 15000, "北京", true, [1,2,3] |

**使用场景**：
- 业务常量：薪资标准、税率、折扣率等
- 配置参数：API地址、公司信息、系统参数等
- 固定参数：在多个变量中共享的不变值

**特点**：
- 值在模板定义时固定，不会动态生成
- **自动预执行**：常量总是在所有其他变量之前自动执行
- **无需声明依赖**：其他变量使用常量时无需在 `dependencies` 中声明
- 支持所有数据类型（number, string, boolean, array, object）
- 执行速度最快（直接返回值）
- 配合智能类型保持，数字/布尔等类型自动保持

**示例**：

```yaml
# 业务常量
min_salary_standard:
  type: number
  source: constant
  description: "公司最低薪资标准"
  value: 15000

# 配置常量
api_base_url:
  type: string
  source: constant
  description: "API基础地址"
  value: "http://10.10.20.10:5000"

# 税率常量
vat_rate:
  type: number
  source: constant
  description: "增值税率"
  value: 0.13

# 布尔常量
enable_cache:
  type: boolean
  source: constant
  description: "是否启用缓存"
  value: true

# 数组常量
major_cities:
  type: array
  source: constant
  description: "主要城市列表"
  value: ["北京", "上海", "深圳", "杭州"]

# 对象常量
default_config:
  type: object
  source: constant
  description: "默认配置"
  value:
    timeout: 30
    retry: 3
    cache_ttl: 1800
```

**在其他变量中使用常量**：

```yaml
# 定义常量
api_base_url:
  type: string
  source: constant
  value: "http://10.10.20.10:5000"

min_salary:
  type: number
  source: constant
  value: 15000

# 在 API 调用中引用常量
user_query:
  type: object
  source: api
  # ✅ 无需声明常量依赖！常量自动可用
  # dependencies: []  
  api_config:
    endpoint: "{{api_base_url}}/api/users"  # 常量自动可用
    method: POST
    body:
      min_salary: "{{min_salary}}"  # 引用常量，会保持数字类型
      department: "技术部"
```

**自动注入说明**：

从 2025-10-29 版本开始，所有常量变量会自动预先执行并注入到执行上下文：

- ✅ **无需声明依赖**：使用常量时不需要在 `dependencies` 中声明
- ✅ **自动优先执行**：常量总是在其他变量之前执行
- ✅ **向后兼容**：显式声明依赖仍然有效
- ✅ **类型保持**：配合智能类型保持功能，数字/布尔等类型自动保持

**示例对比**：

```yaml
# 【推荐】新方式：无需声明常量依赖
tech_users:
  type: object
  source: api
  api_config:
    endpoint: "{{api_base_url}}/api/users"  # 常量自动可用

# 【兼容】旧方式：显式声明依赖也可以
products_api:
  type: object
  source: api
  dependencies: [api_base_url]  # 可选，向后兼容
  api_config:
    endpoint: "{{api_base_url}}/api/products"
```

详见 [常量自动注入功能说明](../常量自动注入功能说明.md)

**真实案例：修复前后对比**

## 配置对象详细说明

### Schema 数据结构定义

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| type | string | 是 | 数据类型 | "object", "array", "string", "number" |
| properties | object | 对象必填 | 对象属性定义 | {"name": {"type": "string"}} |
| items | object | 数组必填 | 数组元素定义 | {"type": "object", "properties": {...}} |
| format | string | 否 | 数据格式 | "date", "email", "url" |
| enum | array | 否 | 枚举值列表 | ["高", "中", "低"] |

### API Response Mapping 响应映射

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| {field_name} | string | 对象映射 | 字段映射路径 | "data.current.temp" |
| extract_array | string | 数组映射 | 数组提取路径 | "data.stocks" |
| item_mapping | object | 数组映射 | 数组元素字段映射 | {"symbol": "symbol", "price": "price"} |

### System Config 系统配置

| 字段名 | 类型 | 必填 | 描述 | 示例值 |
|--------|------|------|------|---------|
| generator | string | 是 | 生成器类型 | "datetime", "uuid" |
| format | string | 否 | 输出格式 | "YYYY-MM-DD HH:mm:ss" |
| value | any | 否 | 固定值 | "1.0" |

## 数据类型说明

### type 字段可选值

| 类型值 | 描述 | 使用场景 |
|--------|------|----------|
| "string" | 字符串类型 | 文本内容、名称、描述等 |
| "number" | 数字类型 | 数量、百分比、价格等 |
| "boolean" | 布尔类型 | 开关状态、是否标记等 |
| "array" | 数组类型 | 列表数据、多项选择等 |
| "object" | 对象类型 | 复杂数据结构、嵌套数据等 |

### source 字段可选值

| 来源值 | 描述 | 使用场景 |
|--------|------|----------|
| "user_input" | 用户输入 | 需要用户填写或选择的数据 |
| "sql" | 数据库查询 | 从数据库中获取的数据 |
| "api" | 接口调用 | 从外部API获取的数据 |
| "ai_generation" | AI生成 | 通过AI模型生成的内容 |
| "constant" | 常量值 | 固定不变的配置值、业务常量（**自动预执行，无需声明依赖**） |
| "image" | 图片获取 | 从HTTP API获取图片数据 |
| "vision_ai" | 视觉AI分析 | 使用视觉大模型分析图片内容 |
| "system" | 系统生成 | 系统自动生成的数据（时间、ID等） |

### SQL数据库变量示例

```yaml
# 示例1：获取单个用户档案（返回对象）
user_profile:
  type: "object"
  source: "sql"
  required: false
  default: {}
  description: "用户档案信息"
  dependencies: ["user_id"]
  sql_config:
    connection: "user_db"
    query: "SELECT name, email, department, role FROM users WHERE id = {{user_id}}"
    parameters: ["user_id"]
    timeout: 10
    result_mode: "first_row"  # 明确返回第一行作为对象
  schema:
    type: "object"
    properties:
      name: {type: "string"}
      email: {type: "string"}
      department: {type: "string"}
      role: {type: "string"}

# 示例2：获取用户列表（返回数组）
team_members:
  type: "array"
  source: "sql"
  required: false
  default: []
  description: "团队成员列表"
  dependencies: ["department_id"]
  sql_config:
    connection: "user_db"
```

## 常见问题和解决方案

### 问题1：字典方法名冲突导致渲染失败

**错误信息**：
```
TypeError: object of type 'builtin_function_or_method' has no len()
```

**原因**：
SQL 返回的字典包含与 Python 字典方法重名的键（如 `items`, `keys`, `values`）

**解决方案**：
1. **最佳**：在 SQL 中使用别名
```sql
SELECT "items" AS item_list FROM orders
```

2. **临时**：在模板中使用方括号
```jinja2
{{ order["items"] | length }}
```

详见《报告模板编写规范指南》第4节"陷阱8"。

---

### 问题2：API body 中数字被转换为字符串

**现象**：
```yaml
body:
  min_salary: "{{min_salary_value}}"  # 发送 "15000" 而不是 15000
```

**后端报错**：
```json
{"error": "Invalid type for min_salary, expected number but got string"}
```

**解决方案**：
从 2025-10-29 版本开始，纯变量引用会自动保持原始类型，无需额外配置。

如果使用旧版本，可以：
```yaml
# 方案1：使用 SQL 转换
sql_config:
  query: "SELECT CAST(min_salary AS INTEGER) AS min_salary_int"

# 方案2：使用 AI 生成正确格式的 JSON
```

---

### 问题3：常量变量未生效

**现象**：
```
DependencyError: Variable 'api_base_url' not found
```

**原因**：
使用旧版本（2025-10-29 之前），常量未自动注入

**解决方案**：
```yaml
# 旧版本：需要显式声明依赖
variable:
  dependencies: [api_base_url]

# 新版本（推荐）：自动注入，无需声明
variable:
  dependencies: []  # 或不写
```

---

### 问题4：模板渲染失败但没有日志

**现象**：
- 任务状态为 failed
- 报告状态为 running（不一致）
- 没有详细的错误日志

**原因**：
使用旧版本（2025-10-29 之前），渲染日志未记录到数据库

**解决方案**：
升级到最新版本，渲染日志会自动记录，可在前端"执行日志"查看。
