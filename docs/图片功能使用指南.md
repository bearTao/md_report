## 功能概述

P1.1版本新增了两种强大的变量类型：

### 1. 图片变量 (`image`)
- 从HTTP API获取图片
- 支持Base64编码或URL格式
- 自动生成Markdown图片引用
- 支持批量图片获取

### 2. 视觉AI变量 (`vision_ai`)
- 使用GPT-4V/GPT-4o分析图片
- 生成文字描述或分析报告
- 支持多张图片联合分析
- 灵活的Prompt配置

## 快速开始

### 最简单的例子

#### 1. 定义图片变量

```yaml
product_photo:
  type: image
  source: image
  description: 产品照片
  image_config:
    endpoint: "https://example.com/product.jpg"
    output_format: url
```

#### 2. 在模板中使用

```markdown
# 产品报告

## 产品照片
{{product_photo.markdown}}
```

就这么简单！系统会自动：
1. 从API获取图片
2. 转换为所需格式
3. 插入到Markdown中

## 图片变量配置

### 基本配置

```yaml
variable_name:
  type: image  # 或 "array"（批量图片）
  source: image
  description: 变量描述
  dependencies: []  # 可选：依赖的其他变量
  image_config:
    endpoint: ""  # 必填：图片URL或API端点
    method: GET  # 可选：HTTP方法
    headers: {}  # 可选：请求头
    parameters: {}  # 可选：URL参数
    output_format: base64  # 可选：输出格式
    multiple: false  # 可选：是否批量获取
    timeout: 30  # 可选：超时时间（秒）
```

### 配置项详解

#### endpoint（必填）
图片URL或API端点，支持变量插值：

```yaml
# 静态URL
endpoint: "https://example.com/image.jpg"

# 动态URL（使用变量）
endpoint: "https://api.example.com/products/{{product_id}}/photo"

# 批量图片（引用数组变量）
endpoint: "{{image_urls}}"  # image_urls应该是一个URL数组
```

#### output_format（可选）
输出格式选择：

- **`base64`**（默认）: 转换为Base64编码
  - 优点：图片内嵌到Markdown，无需外部资源
  - 缺点：文件较大
  - 适用：小图片、需要离线查看

- **`url`**: 保持原始URL
  - 优点：Markdown文件小
  - 缺点：需要网络访问
  - 适用：大图片、有稳定图床

- **`bytes`**: 原始字节数据
  - 仅用于特殊处理，一般不推荐

#### multiple（可选）
批量图片获取：

```yaml
# 单张图片
multiple: false

# 多张图片
multiple: true
endpoint: "{{image_url_list}}"  # 需要是URL数组
```

#### headers（可选）
自定义请求头，常用于认证：

```yaml
headers:
  Authorization: "Bearer {{api_token}}"
  User-Agent: "MyApp/1.0"
  Accept: "image/*, */*"
```

#### parameters（可选）
URL查询参数：

```yaml
parameters:
  size: large
  quality: high
  format: png
  user_id: "{{user_id}}"  # 支持变量插值
```

### 完整示例

```yaml
company_logo:
  type: image
  source: image
  description: 公司Logo
  image_config:
    endpoint: "https://api.example.com/v1/companies/{{company_id}}/logo"
    method: GET
    headers:
      Authorization: "Bearer {{api_key}}"
    parameters:
      size: medium
      format: png
    output_format: base64
    timeout: 20
```

## 视觉AI配置

### 基本配置

```yaml
variable_name:
  type: string
  source: vision_ai
  description: AI分析结果
  dependencies: [image_variable]  # 必须：依赖的图片变量
  vision_ai_config:
    model: gpt-4o  # 必填：模型名称
    image_source: image_variable  # 必填：图片变量名
    prompt_template: ""  # 必填：分析提示词
    system_prompt: ""  # 可选：系统提示词
    temperature: 0.7  # 可选：温度参数
    max_tokens: 500  # 可选：最大Token数
```

### 配置项详解

#### model（必填）
视觉模型选择：

- **`gpt-4o`**（推荐）: 最新模型，速度快
- **`gpt-4-vision-preview`**: 早期版本

```yaml
model: gpt-4o
```

#### image_source（必填）
图片来源变量名：

```yaml
# 引用单张图片
image_source: product_photo

# 引用多张图片
image_source: site_photos  # 如果site_photos是数组
```

#### prompt_template（必填）
分析提示词，支持变量插值：

```yaml
prompt_template: |
  请分析这张产品图片，包括：
  1. 产品类型：{{product_type}}
  2. 外观质量
  3. 可见缺陷
  4. 总体评分
```

**提示词编写建议**：
- 明确具体的分析维度
- 使用项目列表结构化输出
- 适当限制输出长度
- 可以要求JSON格式输出

#### system_prompt（可选）
系统角色定义：

```yaml
system_prompt: "你是一个专业的产品质检专家，拥有10年的质量检测经验。"
```

#### temperature（可选）
创造性控制（0.0-2.0）：

- **0.0-0.3**: 准确、一致（质检、数据提取）
- **0.4-0.7**: 平衡（默认）
- **0.8-2.0**: 创造性（描述、创意）

```yaml
temperature: 0.3  # 质检场景，要求准确
```

#### max_tokens（可选）
最大输出长度：

```yaml
max_tokens: 500  # 约300-400个中文字
```

### 完整示例

```yaml
quality_analysis:
  type: string
  source: vision_ai
  description: 产品质量AI分析
  dependencies: [product_photo, product_category]
  vision_ai_config:
    model: gpt-4o
    image_source: product_photo
    system_prompt: "你是专业的{{product_category}}产品质检专家"
    prompt_template: |
      请对这个{{product_category}}产品进行质量检测：
      
      检测维度：
      1. 整体外观评价
      2. 表面缺陷（划痕、凹陷、污渍等）
      3. 颜色一致性
      4. 尺寸比例
      
      输出格式：
      - 缺陷描述：[详细描述发现的问题]
      - 严重程度：[轻微/中等/严重]
      - 质量评分：[1-10分]
      - 建议：[处理建议]
    temperature: 0.3
    max_tokens: 600
```

## 使用示例

### 示例1：产品质检报告

#### 变量配置

```yaml
# 1. 用户输入
product_id:
  type: string
  source: user_input
  description: 产品ID
  required: true

# 2. 获取产品照片
product_photo:
  type: image
  source: image
  description: 产品照片
  dependencies: [product_id]
  image_config:
    endpoint: "https://api.company.com/products/{{product_id}}/photo"
    output_format: base64
    timeout: 30

# 3. AI质量分析
quality_report:
  type: string
  source: vision_ai
  description: AI质量检测报告
  dependencies: [product_photo]
  vision_ai_config:
    model: gpt-4o
    image_source: product_photo
    prompt_template: |
      请对这个产品进行质量检测，识别以下问题：
      1. 外观缺陷（划痕、污渍、变形）
      2. 颜色是否均匀
      3. 边缘是否整齐
      4. 整体质量评分（1-10分）
    temperature: 0.3
    max_tokens: 500
```

#### 模板内容

```markdown
# 产品质量检测报告

## 基本信息
- 产品ID: {{product_id}}
- 检测时间: {{current_time}}
- 检测人员: AI自动检测

## 产品照片
{{product_photo.markdown}}

## 质量分析
{{quality_report}}

## 检测结论
基于AI分析结果，该产品质量状况如上所示。
```

### 示例2：现场安全检查（多张图片）

#### 变量配置

```yaml
# 1. 现场ID
site_id:
  type: string
  source: user_input
  description: 现场编号

# 2. 批量获取现场照片
site_photos:
  type: array
  source: image
  description: 现场照片集（3-5张）
  dependencies: [site_id]
  image_config:
    endpoint: "https://api.safety.com/sites/{{site_id}}/photos"
    multiple: true  # 批量模式
    output_format: url

# 3. AI安全分析
safety_analysis:
  type: string
  source: vision_ai
  description: 安全隐患分析
  dependencies: [site_photos]
  vision_ai_config:
    model: gpt-4o
    image_source: site_photos
    system_prompt: "你是资深的安全检查员，擅长识别工作现场的各类安全隐患"
    prompt_template: |
      请分析这些现场照片，识别以下安全问题：
      
      1. 人员防护
         - 是否佩戴安全帽
         - 是否穿着安全鞋
         - 是否有防护服
      
      2. 设备设施
         - 防护栏是否完整
         - 安全标识是否清晰
         - 消防器材是否就位
      
      3. 现场环境
         - 物品堆放是否规范
         - 通道是否畅通
         - 是否有明显危险源
      
      请列出发现的所有安全隐患，并按严重程度分类：
      - 严重隐患（红色预警）
      - 一般隐患（黄色预警）
      - 轻微问题（蓝色提示）
      
      每个问题给出具体的整改建议。
    temperature: 0.5
    max_tokens: 1000
```

#### 模板内容

```markdown
# 现场安全检查报告

## 检查信息
- 现场编号: {{site_id}}
- 检查时间: {{current_time}}
- 检查方式: AI视觉检测

## 现场照片
{% for photo in site_photos %}
### 照片 {{loop.index}}
![现场照片{{loop.index}}]({{photo.url}})
{% endfor %}

## 安全分析
{{safety_analysis}}

## 检查签字
检查员签字：__________  
日期：__________
```

## 常见问题

### Q1: 图片太大，Base64格式的Markdown文件很大怎么办？

**A**: 使用URL格式代替Base64：

```yaml
image_config:
  output_format: url  # 改为url格式
```

或者在API端先进行图片压缩。

---

### Q2: 图片需要认证才能访问怎么办？

**A**: 在headers中添加认证信息：

```yaml
image_config:
  endpoint: "https://api.example.com/secure/image.jpg"
  headers:
    Authorization: "Bearer {{access_token}}"
    X-API-Key: "{{api_key}}"
```

---

### Q3: 如何获取多张图片？

**A**: 两种方式：

**方式1：引用数组变量**
```yaml
# 先从API获取URL数组
image_urls:
  type: array
  source: api
  api_config:
    endpoint: "/api/images"
    response_mapping: "data.urls"

# 然后获取这些图片
images:
  type: array
  source: image
  dependencies: [image_urls]
  image_config:
    endpoint: "{{image_urls}}"
    multiple: true
```

**方式2：API直接返回多张**
```yaml
images:
  type: array
  source: image
  image_config:
    endpoint: "/api/batch-images?id={{resource_id}}"
    multiple: true
```

---

### Q4: Vision AI支持哪些图片格式？

**A**: 支持常见格式：
- PNG (.png)
- JPEG (.jpg, .jpeg)
- GIF (.gif)
- WebP (.webp)

图片大小限制：最大20MB（OpenAI限制）

---

### Q5: Vision AI分析失败怎么办？

**A**: 检查以下几点：

1. **API Key是否配置**：确保OpenAI API Key已设置
2. **图片是否可访问**：检查图片URL是否有效
3. **图片格式是否支持**：参考Q4
4. **Prompt是否合理**：过于复杂的prompt可能失败
5. **查看日志**：检查execution_logs表的错误信息

---

### Q6: 如何在Markdown中显示多张图片？

**A**: 使用Jinja2循环：

```jinja2
{% for image in site_photos %}
![照片{{loop.index}}]({{image.markdown}})

{% endfor %}
```

或横向排列：
```jinja2
<div style="display: flex;">
{% for image in photos %}
  <img src="{{image.url}}" style="width: 30%; margin: 1%;">
{% endfor %}
</div>
```

---

### Q7: Vision AI的成本如何？

**A**: 基于OpenAI定价：

- **gpt-4o**: 约$0.01-0.05 /图片（取决于图片大小和输出长度）
- **输入Token**: 图片 + Prompt
- **输出Token**: AI生成的文字

建议：
- 使用较小的图片
- 控制max_tokens避免过多输出
- 温度设低（0.3）减少不确定性

---

### Q8: 如何调试Vision AI的Prompt？

**A**: 渐进式调试：

1. **从简单开始**：
   ```
   请描述这张图片。
   ```

2. **逐步细化**：
   ```
   请描述这张产品图片的外观。
   ```

3. **添加具体要求**：
   ```
   请描述产品的：
   1. 颜色
   2. 材质
   3. 形状
   ```

4. **结构化输出**：
   ```
   请以JSON格式输出：
   {
     "color": "...",
     "material": "...",
     "defects": []
   }
   ```

---

### Q9: 图片获取超时怎么办？

**A**: 调整timeout参数：

```yaml
image_config:
  timeout: 60  # 增加到60秒
```

或检查：
- 图片服务器是否响应慢
- 网络连接是否稳定
- 图片是否过大

---

### Q10: 如何在模板中只显示图片URL而不是图片本身？

**A**: 直接引用url字段：

```markdown
图片地址：{{product_photo.url}}
```

而不是：
```markdown
{{product_photo.markdown}}  # 这会显示图片
```
