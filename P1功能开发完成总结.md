## ✅ Phase 1: 任务控制功能

### 后端实现

#### 1. 取消检查机制
- **文件**: `backend/app/services/context.py`
- **功能**: ExecutionContext类增加取消标志管理
  - `set_cancellation_flag(task_id)` - 设置取消标志
  - `is_cancelled(task_id)` - 检查是否已取消
  - `clear_cancellation_flag(task_id)` - 清除取消标志
  - `is_task_cancelled()` - 实例方法检查当前任务

#### 2. BaseExecutor集成取消检查
- **文件**: `backend/app/executors/base.py`
- **功能**: 在变量执行前后检查取消状态
  - 执行前检查取消标志
  - 执行后再次检查（长时间执行的变量）
  - 捕获`TaskCancelledException`异常

#### 3. 取消任务API
- **端点**: `POST /api/reports/tasks/{task_id}/cancel`
- **文件**: `backend/app/api/reports.py`
- **功能**:
  - 只允许取消pending/running状态的任务
  - 设置取消标志到ExecutionContext
  - 更新任务状态为cancelled
  - 更新运行中的变量状态为cancelled
  - 推送WebSocket取消事件

#### 4. 重试变量API
- **端点**: `POST /api/reports/tasks/{task_id}/variables/{variable_name}/retry`
- **文件**: `backend/app/api/reports.py`
- **功能**:
  - 只允许重试failed/cancelled状态的变量
  - 重置变量状态为pending
  - 异步触发变量重新执行
  - 复用已成功的其他变量结果
  - 如果所有变量成功，继续生成报告

#### 5. API Schema扩展
- **文件**: `backend/app/schemas/api_schemas.py`
- **新增**:
  - `TaskCancelRequest` - 取消任务请求
  - `TaskCancelResponse` - 取消任务响应
  - `VariableRetryResponse` - 重试变量响应

### 前端实现

#### 1. API集成
- **文件**: `frontend/src/api/reports.ts`
- **新增**:
  - `cancelTask(taskId, reason?)` - 取消任务
  - `retryVariable(taskId, variableName)` - 重试变量

#### 2. 任务进度页面更新
- **文件**: `frontend/src/pages/generate/ReportProgress.tsx`
- **新增功能**:
  - 取消任务按钮（带确认对话框）
  - 变量级别重试按钮
  - 查看日志按钮
  - 取消状态显示
  - WebSocket事件处理（task_cancelled、variable_retry_started）

## ✅ Phase 2: 执行日志系统

### 后端实现

#### 1. 日志服务
- **文件**: `backend/app/services/execution_logger.py`
- **功能**: ExecutionLogger类
  - `log(task_id, message, level, variable_name?, context?)` - 写入日志
  - `debug/info/warning/error()` - 便捷方法
  - 同时写入数据库和标准日志
  - 异步写入避免阻塞

#### 2. BaseExecutor集成日志
- **文件**: `backend/app/executors/base.py`
- **日志记录点**:
  - 变量开始执行 (INFO)
  - 变量执行成功 (INFO + duration_ms)
  - 变量执行失败 (ERROR + error_type)
  - 依赖检查失败 (WARNING)
  - 任务取消 (WARNING)
  - 使用默认值 (INFO)

#### 3. 日志查询API
- **端点**: `GET /api/reports/tasks/{task_id}/logs`
- **文件**: `backend/app/api/reports.py`
- **参数**:
  - `level` - 按日志级别筛选（DEBUG/INFO/WARNING/ERROR）
  - `variable_name` - 按变量名筛选
  - `limit` - 限制返回数量（默认100，最大1000）
  - `offset` - 分页偏移
- **返回**: 日志列表和总数

#### 4. API Schema扩展
- **文件**: `backend/app/schemas/api_schemas.py`
- **新增**:
  - `ExecutionLogItem` - 单条日志项
  - `ExecutionLogListResponse` - 日志列表响应

### 前端实现

#### 1. API集成
- **文件**: `frontend/src/api/reports.ts`
- **新增**:
  - `getTaskLogs(params)` - 获取执行日志

#### 2. 日志查看页面
- **文件**: `frontend/src/pages/logs/TaskLogs.tsx`
- **功能**:
  - 日志列表展示（Table）
  - 日志级别筛选（Select）
  - 变量名筛选
  - 分页功能
  - 上下文详情查看（JSON格式）
  - 日志级别彩色标签
  - 返回任务详情按钮

#### 3. 路由配置
- **文件**: `frontend/src/App.tsx`
- **新增**: `/logs/:taskId` 路由

## ✅ Phase 3: 模板校验功能

### 后端实现

#### 1. 模板校验服务
- **文件**: `backend/app/services/template_validator.py`
- **功能**: TemplateValidator类
  - **Jinja2语法校验**: 使用`jinja2.Environment.parse()`
  - **变量引用检查**: 模板中使用的变量是否都在元数据中定义
  - **元数据结构检查**: 必需字段（type、source）、source配置
  - **依赖关系检查**: 依赖变量是否存在、循环依赖检测（DAG）
  - **Schema格式检查**: AI变量的schema是否符合JSON Schema格式

#### 2. ValidationIssue类
- **字段**:
  - `level` - 级别（error或warning）
  - `category` - 类别（syntax、metadata、dependency、schema）
  - `message` - 详细错误信息
  - `location` - 问题位置（可选）

#### 3. 模板校验API
- **端点**: `POST /api/templates/{template_id}/validate`
- **文件**: `backend/app/api/templates.py`
- **功能**:
  - 执行完整的模板校验
  - 返回valid标志和issues列表
  - 有错误时valid=false，仅有警告时valid=true

#### 4. API Schema扩展
- **文件**: `backend/app/schemas/api_schemas.py`
- **新增**:
  - `ValidationIssue` - 单个校验问题
  - `TemplateValidationResponse` - 校验结果响应

### 前端实现

#### 1. API集成
- **文件**: `frontend/src/api/templates.ts`
- **新增**:
  - `validateTemplate(templateId)` - 校验模板

#### 2. 模板编辑页面更新
- **文件**: `frontend/src/pages/templates/TemplateEdit.tsx`
- **新增功能**:
  - 校验模板按钮（仅编辑现有模板时显示）
  - 校验结果Modal
  - 错误/警告分类显示
  - 问题详情列表（级别、类别、位置、消息）
  - 成功/失败状态Alert

## 📖 使用指南

### 取消任务
1. 在任务进度页面（`/generate/:taskId`）
2. 当任务状态为pending或running时
3. 点击"取消任务"按钮
4. 确认取消操作
5. 任务和运行中的变量状态更新为cancelled

### 重试变量
1. 在任务进度页面
2. 展开失败的变量详情
3. 点击变量状态旁的"重试"按钮
4. 变量重新执行
5. 如果所有变量成功，自动完成报告生成

### 查看日志
1. 在任务进度页面点击"查看日志"按钮
2. 或直接访问`/logs/:taskId`
3. 可按日志级别筛选
4. 可查看变量级别日志
5. 点击"查看详情"查看上下文JSON

### 校验模板
1. 在模板编辑页面（`/templates/:templateId/edit`）
2. 点击"校验模板"按钮
3. 查看校验结果Modal
4. 错误和警告分别显示
5. 根据提示修复问题

## ⚠️ 已知限制

1. **日志存储**: 日志永久存储在数据库中，可能需要定期清理
2. **取消延迟**: 取消操作依赖检查点，可能有短暂延迟
3. **重试限制**: 不支持设置最大重试次数
4. **日志级别**: 当前为固定级别，不支持动态调整
