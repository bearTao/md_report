# 报告预览功能快速参考

## 🎯 核心功能

### ✅ 支持的Markdown功能

| 功能 | 语法 | 示例 |
|------|------|------|
| **表格** | `\| 列1 \| 列2 \|` | 参见下方 |
| **删除线** | `~~文本~~` | ~~删除~~ |
| **任务列表** | `- [x]` / `- [ ]` | - [x] 完成 |
| **自动链接** | 直接输入URL | https://example.com |
| **Base64图片** | `![](data:image/...)` | 嵌入式图片 |

## 📝 常用语法示例

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
| 1   | 2   | 3   |
```

**对齐方式**:
```markdown
| 左对齐 | 居中 | 右对齐 |

| Left   | Mid  | Right  |
```

```markdown
- [x] 已完成项目
- [ ] 未完成项目
- [x] 另一个已完成项
```

### 删除线

```markdown
~~旧内容~~
新内容
```

## 🔒 安全特性

### 自动过滤的内容

❌ **被阻止**:
- `<script>` 标签
- `javascript:` 链接
- `onerror` 等事件处理器
- `<iframe>`, `<object>` 等嵌入标签

✅ **被允许**:
- 安全的HTML标签（table, img等）
- `http://` 和 `https://` 链接
- Base64图片（仅限图片格式）

## 🐛 常见问题

### Q: 表格不显示？
A: 检查Markdown语法，确保有表头分隔行 `|-----|`

### Q: 图片不显示？
A: 验证Base64格式是否正确，必须是 `data:image/...;base64,...`

### Q: 链接没有自动转换？
A: 确保URL格式完整（包含 `http://` 或 `https://`）
