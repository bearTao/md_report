# 数据库连接问题修复说明

## 🐛 问题描述

用户报告在生成报告时遇到错误：
```
Variable 'overview' execution failed: SQL execution failed: 
Database connection 'microgrid_db' not registered
```

同时发现 `app.log` 没有日志输出。

## 🔍 问题分析

### 问题1: 数据库连接未注册

**根本原因**:
- 数据库连接配置通过前端界面保存在MySQL的 `db_connections` 表中
- 但SQL执行器使用的是内存中的 `db_connector` 单例对象
- 这两者**没有自动同步机制**

**调用链路**:
```
报告生成 → SqlExecutor → db_connector.execute_query('microgrid_db')
                            ↓
                       _engines 字典中查找
                            ↓
                       未找到 → 抛出错误
```

**问题示意**:
```python
# 前端创建连接 → 保存到MySQL db_connections表
POST /api/config/db-connections/

# 报告生成时
execute_report_generation()
  → SqlExecutor.execute()
    → db_connector.execute_query('microgrid_db')  # ❌ 未注册！
      → raise ValueError("Database connection 'microgrid_db' not registered")
```

### 问题2: 日志文件为空

**原因**: 之前的后端服务实例已关闭，日志文件被清空或重置。

**实际状态**: 日志系统配置正常，最新的服务启动日志已正常记录。

## ✅ 解决方案

### 修改内容

**文件**: `backend/app/api/reports.py`

**修改位置**: `execute_report_generation()` 函数

**添加的代码**:

```python
async def execute_report_generation(...):
    """Background task to execute report generation"""
    
    # ... 任务状态更新 ...
    
    try:
        # ✅ 新增：从数据库加载所有活动的数据库连接
        from app.models.db_models import DBConnection
        from app.connectors.database import db_connector
        
        db_connections = db_session.query(DBConnection).filter(
            DBConnection.is_active == "true"
        ).all()
        
        for db_conn in db_connections:
            try:
                # 构建连接URL
                engine_dialects = {
                    "postgresql": "postgresql+psycopg2",
                    "mysql": "mysql+pymysql",
                    "sqlserver": "mssql+pyodbc",
                    "oracle": "oracle+cx_oracle"
                }
                
                dialect = engine_dialects.get(db_conn.engine.value, db_conn.engine.value)
                from urllib.parse import quote_plus
                password = db_conn.password_ciphertext
                
                connection_url = f"{dialect}://{db_conn.username}:{quote_plus(password)}@{db_conn.host}:{db_conn.port}/{db_conn.database}"
                
                # 注册到内存连接池
                db_connector.register_connection(
                    name=db_conn.name,
                    connection_url_or_engine=connection_url,
                    pool_size=5,
                    max_overflow=10
                )
                
                print(f"Registered database connection: {db_conn.name}")
                
            except Exception as e:
                print(f"Failed to register connection {db_conn.name}: {str(e)}")
        
        # 继续原有的报告生成流程...
```

### 工作原理

```
报告生成开始
    ↓
1. 从MySQL加载所有活动连接配置
    ↓
2. 遍历每个连接，构建连接URL
    ↓
3. 注册到 db_connector._engines 字典
    ↓
4. 继续执行变量解析和SQL查询
    ↓
SqlExecutor 查询时，连接已存在 ✅
```

### 连接池管理

```python
# 定期清理未使用的连接
async def cleanup_unused_connections():
    current_time = time.time()
    for name, (engine, last_used) in db_connector._engines.items():
        if current_time - last_used > 3600:  # 1小时未使用
            engine.dispose()
            del db_connector._engines[name]
```
