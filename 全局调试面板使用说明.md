# 全局调试面板使用说明

## 📖 功能概述

系统在**主界面右侧**提供了一个全局调试面板，您可以在**任意页面**直接测试模板渲染效果，无需创建或选择模板。

## 🎨 界面位置

调试面板位于：
- **主界面最右侧**（宽度400px）
- 所有页面都可见（全局固定）
- 独立于当前页面内容

**布局结构**：
```
+--------+------------------+------------+
| 左侧菜单 |    主内容区       |  调试面板   |
| (200px)|   (自适应宽度)    |  (400px)   |
+--------+------------------+------------+
```

## 🚀 使用步骤

### 1. 编辑模板内容

在调试面板顶部的"模板内容"编辑器中输入Jinja2模板，例如：
```jinja2
# {{title}}

## 内容
{{content}}
```

### 2. 配置变量元数据

在"变量元数据"编辑器中定义变量（YAML格式），例如：
```yaml
title:
  type: string
  source: user_input
  required: true
  description: 标题

content:
  type: string
  source: user_input
  required: true
  description: 内容
```

### 3. 填写用户输入

系统会自动根据元数据生成用户输入表单：
- 只显示 `source: user_input` 的变量
- 必填项会有红色星号标记
- 填写完成后点击"执行调试"

### 4. 查看结果

- **渲染结果**：显示最终生成的Markdown报告
- **变量详情**：显示每个变量的执行状态和耗时

## 📝 使用场景

### 场景1：快速测试语法
在编写模板前，先在调试面板测试Jinja2语法是否正确。

### 场景2：验证变量配置
测试SQL查询、API调用、AI配置是否正确工作。

### 场景3：学习和实验
新用户可以在调试面板中学习模板语法和变量配置，无需创建实际模板。

### 场景4：临时报告
快速生成一次性报告，无需保存模板。

## ⚠️ 重要提醒

### 真实执行
点击"执行调试"会：
- ✅ 真实连接数据库执行SQL
- ✅ 真实调用外部API
- ✅ 真实调用AI模型（**消耗API配额**）

### 不会保存
- 调试配置和结果不会保存到数据库
- 刷新页面后内容会重置
- 如需保存，请到"模板管理"页面创建模板

## 💡 使用技巧

### 技巧1：复制现有模板
1. 打开模板编辑页面
2. 复制模板内容和元数据
3. 粘贴到调试面板测试

### 技巧2：分步测试
- 先只用 user_input 变量测试模板语法
- 确认无误后再添加 SQL/API/AI 变量

### 技巧3：快速原型
在创建正式模板前，先在调试面板设计和测试。

### 技巧4：教学演示
向新用户演示系统功能时，使用调试面板实时展示。

## 🛠️ 故障排查

### 问题1：YAML解析错误
**现象**: 出现红色错误提示

**解决**: 检查YAML格式，常见错误：
- 缩进不正确（必须使用空格，不能用Tab）
- 冒号后缺少空格
- 字符串包含特殊字符未加引号

### 问题2：执行调试无反应
**可能原因**:
- 必填的用户输入未填写
- 后端服务未启动

**解决**: 
- 检查表单必填项
- 运行 `python3 verify_services.py` 检查后端

### 问题3：变量执行失败
**查看**: 变量详情表格中的状态列
- ✅ 绿色勾号 = 成功
- ❌ 红色叉号 = 失败

**常见原因**:
- SQL：数据库连接未配置
- API：网络问题或API密钥错误
- AI：OpenAI API密钥未配置
