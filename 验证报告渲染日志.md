## 5. 故障排查示例

### 场景：报告生成失败

1. **查看任务状态**：
```bash
curl http://localhost:8000/api/reports/tasks/{task_id}/status
```

2. **查看渲染日志**：
```bash
grep "task_{task_id}" logs/app.log | grep -E "render|Render"
```

3. **检查错误详情**：
```bash
# 查看完整错误堆栈
grep "task_{task_id}" logs/app.log | grep -A 20 "ERROR"
```

### 典型错误模式

**变量是函数对象**：
```
ERROR - Unexpected error during rendering: object of type 'builtin_function_or_method' has no len()
```
→ 检查变量执行结果，某个变量返回的是函数而不是数据

**模板语法错误**：
```
ERROR - Template rendering failed: unexpected 'end of template'
```
→ 检查模板的 Jinja2 语法，可能缺少 `{% endif %}` 等

**变量未定义**：
```
ERROR - Template rendering failed: 'variable_name' is undefined
```
→ 检查模板中使用的变量是否都已正确定义和执行

## 6. 日志级别配置建议

### 开发环境
```python
# backend/app/main.py
import logging

logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - [%(filename)s:%(lineno)d] - %(message)s'
)
```

### 生产环境
```python
# backend/app/main.py
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[
        logging.FileHandler('logs/app.log'),
        logging.StreamHandler()  # 同时输出到控制台
    ]
)
```

### 针对性调试
```python
# 只调试渲染模块
logging.getLogger('app.services.renderer').setLevel(logging.DEBUG)
logging.getLogger('app.api.reports').setLevel(logging.DEBUG)

# 其他模块保持 INFO
logging.getLogger().setLevel(logging.INFO)
```

## 7. 性能监控

通过日志可以监控渲染性能：

```bash
# 统计渲染耗时
grep "Template rendered successfully" logs/app.log | grep -oP '\d+ms' | sort -n

# 统计输出大小
grep "output size" logs/app.log | grep -oP '\d+ characters'

# 查找慢渲染（超过100ms）
grep "Template rendered successfully" logs/app.log | awk -F'in ' '{print $2}' | awk -F'ms' '{if ($1 > 100) print $0}'
```
