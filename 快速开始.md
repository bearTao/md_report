# 快速开始指南

## 系统要求

- Python 3.10+
- Node.js 18+
- npm 9+

## 一、启动后端服务

```bash
# 1. 进入后端目录
cd backend

# 2. 安装依赖（如果还没安装）
pip install -r requirements.txt

# 3. 启动后端服务
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端服务将运行在: http://localhost:8000

API文档: http://localhost:8000/docs

## 二、启动前端服务

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖（如果还没安装）
npm install

# 3. 启动前端服务
npm run dev
```

前端服务将运行在: http://localhost:5173

## 三、快速体验

### 1. 配置AI服务（可选）

1. 访问 http://localhost:5173/settings/ai
2. 输入您的 OpenAI API Key（格式：sk-...）
3. 点击"保存配置"

> 如果不配置AI，仍然可以使用其他类型的变量（user_input、system）

### 2. 创建第一个模板

1. 访问 http://localhost:5173/templates
2. 点击"创建模板"
3. 填写以下信息：

**模板名称**: 我的第一个报告

**模板描述**: 简单的测试报告模板

**模板内容**:
```jinja2
# {{title}}

生成时间: {{generation_time}}

## 内容

{{content}}

---
生成于 Markdown 报告生成平台
```

**变量元数据**:
```yaml
title:
  type: string
  source: user_input
  required: true
  description: 报告标题
  ui_config:
    input_type: text
    placeholder: 请输入报告标题

content:
  type: string
  source: user_input
  required: true
  description: 报告内容
  ui_config:
    input_type: textarea
    placeholder: 请输入报告内容

generation_time:
  type: string
  source: system
  required: true
  description: 生成时间
  system_config:
    fields:
      timestamp:
        generator: datetime
        format: "%Y-%m-%d %H:%M:%S"
```

4. 点击"保存模板"

### 3. 生成报告

1. 访问 http://localhost:5173/generate
2. 选择刚创建的模板
3. 填写表单：
   - 报告标题: 我的第一份报告
   - 报告内容: 这是我使用 Markdown 报告生成平台创建的第一份报告！
4. 点击"开始生成"
5. 等待生成完成（通常只需几秒）

### 4. 查看报告

1. 生成完成后，点击"查看报告"
2. 查看渲染后的 Markdown 报告
3. 点击"下载报告"保存到本地

## 五、进阶使用

### 创建带AI生成的模板

如果已配置 OpenAI API Key，可以创建包含AI生成的模板：

```yaml
# 在元数据中添加AI生成变量
project_name:
  type: string
  source: user_input
  required: true
  description: 项目名称

ai_summary:
  type: string
  source: ai_generation
  required: true
  description: AI生成的项目总结
  dependencies: [project_name]
  ai_config:
    model: gpt-3.5-turbo
    prompt_template: |
      请为以下项目生成一段简短的介绍（不超过100字）：
      项目名称：{{project_name}}
      
      要求：
      1. 描述项目的核心功能
      2. 突出项目的创新点
      3. 语言简洁专业
    temperature: 0.7
  schema:
    type: string
```

### 使用SQL数据源

```yaml
user_count:
  type: number
  source: sql
  required: true
  description: 用户总数
  sql_config:
    connection_name: default
    query: "SELECT COUNT(*) as count FROM users"
```

> 注意：P0版本的SQL功能需要在代码中配置数据库连接，P1将提供界面配置

### 使用API数据源

```yaml
weather:
  type: object
  source: api
  required: false
  description: 天气信息
  api_config:
    url: "https://api.weather.com/v1/current?city={{city}}"
    method: GET
    headers:
      Authorization: "Bearer YOUR_API_KEY"
```

## 六、常见问题

### Q: 前端无法连接后端？
A: 
1. 确认后端运行在 http://localhost:8000
2. 检查 frontend/.env.development 中的 VITE_API_BASE_URL 配置
3. 查看浏览器控制台的错误信息

### Q: 模板创建失败？
A:
1. 检查 YAML 格式是否正确
2. 验证 Jinja2 语法
3. 查看后端日志获取详细错误

### Q: 报告生成失败？
A:
1. 检查必填的 user_input 变量是否都已填写
2. 如果使用 AI，确认 API Key 已正确配置
3. 查看进度页面的变量执行详情了解失败原因

### Q: 下载报告失败？
A:
1. 确认报告已成功生成（状态为 success）
2. 检查浏览器的下载设置
3. 尝试刷新页面重新下载
