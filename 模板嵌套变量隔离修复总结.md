# 模板嵌套变量隔离修复总结

## 📋 问题描述

**Bug**: 嵌套模板中同名变量会相互影响

**现象**:
- 主模板有 `title` 变量
- 子模板也有 `title` 变量
- 用户输入一个值，两个模板都使用同一个值 ❌

**期望行为**:
- 每个模板的变量完全独立
- 用户分别为主模板和子模板的 `title` 输入不同的值
- 渲染时各自使用自己的值 ✅

---

## 🎯 实施方案

### 核心思路

使用**嵌套inputs结构**：
```python
{
    "main_template_id": {"title": "主标题"},
    "sub_template_id": {"title": "子标题"}
}
```

每个模板只使用 `nested_inputs[template_id]` 中的变量，实现完全隔离。

---

## 📝 修改内容

### 1. 后端 - renderer.py

**修改**: `_resolve_includes()` 和 `_execute_and_render_template()`

**关键变化**:
```python
# 原来：所有模板共享同一个 user_inputs
await self._execute_and_render_template(..., user_inputs)

# 现在：每个模板使用自己的 user_inputs
sub_user_inputs = nested_user_inputs.get(template_id, {})
await self._execute_and_render_template(..., sub_user_inputs)
```

### 2. 后端 - reports.py

**修改**: `execute_report_generation()`

**新增逻辑**:
```python
# 检测inputs结构（兼容旧格式）
if template_id in user_inputs and isinstance(user_inputs[template_id], dict):
    # 嵌套结构
    nested_user_inputs = user_inputs
    main_template_inputs = user_inputs[template_id]
else:
    # 扁平结构（向后兼容）
    nested_user_inputs = {template_id: user_inputs}
    main_template_inputs = user_inputs
```

### 3. 前端 - GenerateReport.tsx

**新增功能**:

1. **扫描include标签**:
```typescript
useEffect(() => {
    const includePattern = /{%\s*include\s+"([^"]+)"\s*%}/g;
    const matches = [...template.template_content.matchAll(includePattern)];
    // 加载所有被include的模板
}, [template]);
```

2. **分层表单显示**:
```tsx
{/* 主模板变量 */}
<Card title="主模板: xxx">
    {mainTemplate.variables.map(...)}
</Card>

{/* 子模板变量 */}
{includedTemplates.map(temp => (
    <Card title={`子模板: ${temp.name}`}>
        {temp.variables.map((name) =>
            renderFormItem(`${temp.id}__${name}`, config)
        )}
    </Card>
))}
```

3. **构建嵌套inputs**:
```typescript
const finalInputs = {
    [mainTemplateId]: {},
};

for (const [key, value] of Object.entries(values)) {
    if (key.includes('__')) {
        const [templateId, varName] = key.split('__');
        finalInputs[templateId][varName] = value;
    } else {
        finalInputs[mainTemplateId][key] = value;
    }
}
```

## ⚙️ 技术要点

### 1. 向后兼容

保持对旧格式的支持：
```python
# 检测结构
if template_id in user_inputs and isinstance(user_inputs[template_id], dict):
    # 嵌套结构
else:
    # 扁平结构（旧格式）
```

### 2. 变量名前缀

前端使用 `template_id__variable_name` 作为表单field name：
```typescript
renderFormItem(`${template.id}__title`, config)
```

提交时split还原：
```typescript
const [templateId, varName] = key.split('__', 2);
finalInputs[templateId][varName] = value;
```

### 3. 递归扫描

前端只扫描主模板的一层include（不递归），简化实现：
```typescript
const matches = [...template.template_content.matchAll(includePattern)];
```

后端递归处理所有层级。

## 📚 使用示例

### 示例：创建月报（主模板）+ 销售总结（子模板）

**子模板** (ID: `sales_summary`):
```jinja2
销售数据：
- 负责人：{{owner}}
- 销售额：{{amount}}元
```

元数据：
```yaml
owner:
  type: string
  source: user_input
  description: 销售负责人

amount:
  type: number
  source: user_input
  description: 销售额
```

**主模板** (ID: `monthly_report`):
```jinja2
# {{month}}月度报告

负责人：{{owner}}

{% include "sales_summary" %}
```

元数据：
```yaml
month:
  type: string
  source: user_input
  description: 月份

owner:
  type: string
  source: user_input
  description: 报告负责人
```

**生成报告时的表单**:

```
主模板: 月度报告
  月份: [10]
  报告负责人: [张三]

子模板: 销售总结
  销售负责人: [李四]
  销售额: [100000]
```

**生成结果**:
```markdown
# 10月度报告

负责人：张三

销售数据：
- 负责人：李四
- 销售额：100000元
```

注意：主模板的 `owner` 是"张三"，子模板的 `owner` 是"李四"，完全独立！
