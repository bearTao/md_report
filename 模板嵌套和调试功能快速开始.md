## 📝 示例：创建带嵌套的模板

### 步骤1：创建子模板

1. 访问模板列表，点击"新建模板"
2. 填写：
   - 名称：`问候语模板`
   - 描述：`可复用的问候语组件`
   - ID: `greeting`（创建后在列表中查看）

3. 模板内容：
```jinja2
你好，{{user_name}}！欢迎使用报告系统。
```

4. 元数据：
```yaml
user_name:
  type: string
  source: user_input
  required: true
  description: 用户姓名
  ui_config:
    input_type: text
```

5. 点击"保存模板"

### 步骤2：创建主模板（使用嵌套）

1. 再次点击"新建模板"
2. 填写：
   - 名称：`月度报告`
   - 描述：`包含问候语的月度报告`

3. 模板内容：
```jinja2
# {{report_title}}

## 欢迎信息
{% include "greeting" %}

## 报告正文
本月完成任务数：{{task_count}}个
```

4. 元数据：
```yaml
report_title:
  type: string
  source: user_input
  required: true
  description: 报告标题
  ui_config:
    input_type: text

user_name:
  type: string
  source: user_input
  required: true
  description: 用户姓名
  ui_config:
    input_type: text

task_count:
  type: number
  source: user_input
  required: true
  description: 任务数量
  ui_config:
    input_type: number
```

5. **在右侧调试面板测试**：
   - 填写：
     - 报告标题：`2024年10月报告`
     - 用户姓名：`张三`
     - 任务数量：`15`
   - 点击"执行调试"
   - 查看渲染结果（应该包含问候语）

6. 确认无误后点击"保存模板"

## ⚠️ 注意事项

### 调试功能

1. **真实执行**：点击"执行调试"会：
   - 真实连接数据库执行SQL
   - 真实调用API
   - 真实调用AI模型（**消耗API配额**）

2. **不会保存**：调试配置和结果不会保存，仅用于测试

3. **性能影响**：
   - AI调用可能需要几秒钟
   - SQL查询取决于数据库性能
   - API调用取决于网络延迟

### 模板嵌套

1. **变量作用域**：
   - 子模板只能访问自己元数据中定义的变量
   - 用户输入在所有模板间共享

2. **性能考虑**：
   - 每个子模板都会独立执行变量
   - 同一子模板被多次include时会重复执行

3. **循环嵌套**：
   - 系统会自动检测并显示错误
   - 不会导致无限递归

## 💡 使用技巧

### 技巧1：快速测试语法
修改模板内容后，立即在右侧调试面板测试，无需保存。

### 技巧2：分步调试
先只用user_input变量测试模板语法，确认无误后再添加SQL/API/AI变量。

### 技巧3：查看执行时间
通过变量执行详情的"耗时"列，找出性能瓶颈。

### 技巧4：复用子模板
将常用的报告组件（如页眉、表格格式等）保存为独立模板，在多个报告中复用。
