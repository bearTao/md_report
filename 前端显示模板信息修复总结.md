# 前端显示模板信息修复总结

## 问题描述

用户反馈：在 `/generate/task_xxx` 页面能看到所有变量的执行情况，但不知道该变量隶属于哪个模板。

## 解决方案

### 修改的文件

**`frontend/src/pages/generate/ReportProgress.tsx`**

### 修改内容

#### 1. 在变量列表的标题栏添加模板路径标签

在Timeline每个项的header中添加蓝色模板路径标签：

```tsx
<Panel
  header={
    <Space>
      <strong>{variable.variable_name}</strong>
      <Tag>{variable.source}</Tag>
      {/* ✅ 新增：显示模板路径 */}
      {variable.template_path && (
        <Tag color="blue" style={{ maxWidth: 200 }} title={variable.template_path}>
          {variable.template_path}
        </Tag>
      )}
      {getStatusTag(variable.status)}
      {variable.duration_ms && (
        <span style={{ color: '#999' }}>
          {formatDuration(variable.duration_ms)}
        </span>
      )}
    </Space>
  }
>
```

#### 2. 在变量详情中添加"所属模板"字段

在展开的详情描述中添加所属模板信息：

```tsx
<Descriptions column={1} size="small">
  <Descriptions.Item label="变量名称">
    {variable.variable_name}
  </Descriptions.Item>
  <Descriptions.Item label="数据源">
    {variable.source}
  </Descriptions.Item>
  {/* ✅ 新增：所属模板字段 */}
  {variable.template_path && (
    <Descriptions.Item label="所属模板">
      <Tag color="blue">{variable.template_path}</Tag>
    </Descriptions.Item>
  )}
  <Descriptions.Item label="状态">
    ...
  </Descriptions.Item>
</Descriptions>
```

#### 3. 按模板分组显示变量（核心改进）

将所有变量按 `template_path` 分组显示，每个模板一个区块：

```tsx
{/* 按模板分组显示变量 */}
{(() => {
  // 按 template_path 分组
  const groupedVariables: Record<string, TaskVariableDetail[]> = {};
  taskStatus.variables.forEach((variable: TaskVariableDetail) => {
    const templateKey = variable.template_path || '主模板';
    if (!groupedVariables[templateKey]) {
      groupedVariables[templateKey] = [];
    }
    groupedVariables[templateKey].push(variable);
  });

  return Object.entries(groupedVariables).map(([templatePath, variables]) => (
    <div key={templatePath} style={{ marginBottom: 24 }}>
      {/* 模板分组标题 */}
      <div style={{ 
        marginBottom: 16, 
        padding: '8px 12px', 
        background: '#f0f2f5', 
        borderRadius: 4,
        display: 'flex',
        alignItems: 'center',
        gap: 8
      }}>
        <strong style={{ fontSize: 14 }}>📁 {templatePath}</strong>
        <Tag color="blue">{variables.length} 个变量</Tag>
      </div>
      
      {/* 该模板的变量列表 */}
      <Timeline>
        {variables.map((variable: TaskVariableDetail) => (
          // ... 变量详情 ...
        ))}
      </Timeline>
    </div>
  ));
})()}
```

## 技术要点

### 1. 分组逻辑
使用 JavaScript 的 `Record` 类型按 `template_path` 分组：

```typescript
const groupedVariables: Record<string, TaskVariableDetail[]> = {};
taskStatus.variables.forEach((variable: TaskVariableDetail) => {
  const templateKey = variable.template_path || '主模板';
  if (!groupedVariables[templateKey]) {
    groupedVariables[templateKey] = [];
  }
  groupedVariables[templateKey].push(variable);
});
```

### 2. 空值处理
- 如果 `template_path` 为空或 null，显示为"主模板"
- 只在有值时显示模板路径标签

### 3. 样式优化
- 模板路径标签设置 `maxWidth: 200` 避免过长
- 使用 `title` 属性显示完整路径（悬停提示）
- 分组标题使用灰色背景区分
