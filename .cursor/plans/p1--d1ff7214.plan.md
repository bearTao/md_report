# 修复图片功能Bug计划

## 问题分析

根据错误日志和用户反馈，发现3个关键问题：

1. **数据库枚举不匹配** - `source` 字段无法存储 `IMAGE` 和 `VISION_AI`
   ```
   DataError: (1265, "Data truncated for column 'source' at row 1")
   INSERT INTO generation_task_variables ... source='IMAGE'
   ```

2. **后台任务Session错误** - Request-scoped session在请求结束后关闭
   ```
   PendingRollbackError: This Session's transaction has been rolled back
   ```

3. **数据库连接过多** - 缺少连接池配置和Session管理

## 修复步骤

### 1. 修复数据库枚举类型

**文件**: `backend/fix_enum_migration.sql` (新建)

创建SQL脚本修改 `generation_task_variables` 表的 `source` 字段：

```sql
-- 修改枚举类型以支持新的变量源
ALTER TABLE generation_task_variables 
MODIFY COLUMN source 
ENUM('user_input', 'sql', 'api', 'ai_generation', 'system', 'image', 'vision_ai') 
NOT NULL;

-- 验证修改
SHOW COLUMNS FROM generation_task_variables LIKE 'source';
```

**执行方式**：

```bash
mysql -h 10.10.20.10 -P 24406 -u root -p123456 md_agent < backend/fix_enum_migration.sql
```

### 2. 修复后台任务Session管理

**文件**: `backend/app/api/reports.py`

**问题代码** (第469行):

```python
background_tasks.add_task(
    execute_report_generation,
    ...
    db_session=db,  # 错误：传递request-scoped session
    ...
)
```

**修复方案A** - 修改 `execute_report_generation` 函数 (第221行):

移除 `db_session` 参数，在函数内创建新session：

```python
async def execute_report_generation(
    task_id: str,
    template_id: str,
    template_content: str,
    metadata: Dict[str, Any],
    user_inputs: Dict[str, Any],
    # 移除 db_session: Session,
    openai_api_key: Optional[str] = None,
    openai_api_base: Optional[str] = None
):
    # 创建新的database session
    from app.database import SessionLocal
    db_session = SessionLocal()
    
    try:
        # 原有逻辑保持不变
        task = db_session.query(GenerationTask).filter(
            GenerationTask.id == task_id
        ).first()
        # ... 其他代码 ...
        
    except Exception as e:
        # 错误处理
        if task:
            task.status = ReportStatus.FAILED
            db_session.commit()
        raise
    finally:
        # 关闭session
        db_session.close()
```

**修复方案B** - 修改 `generate_report` 函数调用 (第469行):


### 4. 添加Session错误处理

**文件**: `backend/app/api/reports.py`

在 `execute_report_generation` 的异常处理中添加Session回滚：

```python
except Exception as e:
    logger.error(f"Report generation failed: {str(e)}")
    
    try:
        # 回滚失败的事务
        db_session.rollback()
        
        # 更新任务状态
        task = db_session.query(GenerationTask).filter(
            GenerationTask.id == task_id
        ).first()
        if task:
            task.status = ReportStatus.FAILED
            db_session.commit()
    except Exception as commit_error:
        logger.error(f"Failed to update task status: {commit_error}")
    
    raise
```
