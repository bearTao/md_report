## 🎯 实现功能

### 1. ✅ 模板嵌套/包含功能

**核心特性**:
- 支持 `{% include "template_id" %}` 语法嵌套其他模板
- 每个子模板独立执行其变量定义
- 子模板只能访问其自己元数据中定义的变量
- 自动检测并防止循环嵌套
- 支持多级嵌套（A包含B，B包含C）
- 错误处理优雅降级（显示错误信息而不中断）

**实现文件**:
- `backend/app/services/renderer.py`
  - `_resolve_includes()` - 递归解析include标签
  - `_execute_and_render_template()` - 独立执行子模板
- `backend/app/api/reports.py` - 集成include解析到报告生成

**测试状态**: ✅ 6/6 通过

## 🚀 使用方法

### 访问调试页面

1. 在浏览器中访问系统主页
2. 点击左侧菜单的"**调试测试**"（🐛图标）
3. 进入独立的调试页面

### 使用步骤

1. **编辑模板内容**（左上）
   - 使用Monaco编辑器编写Jinja2模板
   - 支持语法高亮和自动补全

2. **配置变量元数据**（左中）
   - 使用YAML格式定义变量
   - 实时语法检查

3. **填写用户输入**（左下）
   - 系统自动生成表单（仅user_input类型）
   - 填写必填项

4. **执行调试**
   - 点击"执行调试"按钮
   - 等待变量执行完成

5. **查看结果**
   - 右上：查看Markdown渲染效果
   - 右下：查看变量执行详情（状态/耗时/结果）

## ⚙️ 技术实现

### 模板嵌套流程

```
主模板 → _resolve_includes() → 查找include标签
  ↓
for each include:
  - 检查循环嵌套
  - 加载子模板
  - 递归解析子模板includes
  - 创建独立ExecutionContext
  - 执行子模板变量
  - 渲染子模板
  - 替换include标签
  ↓
返回完全解析的模板 → 主模板渲染
```

### 调试API流程

```
前端（DebugTest页面）:
  用户编辑 → 点击"执行调试"
  ↓
  POST /api/debug/render

后端:
  解析YAML → 创建ExecutionContext
  ↓
  执行所有变量（真实SQL/API/AI）
  ↓
  解析includes
  ↓
  渲染模板
  ↓
  返回结果

前端:
  显示渲染结果 + 变量详情
```
