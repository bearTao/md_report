### Bug 2: Datetime对象切片错误 ❌ → ✅

**问题描述：**
```jinja2
{{ overview.starttime[:10] if overview.starttime else '-' }}
```
- SQL返回的是 `datetime.datetime` 对象，不是字符串
- Python的datetime对象不支持切片操作 `[:10]`
- 错误: `TypeError: 'datetime.datetime' object is not subscriptable`

**位置：**
- 模板第22行
- 模板第496行

**解决方案：**
使用 `strftime()` 方法格式化日期：
```jinja2
{{ overview.starttime.strftime('%Y-%m-%d') if overview.starttime else '-' }}
{{ overview.endtime.strftime('%Y-%m-%d') if overview.endtime else '-' }}
```

---

### Bug 3: 字段名冲突 - `values` ❌ → ✅

**问题描述：**
- SQL查询返回的字段名为 `values`
- Python字典有一个内置方法也叫 `values()`
- Jinja2在访问 `score.values` 时混淆了字段和方法
- 错误: `TypeError: must be real number, not builtin_function_or_method`

**位置：**
- 模板第80行: `score.values`
- SQL查询: `SELECT "index", "values", index_weight, index_score, index_deduction FROM micro_grid_index_score_w`

**解决方案：**
1. 修改SQL查询，给 `values` 字段添加别名：
```sql
SELECT 
  `index`,
  `values` as value_data,  -- 使用别名避免冲突
  index_weight,
  index_score,
  index_deduction
FROM microgrid.micro_grid_index_score_w
WHERE wgid = :wgid
ORDER BY `index`
```

2. 修改模板中所有 `score.values` 为 `score.value_data`
