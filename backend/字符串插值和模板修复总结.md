# 字符串插值和模板修复总结

## 修复日期
2025-10-20

## 问题描述

用户报告在生成报告时遇到以下错误：
```
AttributeError: 'ExecutionContext' object has no attribute '_variables'
```

经过排查，发现两个主要问题：

### 1. 字符串插值函数错误
**问题**: `app/services/context.py` 中的 `interpolate_string` 方法使用了错误的属性名 `self._variables`，而实际应该是 `self.variables`。

**影响**: 所有使用字符串插值的功能（AI提示词、SQL查询等）都会失败。

### 2. AI提示词包含不支持的Jinja2语法
**问题**: 部分AI变量的提示词包含Jinja2控制流语法（`{% if %}`、`{% for %}`、`{% set %}`），而 `interpolate_string` 只支持简单的 `{{variable}}` 替换和过滤器。

**影响**: AI变量执行时提示词插值失败。

---

## 修复内容

### 1. 修复 ExecutionContext 属性引用错误

**文件**: `backend/app/services/context.py`

**修改**: 将所有 `self._variables` 改为 `self.variables`

**详细变更**:
```python
# 修复前
if base_var not in self._variables:
    raise DependencyError(...)
value = self._variables[base_var]

# 修复后
if base_var not in self.variables:
    raise DependencyError(...)
value = self.variables[base_var]
```

**测试验证**:

```bash
✅ 嵌套属性访问 (overview.micro_grid_name)
✅ length过滤器 (items | length)

✅ 链式过滤器 (text | trim | upper)
✅ 嵌套属性 + 过滤器

```

### 2. 修复数据库模板中的AI提示词

**文件**: 数据库中的模板 `tpl_f202f946b0da`（微网格预分析_ai_pgsql）

**修改内容**:

#### a) executive_summary 变量
- **移除**: Jinja2控制流 `{% if %} {% for %} {% set %}`
- **移除**: 错误的变量引用 `{{score.index}}`、`{{score.index_deduction}}`
- **添加**: 依赖声明 `wgid`
- **添加**: 使用过滤器 `{{problem_buildings | length}}`

**修复前**:
```jinja2
## 主要失分项
{% if index_scores %}
{% set sorted_scores = index_scores | sort(attribute='index_deduction', reverse=true) %}
{% for score in sorted_scores[:3] %}
- {{score.index}}: 失分{{score.index_deduction}}分
{% endfor %}
{% endif %}
```

**修复后**:
```jinja2
## 问题统计
- 覆盖问题: {{overview.problem_count}} 个
- 干扰问题: {{overview.gr_problem_count}} 个
- 问题楼宇总数: {{problem_buildings | length}} 个
```

#### b) detailed_optimization_plan 变量
- **移除**: 所有 `{% if %} {% else %} {% endif %}` 控制流

#### c) problem_root_cause_analysis 变量
- **移除**: 所有 `{% if %} {% endif %}` 控制流

**使用工具**: SQLAlchemy `flag_modified()` 强制更新JSON字段

### 3. 创建MySQL版本的AI模板

**新模板ID**: `tpl_ai_mysql`
**名称**: 微网格预分析_ai_mysql

**目的**: 
- PostgreSQL版本模板需要 `microgrid_db_pg` 连接
- MySQL版本模板使用 `microgrid_db` 连接
- 方便不同环境使用

**创建方法**:
```python
# 复制PostgreSQL版本
# 将所有SQL变量的connection改为 'microgrid_db'
for var_name, var_meta in metadata.items():
    if var_meta.get('source') == 'sql':
        var_meta['sql_config']['connection'] = 'microgrid_db'
```

## 已知限制

### interpolate_string 不支持的功能
❌ Jinja2控制流（`{% if %}`, `{% for %}`, `{% set %}`）
❌ Jinja2注释（`{# comment #}`）
❌ 复杂的Jinja2过滤器（仅支持基础过滤器）

### 解决方案
如果需要复杂逻辑，应该：
1. 在SQL层面处理（使用`CASE WHEN`、聚合函数等）
2. 在AI提示词中用自然语言描述，让AI自己判断
3. 创建额外的SQL变量进行数据预处理

## 使用示例

### 前端调用API
```javascript
// 使用MySQL AI版本
POST /api/reports/generate
{
  "template_id": "tpl_ai_mysql",
  "inputs": {
    "wgid": "ZQGY0174"
  }
}

// 使用PostgreSQL AI版本
POST /api/reports/generate
{
  "template_id": "tpl_f202f946b0da",
  "inputs": {
    "wgid": "ZQGY0174"
  }
}
```

### Python代码中使用
```python
from app.services.context import ExecutionContext

context = ExecutionContext(
    task_id="test",
    template_id="tpl_ai_mysql",
    user_inputs={"wgid": "ZQGY0174"},
    metadata=metadata_dict
)

# 设置变量
context.set_variable('overview', {
    'micro_grid_name': '测试微网格',
    'problem_count': 10
})

# 字符串插值
result = context.interpolate_string('名称: {{overview.micro_grid_name}}')
# 输出: "名称: 测试微网格"

# 使用过滤器
result = context.interpolate_string('问题数: {{overview.problem_count | default}}')
# 输出: "问题数: 10"
```

## 后续建议

### 1. 编写AI提示词时
✅ **推荐做法**:
```jinja2
- 问题楼宇: {{problem_buildings | length}} 个
- 网格名称: {{overview.micro_grid_name | upper}}
- 城市: {{overview.city}}
```

❌ **避免做法**:
```jinja2
{% if problem_buildings %}
  {% for building in problem_buildings %}
    - {{building.name}}
  {% endfor %}
{% endif %}
```

### 2. 处理复杂数据
如果需要循环、条件判断等复杂逻辑：
- **方案1**: 在SQL中处理，返回处理后的结果
- **方案2**: 在Python中预处理，存为变量
- **方案3**: 用自然语言描述，让AI自己判断

### 3. 调试技巧
```python
# 查看插值结果
try:
    result = context.interpolate_string(prompt_template)
    print(f"✅ 插值成功: {result[:200]}...")
except Exception as e:
    print(f"❌ 插值失败: {e}")
```
