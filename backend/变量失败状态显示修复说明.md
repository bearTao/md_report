# 变量失败状态显示修复说明

## 🐛 用户报告的问题

### 问题1：失败处理逻辑
用户指出调度器的失败处理逻辑本身没有问题：
- ✅ `required=True` 且无 `default` → 任务失败并中止
- ✅ `required=False` 或有 `default` → 任务继续执行

**但关键问题是**：当 `required=False` 或有 `default` 的变量失败时，**前端必须能看到该变量的失败状态**。

### 问题2：前端显示不一致
- 前端显示 `executive_summary` 执行成功
- 但后台日志显示该变量执行失败
- 需要修正这个不一致

### 问题3：Jinja2过滤器不支持
从错误日志看到：
```
Variable 'problem_buildings | length' not found for interpolation
Variable 'problem_building_clusters | length' not found for interpolation
Variable 'score' not found for interpolation
```

`interpolate_string` 不支持Jinja2过滤器（如 `| length`），导致AI提示词插值失败。

---

## ✅ 修复方案

### 修复1：保持原有失败处理逻辑

**回滚之前的过度修复** (`scheduler.py`)：

```python
# ✅ 保持原有逻辑
for result in batch_results:
    all_results[result.variable_name] = result
    
    # 关键：通过WebSocket通知前端变量状态（成功或失败）
    if progress_callback:
        await progress_callback(
            result.variable_name,
            result.status,  # 如果失败，这里是 FAILED
            result
        )
    
    # 只有必需变量且无默认值才中止任务
    if result.status == VariableStatus.FAILED:
        metadata = context.metadata[result.variable_name]
        if metadata.required and metadata.default is None:
            raise Exception(f"Required variable '{result.variable_name}' failed")
        # 可选变量失败，继续执行，但前端会看到FAILED状态
```

**关键点**：
- ✅ `progress_callback` 会调用 `broadcast_variable_failed` 通知前端
- ✅ 前端WebSocket会收到 `variable_failed` 事件
- ✅ 前端应该显示该变量为失败状态

### 修复2：支持Jinja2过滤器

增强 `interpolate_string` 支持常用的Jinja2过滤器 (`context.py`)：

```python
def interpolate_string(self, template_str: str) -> str:
    def replace_var(match):
        var_expr = match.group(1).strip()
        
        # 支持Jinja2过滤器（如 "variable | length"）
        if '|' in var_expr:
            parts = var_expr.split('|')
            var_name = parts[0].strip()
            filters = [f.strip() for f in parts[1:]]
            
            # 获取变量值
            value = _get_variable_value(var_name)
            
            # 应用过滤器
            for filter_name in filters:
                value = _apply_filter(filter_name, value, var_name)
            
            return str(value) if value is not None else ''
        else:
            # 无过滤器，直接获取变量
            value = _get_variable_value(var_expr)
            ...
```

**支持的过滤器**：
- ✅ `length` - 获取列表/字典/字符串长度
- ✅ `default` - 提供默认值
- ✅ `upper` - 转大写
- ✅ `lower` - 转小写
- ✅ `trim` - 去除首尾空格
- ✅ `first` - 获取第一个元素
- ✅ `last` - 获取最后一个元素
- ⚠️ 未知过滤器 - 忽略并警告

## 🎨 前端实现建议

### 1. 监听WebSocket事件

```javascript
ws.addEventListener('message', (event) => {
  const data = JSON.parse(event.data);
  
  switch(data.type) {
    case 'variable_completed':
      updateVariableStatus(data.variable_name, 'success', {
        duration: data.duration_ms,
        preview: data.result_preview
      });
      break;
    
    case 'variable_failed':
      updateVariableStatus(data.variable_name, 'failed', {
        duration: data.duration_ms,
        error: data.error
      });
      break;
    
    case 'task_completed':
      // 任务完成（某些变量可能失败了）
      showTaskCompleted(data.report_id);
      break;
    
    case 'task_failed':
      // 任务完全失败（必需变量失败）
      showTaskFailed(data.error);
      break;
  }
});
```

### 2. 显示变量状态

```javascript
function updateVariableStatus(variableName, status, details) {
  const element = document.querySelector(`[data-variable="${variableName}"]`);
  
  if (status === 'success') {
    element.classList.remove('loading', 'failed');
    element.classList.add('success');
    element.querySelector('.status-icon').textContent = '✅';
    element.querySelector('.duration').textContent = `${details.duration}ms`;
  } else if (status === 'failed') {
    element.classList.remove('loading', 'success');
    element.classList.add('failed');
    element.querySelector('.status-icon').textContent = '❌';
    element.querySelector('.error-message').textContent = details.error.message;
    element.querySelector('.duration').textContent = `${details.duration}ms`;
    
    // 显示"查看详情"按钮
    const detailsButton = element.querySelector('.view-details');
    detailsButton.style.display = 'block';
    detailsButton.onclick = () => showErrorDetails(details.error);
  }
}
```

## 📚 过滤器参考

| 过滤器 | 说明 | 示例 | 结果 |
|--------|------|------|------|
| `length` | 获取长度 | `{{items \| length}}` | `5` |
| `default` | 默认值 | `{{value \| default}}` | 如果value为None则返回`''` |
| `upper` | 转大写 | `{{name \| upper}}` | `JOHN DOE` |
| `lower` | 转小写 | `{{name \| lower}}` | `john doe` |
| `trim` | 去空格 | `{{text \| trim}}` | 去除首尾空格 |
| `first` | 第一项 | `{{items \| first}}` | `items[0]` |
| `last` | 最后项 | `{{items \| last}}` | `items[-1]` |

**注意**：
- 过滤器可以链式使用：`{{name | trim | upper}}`
- None值在length过滤器中返回0
- 未知过滤器会被忽略并记录警告
