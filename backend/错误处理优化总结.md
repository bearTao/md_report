## 🎯 核心特性

### 1. 智能错误识别

系统能够自动识别和分类常见错误：

| 错误模式 | 识别关键词 | 修复建议 |
|---------|----------|---------|
| 空值运算 | `'NoneType' object`, `has no attribute` | 使用 `or` 提供默认值 |
| 除以零 | `division by zero` | 添加分母检查 |
| 类型错误 | `unsupported operand type` | 确保类型一致，添加默认值 |
| 索引错误 | `not subscriptable` | 检查变量类型 |
| 字典访问 | `'dict object' has no attribute` | 使用 `.get()` 方法 |
| SQL表不存在 | `table doesn't exist` | 检查数据库连接和表名 |
| SQL语法错误 | `syntax error` | 检查SQL语法兼容性 |
| SQL超时 | `timeout` | 优化查询或增加超时 |
| AI密钥错误 | `api key`, `authentication` | 检查API配置 |
| AI频率限制 | `rate limit` | 稍后重试 |

### 2. 行号定位

对于模板渲染错误，自动提取行号：

```
模板渲染失败（第123行）
```

前端可以直接高亮显示出错位置。

### 3. 变量追踪

对于变量执行错误，明确指出出错的变量名：

```
SQL查询执行失败: 变量 'overview'
AI生成失败: 变量 'analysis_summary'
```

### 4. 字段名提示

对于字段访问错误，提取具体的字段名：

```
字段 'values' 不存在或为空。
使用 .get('values', default_value) 安全访问
```

### 5. 堆栈跟踪

保留最后10行堆栈信息，便于技术人员调试：

```python
error_info["details"]["traceback"] = [
  "File /app/services/renderer.py, line 45, in render",
  "File jinja2/environment.py, line 1301, in render",
  "File <template>, line 123, in top-level template code",
  "jinja2.exceptions.UndefinedError: ..."
]
```

## 📱 前端使用指南

### 1. 监听WebSocket错误事件

```javascript
ws.addEventListener('message', (event) => {
  const data = JSON.parse(event.data);
  
  if (data.type === 'task_failed') {
    const error = data.error;
    displayErrorDialog(error);
  }
});
```

### 2. 根据错误类型显示不同UI

```javascript
function displayErrorDialog(error) {
  // 基本信息
  const title = getErrorTitle(error.code);
  const icon = getErrorIcon(error.code);
  const color = getErrorColor(error.code);
  
  // 构建消息
  let message = error.message;
  
  if (error.details.line) {
    message += `\n位置: 第 ${error.details.line} 行`;
  }
  
  if (error.details.variable) {
    message += `\n变量: ${error.details.variable}`;
  }
  
  if (error.details.field) {
    message += `\n字段: ${error.details.field}`;
  }
  
  // 显示建议
  if (error.suggestion) {
    message += `\n\n💡 建议: ${error.suggestion}`;
  }
  
  // 根据错误类型提供操作按钮
  const actions = getErrorActions(error.code);
  
  // 显示对话框
  showDialog({
    title,
    icon,
    color,
    message,
    actions,
    details: error.details  // 可选：技术详情
  });
}
```

### 3. 错误类型对应的操作

```javascript
function getErrorActions(code) {
  switch(code) {
    case 'TEMPLATE_RENDER_ERROR':
      return [
        { text: '编辑模板', action: 'edit_template' },
        { text: '查看文档', action: 'view_docs' }
      ];
    
    case 'SQL_EXECUTION_ERROR':
      return [
        { text: '检查数据库', action: 'check_database' },
        { text: '查看SQL', action: 'view_sql' }
      ];
    
    case 'AI_GENERATION_ERROR':
      return [
        { text: '重试', action: 'retry' },
        { text: '检查配置', action: 'check_config' }
      ];
    
    case 'VALIDATION_ERROR':
      return [
        { text: '修改输入', action: 'edit_input' }
      ];
    
    default:
      return [
        { text: '重试', action: 'retry' },
        { text: '联系支持', action: 'contact_support' }
      ];
  }
}
```

### 4. 示例：完整的错误显示

```html
<div class="error-dialog" v-if="showError">
  <div class="error-header" :style="{ borderColor: errorColor }">
    <span class="error-icon">{{ errorIcon }}</span>
    <h3>{{ errorTitle }}</h3>
  </div>
  
  <div class="error-body">
    <p class="error-message">{{ errorMessage }}</p>
    
    <div class="error-location" v-if="errorDetails.line">
      📍 位置: 第 {{ errorDetails.line }} 行
    </div>
    
    <div class="error-variable" v-if="errorDetails.variable">
      🔖 变量: {{ errorDetails.variable }}
    </div>
    
    <div class="error-suggestion" v-if="errorSuggestion">
      <div class="suggestion-header">💡 修复建议</div>
      <div class="suggestion-content">{{ errorSuggestion }}</div>
    </div>
    
    <details class="error-technical" v-if="isDevelopmentMode">
      <summary>技术详情</summary>
      <pre>{{ errorDetails.traceback.join('\n') }}</pre>
    </details>
  </div>
  
  <div class="error-actions">
    <button 
      v-for="action in errorActions" 
      :key="action.action"
      @click="handleAction(action.action)"
      class="error-action-btn">
      {{ action.text }}
    </button>
  </div>
</div>
```
