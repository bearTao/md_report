## 1. 快速开始

### 1.1 第一次修改报告

假设您已经生成了一份报告,现在想要修改其中的某个参数。

**示例场景**: 修改报告中的微电网ID

1. **打开报告详情页**
2. **在修改对话框中输入**:
   ```
   把wgid改成ZQGY0175
   ```
3. **点击"发送"**
4. **查看结果**: 系统会自动:
   - 更新wgid参数
   - 重新执行依赖wgid的变量(如数据查询)
   - 重新生成受影响的内容
   - 显示更新后的报告

**就这么简单!** 无需了解任何技术细节。

### 1.2 继续修改

在同一个对话中,您可以继续提出修改请求:

```
让分析部分更详细一些
```

系统会记住之前的修改,并在此基础上继续优化。

## 2. 基本概念

### 2.1 什么是报告修改?

报告修改功能允许您通过自然语言对话的方式修改已生成的报告,而不是重新生成整份报告。这样可以:

- ✅ **节省时间**: 只修改需要改变的部分
- ✅ **保持上下文**: 系统记住您的修改历史
- ✅ **灵活调整**: 随时微调内容、参数或结构

### 2.2 支持的修改类型

#### 📝 参数更新
修改报告输入参数的值。

**示例**:
- "把时间范围改为最近一个月"
- "将wgid改为ZQGY0175"
- "时间改为2024年1月"

**效果**: 系统会自动重新执行依赖该参数的所有变量。

#### 🤖 AI内容优化
改进AI生成的文本内容。

**示例**:
- "让分析更详细"
- "使结论更简洁"
- "增加具体的数据支撑"
- "写得更专业一些"

**效果**: AI会根据您的要求重新生成该部分内容。

#### 📄 模板修改
添加、修改或删除报告章节。

**示例**:
- "添加竞争对手分析章节"
- "删除附录部分"
- "修改第一章的标题"

**效果**: 系统会自动:
1. 分析需要什么数据
2. 生成相应的查询或API调用
3. 生成新的章节内容
4. 插入到合适的位置

### 2.3 会话和版本

- **会话(Session)**: 一次对话的完整过程,可以包含多轮修改
- **版本(Version)**: 每次修改后报告的新版本号
- **上下文**: 系统记住的对话历史和修改记录

**示例流程**:
```
版本1 (原始报告)
  ↓ 修改wgid
版本2
  ↓ 优化分析
版本3
  ↓ 添加新章节
版本4
```

## 3. 常见使用场景

### 场景1: 修改参数值

**需求**: 报告生成后发现参数填错了,需要修改。

**操作**:
```
把wgid改成ZQGY0175
```

**说明**:
- 系统会自动识别参数名称
- 依赖该参数的内容会自动更新
- 大小写不敏感 (wgid, WGID, Wgid都可以)

**更多示例**:
```
将时间范围改为2024年1-3月
把地区改为华东
时间改为最近一周
```

### 场景2: 优化AI生成的内容

**需求**: AI生成的分析太简单,想要更详细的内容。

**操作**:
```
让分析部分更详细,增加具体的数据支撑
```

**系统会**:
1. 识别"分析部分"对应的变量
2. 理解"更详细"和"数据支撑"的要求
3. 重新生成内容
4. 保持其他部分不变

**更多示例**:
```
让结论更简洁一些
使市场分析更专业
增加具体案例
写得通俗易懂一些
加上趋势预测
```

### 场景3: 添加新章节

**需求**: 报告缺少竞争对手分析,想要添加。

**操作**:
```
添加竞争对手分析章节
```

**系统会**:
1. 分析需要什么数据(如竞争对手列表、市场份额等)
2. 自动生成数据查询
3. 生成章节内容的Jinja2模板
4. 在合适的位置插入新章节

**更详细的表述**:
```
添加竞争对手分析章节,包含市场份额对比和SWOT分析
```

**更多示例**:
```
添加风险评估部分
增加附录:数据来源说明
添加执行摘要
增加图表:市场趋势
```

### 场景4: 多个修改一起做

**需求**: 需要同时修改多个地方。

**操作**:
```
把wgid改成ZQGY0175,同时让分析更详细,还要添加风险评估章节
```

**说明**:
- 使用"同时"、"并且"、"还要"等连接词
- 系统会识别多个意图并按序执行
- 一次完成所有修改

### 场景5: 使用上下文引用

**需求**: 在对话中引用之前提到的内容。

**第一轮**:
```
把wgid改成ZQGY0175
```

**第二轮** (使用代词):
```
把它再改回去
```

系统会理解"它"指的是wgid。

**更多引用示例**:
```
让这个更详细  (指最近操作的内容)
把刚才的延长一周  (指最近修改的时间)
上一个改长一点  (指前一个修改的变量)
```

### 场景6: 相对值修改

**需求**: 在现有值的基础上进行调整。

**示例**:
```
时间延长一周
数量增加100个
把它缩短20%
往前推3天
```

系统会:
1. 获取当前值
2. 计算新值
3. 应用修改

## 4. 高级技巧

### 4.1 如何写出好的修改请求

#### ✅ 好的示例

**明确具体**:
```
把wgid改成ZQGY0175
```

**描述清楚**:
```
让市场分析部分更详细,增加行业趋势和竞争格局的分析
```

**逻辑清晰**:
```
把时间改为2024年第一季度,同时更新相关的数据查询
```

#### ❌ 需要改进的示例

**过于模糊**:
```
改一下  ← 改什么?
```
改进为:
```
把wgid改成ZQGY0175
```

**缺乏上下文**:
```
让它更好  ← 让什么更好?怎样算更好?
```
改进为:
```
让分析部分更详细,增加数据支撑
```

### 4.2 利用对话历史

系统会记住您的修改历史,您可以:

1. **引用之前的内容**:
   ```
   把刚才修改的那个再改长一点
   ```

2. **基于上次的修改**:
   ```
   第一轮: 把wgid改成ZQGY0175
   第二轮: 基于新的wgid,让分析更新一下数据
   ```

3. **逐步细化**:
   ```
   第一轮: 添加竞争对手分析
   第二轮: 让竞争对手分析包含SWOT矩阵
   第三轮: 在SWOT中增加具体案例
   ```

### 4.3 批量修改

如果需要修改多个地方,推荐一次性说明:

```
把wgid改成ZQGY0175,时间改为2024年1-3月,地区改为华东,
同时让分析更详细,添加竞争对手分析和风险评估章节
```

这样比分6次修改更高效。

### 4.4 保存修改后的模板

如果您对修改后的报告结构很满意,可以保存为新模板:

1. 完成所有修改
2. 点击"保存为模板"
3. 输入模板名称和描述
4. 下次可以直接使用这个模板

## 5. 最佳实践

### 5.1 修改前

1. **明确修改目标**: 想清楚要改什么
2. **检查当前内容**: 了解现有报告的状态
3. **规划修改步骤**: 如果修改较多,按逻辑顺序安排

### 5.2 修改中

1. **使用清晰的表述**: 避免歧义
2. **一次不要改太多**: 建议一次不超过5个修改
3. **检查每次结果**: 确认修改符合预期后再继续
4. **利用会话上下文**: 在同一会话中连续修改更高效

### 5.3 修改后

1. **验证结果**: 检查修改是否正确应用
2. **查看影响范围**: 了解哪些内容被更新了
3. **保存重要版本**: 如需要,保存为新模板
4. **记录修改历史**: 系统会自动记录,可随时查看

## 6. 问题排查

### Q1: 系统说"请求不够明确"

**原因**: 系统无法理解您的意图。

**解决**:
- 提供更具体的信息
- 明确说明要修改的变量或章节名称
- 避免使用过于简短的表述

**示例**:
```
❌ 改一下
✅ 把wgid改成ZQGY0175
```

### Q2: 修改后内容不符合预期

**原因**: AI理解可能与您的预期有偏差。

**解决**:
- 在同一会话中继续修改:
  ```
  不是这样,我是想让它包含具体的数据对比
  ```
- 提供更详细的要求:
  ```
  让分析更详细,要包含至少3个具体案例和数据趋势图
  ```

### Q3: 变量名识别错误

**原因**: 变量名拼写或系统识别问题。

**解决**:
- 使用准确的变量名
- 如不确定变量名,可以用描述:
  ```
  把微电网ID改成ZQGY0175
  ```
  而不是:
  ```
  把wgid改成ZQGY0175
  ```

### Q4: 修改耗时较长

**原因**: 某些操作(如添加章节)需要生成新内容,较耗时。

**说明**:
- 参数更新: 通常< 2秒
- AI内容优化: 3-10秒
- 添加章节: 5-15秒
- 多个修改: 累计时间

**如果超过30秒**: 
- 检查网络连接
- 刷新页面重试
- 联系技术支持

### Q5: 如何撤销修改

**当前限制**: 系统暂不支持直接撤销。

**替代方案**:
1. 继续修改,改回原来的值
2. 查看对话历史中的原始值
3. 重新生成报告

**未来计划**: 将支持版本回滚功能。

## 7. 使用技巧总结

### 🎯 核心技巧

1. **明确表述**: 清楚说明要修改什么、怎么修改
2. **善用上下文**: 利用对话历史和代词引用
3. **批量操作**: 多个修改一次完成
4. **逐步细化**: 先大致调整,再精细优化
5. **保存模板**: 满意的结构保存下来重复使用

### 📝 表述模板

```
【参数更新】
- 把<参数名>改成<新值>
- 将<参数名>修改为<新值>
- <参数名>改为<新值>

【AI优化】
- 让<内容部分>更<要求>
- 使<内容部分><要求>
- <内容部分>写得<要求>一些

【模板修改】
- 添加<章节名>章节
- 删除<章节名>部分
- 修改<章节名>的<特征>

【组合修改】
- <修改1>,同时<修改2>,还要<修改3>
- <修改1>,并且<修改2>
```
