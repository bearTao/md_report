# 数据库连接配置问题修复记录

## 问题描述

用户在前端使用 `wgid=ZQGY0174` 生成微网格预分析报告时，后台报错：

```
Template rendering failed: 'None' has no attribute 'problem_count'
jinja2.exceptions.UndefinedError: 'None' has no attribute 'problem_count'
```

## 问题分析

### 错误根源

模板渲染时 `overview` 变量为 `None`，说明SQL查询没有返回任何结果。

### 排查过程

1. **检查数据是否存在** ✅
   ```sql
   SELECT COUNT(*) FROM microgrid.micro_grid_overview_w WHERE wgid = 'ZQGY0174'
   -- 结果: 1 (数据存在)
   ```

2. **测试SQL查询** ✅
   ```python
   # 使用正确的连接参数直接查询
   result = await db_connector.execute_query(
       connection_name='microgrid_db',
       query='SELECT * FROM microgrid.micro_grid_overview_w WHERE wgid = :wgid',
       parameters={'wgid': 'ZQGY0174'}
   )
   # 结果: 查询成功，返回1条记录
   ```

3. **检查API中的数据库连接配置** ❌
   ```python
   # API从 DBConnection 表加载连接配置
   db_connections = db_session.query(DBConnection).filter(
       DBConnection.is_active == "true"
   ).all()
   ```

4. **发现问题：数据库连接配置错误**
   
   数据库中 `microgrid_db` 的配置：
   ```
   引擎: PostgreSQL  ❌ (实际应该是 MySQL)
   主机: 10.10.20.10
   端口: 14632       ❌ (实际应该是 24406)
   数据库: microgrid  ✅
   用户名: microgrid  ❌ (实际应该是 root)
   密码: [未知]       ❌ (实际应该是 123456)
   ```

## 问题根源

**API生成报告时使用的数据库连接配置与实际的MySQL数据库配置不匹配！**

- API从 `DBConnection` 表读取配置
- 配置中的引擎类型、端口、用户名都是错误的
- 导致无法连接到正确的MySQL数据库
- SQL查询返回空结果，`overview` 变量为 `None`
- 模板渲染时访问 `overview.problem_count` 失败

## 解决方案

### 1. 更新数据库连接配置

```python
from app.database import SessionLocal
from app.models.db_models import DBConnection, DBEngineType

db = SessionLocal()
conn = db.query(DBConnection).filter(DBConnection.name == 'microgrid_db').first()

# 更新为正确的配置
conn.engine = DBEngineType.MYSQL           # PostgreSQL → MySQL
conn.host = '10.10.20.10'                  # 保持不变
conn.port = 24406                          # 14632 → 24406
conn.database = 'microgrid'                # 保持不变
conn.username = 'root'                     # microgrid → root
conn.password_ciphertext = '123456'        # 更新密码
conn.is_active = 'true'                    # 保持激活

db.commit()
```

## 为什么会出现这个问题？

### 配置分离的架构

系统有两种数据库连接方式：

1. **直接连接（测试/开发）**
   ```python
   # 直接在代码中配置
   DATABASE_URL = "mysql+pymysql://root:123456@10.10.20.10:24406/microgrid"
   engine = create_engine(DATABASE_URL)
   db_connector.register_connection("microgrid_db", engine)
   ```

2. **动态配置（生产/API）**
   ```python
   # 从 DBConnection 表读取配置
   db_connections = db_session.query(DBConnection).filter(
       DBConnection.is_active == "true"
   ).all()
   
   for db_conn in db_connections:
       connection_url = f"{dialect}://{db_conn.username}:{password}@{db_conn.host}:{db_conn.port}/{db_conn.database}"
       db_connector.register_connection(db_conn.name, connection_url)
   ```

### 问题发生原因

- 测试脚本使用**方式1**（直接连接），可以正常工作
- API使用**方式2**（动态配置），但数据库中的配置是错误的
- 导致API无法连接到正确的数据库

## 预防措施

### 1. 配置验证机制

在启动时验证数据库连接配置：

```python
# app/main.py
@app.on_event("startup")
async def validate_db_connections():
    """验证所有数据库连接配置"""
    from app.models.db_models import DBConnection
    from app.connectors.database import db_connector
    
    db = SessionLocal()
    connections = db.query(DBConnection).filter(DBConnection.is_active == "true").all()
    
    for conn in connections:
        try:
            # 测试连接
            if db_connector.test_connection(conn.name):
                logger.info(f"✅ Database connection '{conn.name}' is valid")
            else:
                logger.warning(f"⚠️ Database connection '{conn.name}' failed to connect")
        except Exception as e:
            logger.error(f"❌ Database connection '{conn.name}' error: {e}")
    
    db.close()
```
