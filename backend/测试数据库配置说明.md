## 数据清理

### 自动清理

每个测试执行后会自动清空所有表数据（TRUNCATE），确保测试隔离：

```python
# Disable foreign key checks
db.execute(text("SET FOREIGN_KEY_CHECKS = 0"))

# Truncate all tables
for table in reversed(Base.metadata.sorted_tables):
    db.execute(text(f"TRUNCATE TABLE {table.name}"))

# Re-enable foreign key checks
db.execute(text("SET FOREIGN_KEY_CHECKS = 1"))
```

### 手动清理（如需要）

```sql
-- 删除测试数据库
DROP DATABASE IF EXISTS test_report_generator;

-- 重新创建
CREATE DATABASE test_report_generator 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

## 与SQLite的区别

### 之前（SQLite）

- 使用内存数据库 `sqlite:///:memory:`
- 每个测试独立数据库
- 自动创建/删除表

### 现在（MySQL）

- 使用远程MySQL数据库
- 共享数据库实例
- 每个测试后清空表数据（TRUNCATE）
- 更接近生产环境

## 注意事项

1. **网络连接**: 确保能访问 10.10.20.10:24406
2. **数据库权限**: 需要有创建表/清空表的权限
3. **字符集**: 使用 utf8mb4 支持完整Unicode（包括emoji）
4. **外键约束**: 清理时临时禁用外键检查
5. **连接池**: 测试使用NullPool，每次创建新连接

## 故障排查

### 连接失败

```python
pymysql.err.OperationalError: (2003, "Can't connect to MySQL server...")
```

**解决方案**:
- 检查网络连接
- 验证IP和端口
- 确认MySQL服务运行中

### 权限错误

```python
pymysql.err.OperationalError: (1045, "Access denied for user...")
```

**解决方案**:
- 检查用户名密码
- 确认用户有数据库访问权限

### 表已存在

```python
sqlalchemy.exc.OperationalError: (1050, "Table 'xxx' already exists")
```

**解决方案**:
- 正常情况，create_all会忽略已存在的表
- 如持续出错，手动清理数据库

## 维护建议

1. **定期清理**: 如果测试失败导致数据残留，手动TRUNCATE
2. **备份**: 测试数据库不需要备份
3. **监控**: 观察测试耗时，如明显增加检查网络/数据库性能
4. **隔离**: 建议使用独立的测试数据库，不与开发环境混用

## 生产环境数据库

生产环境可以使用环境变量配置PostgreSQL或MySQL：

```bash
export DATABASE_URL="mysql+pymysql://user:password@host:port/dbname"
```

或

```bash
export DATABASE_URL="postgresql://user:password@host:port/dbname"
```

应用会自动使用配置的数据库。
