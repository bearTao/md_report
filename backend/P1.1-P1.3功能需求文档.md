# P1.1 核心图片支持

## 1. 图片获取功能

### 1.1 功能概述

**需求背景**:  
生产环境中，报告需要包含从外部API获取的图片，如产品照片、检测图片、现场照片等。

**功能目标**:
- 从HTTP/HTTPS API获取图片
- 支持Base64内嵌到Markdown
- 支持外链URL方式
- 支持批量图片获取
- 与现有变量系统无缝集成

### 1.2 技术架构

```
变量元数据 (YAML)
    ↓
ImageExecutor (新增)
    ↓
ImageApiConnector (新增)
    ↓
HTTP API获取图片
    ↓
返回格式化数据 (Base64/URL)
    ↓
Jinja2模板渲染
    ↓
Markdown输出
```

### 1.3 详细需求

#### 1.3.1 ImageApiConnector (图片连接器)

**文件路径**: `backend/app/connectors/image_api.py`

**核心功能**:

1. **fetch_image** - 获取单张图片
   ```python
   async def fetch_image(
       url: str,
       headers: Optional[Dict[str, str]] = None,
       params: Optional[Dict[str, Any]] = None,
       output_format: Literal["base64", "url", "bytes"] = "base64",
       include_mime: bool = True
   ) -> Dict[str, Any]
   ```
   
   **参数说明**:
   - `url`: 图片URL或API端点
   - `headers`: 请求头（如认证信息）
   - `params`: URL参数
   - `output_format`: 输出格式
     - `base64`: 返回Base64编码（适合内嵌MD）
     - `url`: 返回原始URL（适合外链）
     - `bytes`: 返回字节数据
   - `include_mime`: 是否包含MIME类型
   
   **返回格式**:
   ```json
   {
     "data": "base64字符串或url或bytes",
     "mime_type": "image/png",
     "size": 12345,
     "url": "原始URL",
     "markdown": "![图片](data:image/png;base64,...)"
   }
   ```

2. **fetch_multiple_images** - 批量获取
   ```python
   async def fetch_multiple_images(
       urls: list[str],
       **kwargs
   ) -> list[Dict[str, Any]]
   ```

**错误处理**:
- HTTP状态码错误 → 抛出详细异常
- 超时 → 可配置超时时间（默认30秒）
- 网络错误 → 提供重试机制（可选）
- 非图片内容 → 验证Content-Type

**技术要点**:
- 使用 `httpx.AsyncClient` 异步请求
- 自动检测MIME类型
- 支持重定向跟随
- Base64编码优化（大图片警告）

#### 1.3.2 ImageExecutor (图片执行器)

**文件路径**: `backend/app/executors/image.py`

**核心功能**:

1. **执行流程**:
   ```
   读取image_config
     ↓
   插值API端点 ({{variable}})
     ↓
   插值headers和params
     ↓
   调用ImageApiConnector
     ↓
   返回图片数据
   ```

2. **配置结构** (在元数据YAML中):
   ```yaml
   product_image:
     type: "image"
     source: "image"
     description: "产品图片"
     required: true
     dependencies: ["product_id"]
     image_config:
       endpoint: "https://api.example.com/products/{{product_id}}/image"
       method: "GET"  # 可选，默认GET
       headers:
         Authorization: "Bearer {{api_token}}"
       parameters:
         size: "large"
         quality: "high"
       output_format: "base64"  # base64 | url
       multiple: false  # 是否批量获取
       timeout: 30
   ```

3. **支持场景**:
   - 单张图片获取
   - 批量图片获取（返回数组）
   - 带认证的API请求
   - 动态参数插值

**错误处理**:
- 端点插值失败 → ApiExecutionError
- API调用失败 → 详细错误信息
- 超时处理 → 可配置超时
- 默认值支持 → 图片获取失败时使用占位符

## 2. 视觉大模型集成

### 2.1 功能概述

**需求背景**:  
需要AI分析图片内容并生成文字描述，应用于质检、安全检查、现场分析等场景。

**功能目标**:
- 支持GPT-4V、GPT-4o等视觉模型
- 单张图片分析
- 多张图片综合分析
- 与其他变量协同工作
- 结构化输出支持

### 2.2 技术架构

```
图片变量 (image类型)
    ↓
VisionAiExecutor (新增)
    ↓
提取图片URL
    ↓
构建Vision Prompt
    ↓
调用GPT-4V/GPT-4o
    ↓
返回文字描述
    ↓
Jinja2模板渲染
```

### 2.3 详细需求

#### 2.3.1 VisionAiExecutor (视觉AI执行器)

**文件路径**: `backend/app/executors/vision_ai.py`

**核心功能**:

1. **执行流程**:
   ```python
   async def _execute_impl(self) -> Any:
       # 1. 获取图片数据（从依赖变量）
       image_data = self.context.get_variable(image_source)

       # 5. 调用视觉模型
       response = await llm.ainvoke([message])
       
       # 6. 返回结果
       return response.content
   ```

2. **支持的图片格式**:
   - HTTP/HTTPS URL
   - Data URI (Base64)
   - 从ImageExecutor返回的数据结构

3. **支持的模型**:
   - `gpt-4o` (推荐，最新)
   - `gpt-4-vision-preview`
   - 其他兼容OpenAI Vision API的模型

#### 2.3.5 图片URL提取逻辑

```python
def _extract_image_urls(self, image_data: Any) -> List[str]:
    """
    从图片变量中提取URL
    支持多种数据格式：
    1. 字符串URL: "https://..."
    2. ImageExecutor返回: {"data": "...", "url": "...", "markdown": "..."}
    3. 列表: [image1, image2, ...]
    """
    urls = []
    
    if isinstance(image_data, str):
        if image_data.startswith(("http://", "https://", "data:")):
            urls.append(image_data)
    
    elif isinstance(image_data, dict):
        if "data" in image_data and image_data.get("mime_type"):
            # Base64格式，构建data URI
            mime_type = image_data["mime_type"]
            data = image_data["data"]
            urls.append(f"data:{mime_type};base64,{data}")
        elif "url" in image_data:
            urls.append(image_data["url"])
    
    elif isinstance(image_data, list):
        for item in image_data:
            urls.extend(self._extract_image_urls(item))
    
    return urls
```

## 4. 密码加密存储

### 4.1 功能概述

**需求背景**:

**功能目标**:

- 加密密钥通过环境变量管理

- 向后兼容

### 4.2 详细需求

#### 4.2.1 加密服务

**文件路径**: `backend/app/core/encryption.py`

**核心功能**:

```python
from cryptography.fernet import Fernet
import os
import base64

class EncryptionService:
    """密码加密服务"""
    
    def __init__(self):
        # 从环境变量获取密钥
        key = os.getenv("ENCRYPTION_KEY")
        if not key:
            # 开发环境：生成临时密钥（警告）
            # 生产环境：必须配置（抛出异常）
            if os.getenv("ENVIRONMENT") == "production":
                raise ValueError("ENCRYPTION_KEY must be set in production")
            key = Fernet.generate_key()
        
        self.fernet = Fernet(key)
    
    def encrypt(self, plaintext: str) -> str:
        """加密明文"""
        if not plaintext:
            return ""
        encrypted = self.fernet.encrypt(plaintext.encode())
        return base64.b64encode(encrypted).decode()
    
    def decrypt(self, ciphertext: str) -> str:
        """解密密文"""
        if not ciphertext:
            return ""
        encrypted = base64.b64decode(ciphertext)
        decrypted = self.fernet.decrypt(encrypted)
        return decrypted.decode()
    
    @staticmethod
    def generate_key() -> str:
        """生成新密钥"""
        return Fernet.generate_key().decode()

# 全局实例
encryption_service = EncryptionService()
```

```python
"""
数据迁移：加密现有明文密码
使用方法：python migrations/encrypt_passwords.py
"""

from app.database import SessionLocal
from app.models.db_models import DBConnection
from app.core.encryption import encryption_service

def migrate_passwords():
    """迁移现有明文密码为加密密码"""
    db = SessionLocal()
    
    try:
        connections = db.query(DBConnection).all()
        count = 0
        
        for conn in connections:
            # 检查是否已加密（简单判断：加密后长度明显不同）
            if len(conn.password_ciphertext) < 50:
                # 假设是明文，进行加密
                encrypted = encryption_service.encrypt(conn.password_ciphertext)
                conn.password_ciphertext = encrypted
                count += 1
        
        db.commit()
        print(f"✅ 成功加密 {count} 个数据库连接密码")
        
    except Exception as e:
        db.rollback()
        print(f"❌ 迁移失败: {str(e)}")
        raise
    finally:
        db.close()

if __name__ == "__main__":
    migrate_passwords()
```

```python
@router.post("/{task_id}/cancel")
async def cancel_task(
    task_id: str,
    cancel_request: TaskCancelRequest = Body(default=None),
    db: Session = Depends(get_db)
):
    """
    取消任务
    
    功能：
    1. 更新任务状态为cancelled
    2. 设置取消标志（供执行器检查）
    3. 停止正在执行的变量
    4. 推送WebSocket事件
    """
    # 1. 查询任务
    task = db.query(GenerationTask).filter_by(id=task_id).first()
    if not task:
        raise HTTPException(404, "Task not found")
    
    # 2. 检查任务状态
    if task.status not in ["pending", "running"]:
        raise HTTPException(400, f"Cannot cancel task in status: {task.status}")
    
    # 3. 设置取消标志（在任务上下文中）
    task_context.set_cancellation_flag(task_id)
    
    # 4. 更新任务状态
    task.status = "cancelled"
    task.finished_at = datetime.now(timezone.utc)
    
    # 5. 更新运行中的变量状态
    running_vars = db.query(GenerationTaskVariable).filter_by(
        task_id=task_id,
        status="running"
    ).all()
    
    for var in running_vars:
        var.status = "cancelled"
        var.finished_at = datetime.now(timezone.utc)
    
    db.commit()
    
    # 6. 推送WebSocket事件
    await websocket_manager.broadcast_to_task(
        task_id,
        {
            "type": "task_cancelled",
            "task_id": task_id,
            "reason": cancel_request.reason if cancel_request else None,
            "cancelled_at": datetime.now(timezone.utc).isoformat()
        }
    )
    
    return {
        "success": True,
        "data": {
            "task_id": task_id,
            "status": "cancelled",
            "cancelled_at": task.finished_at.isoformat()
        }
    }
```

```python
    """
    重试单个变量
    
    功能：
    1. 重置变量状态为pending

    3. 触发重新执行
    4. 保持其他变量不变

    if var.status not in ["failed", "cancelled"]:

    var.finished_at = None
    var.duration_ms = None
    var.error_code = None

        variable_name=variable_name
    )

        task_id,
        {
            "type": "variable_retry_started",
            "task_id": task_id,
            "variable_name": variable_name
        }
    )
    
    return {
        "success": True,
        "data": {
            "task_id": task_id,
            "variable_name": variable_name,

    }
```

#### 5.3.3 取消检查机制

**文件路径**: `backend/app/core/task_context.py`

```python
class TaskContext:
    """任务上下文（扩展）"""
    
    _cancellation_flags = {}  # {task_id: bool}
    
    @classmethod
    def set_cancellation_flag(cls, task_id: str):
        """设置取消标志"""
        cls._cancellation_flags[task_id] = True
    
    @classmethod
    def is_cancelled(cls, task_id: str) -> bool:
        """检查是否已取消"""
        return cls._cancellation_flags.get(task_id, False)
    
    @classmethod
    def clear_cancellation_flag(cls, task_id: str):
        """清除取消标志"""
        if task_id in cls._cancellation_flags:
            del cls._cancellation_flags[task_id]
```

**在执行器中检查**:

```python
class BaseVariableExecutor:
    """基础执行器（修改）"""
    
    async def execute(self) -> Any:
        """执行变量（带取消检查）"""
        # 执行前检查
        if TaskContext.is_cancelled(self.task_id):
            raise TaskCancelledException(f"Task {self.task_id} was cancelled")
        
        try:
            result = await self._execute_impl()
            
            # 执行后再次检查
            if TaskContext.is_cancelled(self.task_id):
                raise TaskCancelledException(f"Task {self.task_id} was cancelled")
            
            return result
        except TaskCancelledException:
            raise
        except Exception as e:
            # 错误处理
            raise
```

```python
async def retry_variable_execution(task_id: str, variable_name: str):
    """重试变量执行"""
    db = SessionLocal()
    
    try:
        # 1. 获取模板和元数据
        task = db.query(GenerationTask).filter_by(id=task_id).first()
        template = db.query(Template).filter_by(id=task.template_id).first()
        
        # 2. 解析元数据
        metadata = parse_metadata(template.metadata_json)
        variable_meta = metadata.get(variable_name)
        
        # 3. 构建上下文（复用已执行成功的变量）
        context = await build_context_from_task(task_id, db)
        
        # 4. 创建执行器
        executor = create_executor(
            variable_name,
            variable_meta,
            context,
            # ... 其他参数 ...
        )
        
        # 5. 执行变量
        var_record = db.query(GenerationTaskVariable).filter_by(
            task_id=task_id,
            variable_name=variable_name
        ).first()
        
        var_record.status = "running"
        var_record.started_at = datetime.now(timezone.utc)
        db.commit()
        
        # WebSocket: variable_started
        await websocket_manager.broadcast_to_task(task_id, {
            "type": "variable_started",
            "variable_name": variable_name
        })
        
        result = await executor.execute()
        
        # 6. 更新结果
        var_record.status = "success"
        var_record.finished_at = datetime.now(timezone.utc)
        var_record.duration_ms = (var_record.finished_at - var_record.started_at).total_seconds() * 1000
        context.set_variable(variable_name, result)
        db.commit()
        
        # WebSocket: variable_completed
        await websocket_manager.broadcast_to_task(task_id, {
            "type": "variable_completed",
            "variable_name": variable_name
        })
        
        # 7. 检查是否所有变量都成功，如果是则继续渲染报告
        if all_variables_successful(task_id, db):
            await continue_report_generation(task_id, context, db)
        
    except Exception as e:
        # 更新失败状态
        var_record.status = "failed"
        var_record.error_message = str(e)
        db.commit()
        
        # WebSocket: variable_failed
        await websocket_manager.broadcast_to_task(task_id, {
            "type": "variable_failed",
            "variable_name": variable_name,
            "error": str(e)
        })
    finally:
        db.close()
```

```python
from typing import Optional, Dict, Any
from app.database import SessionLocal
from app.models.db_models import ExecutionLog
import logging

logger = logging.getLogger(__name__)


class ExecutionLogger:
    """执行日志服务"""
    
    @staticmethod
    def log(
        task_id: str,
        message: str,
        level: str = "INFO",
        variable_name: Optional[str] = None,
        context: Optional[Dict[str, Any]] = None
    ):
        """
        记录执行日志
        
        Args:
            task_id: 任务ID
            message: 日志消息
            level: 日志级别（DEBUG|INFO|WARNING|ERROR）
            variable_name: 变量名（可选，任务级日志为None）
            context: 额外上下文信息
        """
        db = SessionLocal()
        try:
            log_entry = ExecutionLog(
                task_id=task_id,
                variable_name=variable_name,
                level=level.upper(),
                message=message,
                context_json=context
            )
            db.add(log_entry)
            db.commit()
            
            # 同时输出到标准日志
            log_method = getattr(logger, level.lower(), logger.info)
            log_method(f"[{task_id}] {variable_name or 'TASK'}: {message}")
            
        except Exception as e:
            logger.error(f"Failed to write execution log: {str(e)}")
            db.rollback()
        finally:
            db.close()
    
    @staticmethod
    def debug(task_id: str, message: str, **kwargs):
        ExecutionLogger.log(task_id, message, "DEBUG", **kwargs)
    
    @staticmethod
    def info(task_id: str, message: str, **kwargs):
        ExecutionLogger.log(task_id, message, "INFO", **kwargs)
    
    @staticmethod
    def warning(task_id: str, message: str, **kwargs):
        ExecutionLogger.log(task_id, message, "WARNING", **kwargs)
    
    @staticmethod
    def error(task_id: str, message: str, **kwargs):
        ExecutionLogger.log(task_id, message, "ERROR", **kwargs)
```

```python
@router.get("/{task_id}/logs")
async def get_task_logs(
    task_id: str,
    level: Optional[str] = None,
    variable_name: Optional[str] = None,
    limit: int = Query(100, le=1000),
    offset: int = Query(0, ge=0),
    db: Session = Depends(get_db)
):
    """
    获取任务执行日志
    """
    # 构建查询
    query = db.query(ExecutionLog).filter_by(task_id=task_id)
    
    if level:
        query = query.filter_by(level=level.upper())
    
    if variable_name:
        query = query.filter_by(variable_name=variable_name)
    
    # 总数
    total = query.count()
    
    # 分页
    logs = query.order_by(ExecutionLog.created_at.asc())\
               .offset(offset)\
```

## 7. 模板校验功能

### 7.1 功能概述

**需求背景**:  
用户编写模板时可能出现语法错误、变量引用错误、依赖循环等问题，需要提前发现。

**功能目标**:
- Jinja2语法校验
- 变量元数据完整性检查
- 依赖关系检查（DAG无环）
- Schema格式验证
- 提供详细的错误/警告信息

### 7.2 校验服务

#### 7.2.1 TemplateValidator

**文件路径**: `backend/app/services/template_validator.py`

```python
from typing import List, Dict, Any
from jinja2 import Environment, meta, TemplateSyntaxError
from jinja2.sandbox import SandboxedEnvironment
import yaml
import json


class ValidationIssue:
    """校验问题"""
    def __init__(
        self,
        level: str,  # error | warning
        category: str,  # syntax | metadata | dependency | schema
        message: str,
        location: str = None
    ):
        self.level = level
        self.category = category
        self.message = message
        self.location = location
    
    def to_dict(self):
        return {
            "level": self.level,
            "category": self.category,
            "message": self.message,
            "location": self.location
        }


class TemplateValidator:
    """模板校验器"""
    
    def __init__(self):
        self.env = SandboxedEnvironment()
        self.issues: List[ValidationIssue] = []
    
    def validate_template(
        self,
        template_content: str,
        metadata: Dict[str, Any]
    ) -> Dict[str, Any]:
        """
        校验模板和元数据
        
        Returns:
            {
                "valid": bool,
                "issues": [
                    {
                        "level": "error|warning",
                        "category": "syntax|metadata|dependency|schema",
                        "message": "详细信息",
                        "location": "位置信息"
                    }
                ]
            }
        """
        self.issues = []
        
        # 1. Jinja2语法校验
        self._validate_jinja2_syntax(template_content)
        
        # 2. 变量引用检查
        self._validate_variable_references(template_content, metadata)
        
        # 3. 元数据完整性
        self._validate_metadata_structure(metadata)
        
        # 4. 依赖关系（DAG）
        self._validate_dependencies(metadata)
        
        # 5. Schema格式
        self._validate_schemas(metadata)
        
        # 判断是否有error级别的问题
        has_errors = any(issue.level == "error" for issue in self.issues)
        
        return {
            "valid": not has_errors,
            "issues": [issue.to_dict() for issue in self.issues]
        }
    
    def _validate_jinja2_syntax(self, template_content: str):
        """校验Jinja2语法"""
        try:
            self.env.parse(template_content)
        except TemplateSyntaxError as e:
            self.issues.append(ValidationIssue(
                level="error",
                category="syntax",
                message=f"Jinja2语法错误: {str(e)}",
                location=f"行 {e.lineno}"
            ))
    
    def _validate_variable_references(
        self,
        template_content: str,
        metadata: Dict[str, Any]
    ):
        """检查模板中引用的变量是否都在元数据中定义"""
        try:
            # 解析模板中使用的变量
            ast = self.env.parse(template_content)
            template_vars = meta.find_undeclared_variables(ast)
            
            # 元数据中定义的变量
            defined_vars = set(metadata.keys())
            
            # 找出未定义的变量
            undefined_vars = template_vars - defined_vars
            
            for var in undefined_vars:
                # 排除一些内置变量
                if var not in ['loop', 'range', 'lipsum']:
                    self.issues.append(ValidationIssue(
                        level="error",
                        category="metadata",
                        message=f"模板中使用了未定义的变量: {var}",
                        location="template"
                    ))
            
            # 警告：定义但未使用的变量
            unused_vars = defined_vars - template_vars
            for var in unused_vars:
                self.issues.append(ValidationIssue(
                    level="warning",
                    category="metadata",
                    message=f"变量已定义但未在模板中使用: {var}",
                    location="metadata"
                ))
                
        except Exception as e:
            self.issues.append(ValidationIssue(
                level="error",
                category="syntax",
                message=f"无法解析模板: {str(e)}"
            ))
    
    def _validate_metadata_structure(self, metadata: Dict[str, Any]):
        """校验元数据结构完整性"""
        for var_name, var_meta in metadata.items():
            # 必需字段检查
            if "type" not in var_meta:
                self.issues.append(ValidationIssue(
                    level="error",
                    category="metadata",
                    message=f"变量 {var_name} 缺少 'type' 字段"
                ))
            
            if "source" not in var_meta:
                self.issues.append(ValidationIssue(
                    level="error",
                    category="metadata",
                    message=f"变量 {var_name} 缺少 'source' 字段"
                ))
            else:
                # 检查source是否有效
                valid_sources = [
                    "user_input", "sql", "api", "system",
                    "ai_generation", "image", "vision_ai"
                ]
                if var_meta["source"] not in valid_sources:
                    self.issues.append(ValidationIssue(
                        level="error",
                        category="metadata",
                        message=f"变量 {var_name} 的 source '{var_meta['source']}' 无效"
                    ))
                
                # 检查source对应的config
                self._validate_source_config(var_name, var_meta)
    
    def _validate_source_config(self, var_name: str, var_meta: Dict):
        """检查source对应的配置是否存在"""
        source = var_meta.get("source")
        
        config_map = {
            "sql": "sql_config",
            "api": "api_config",
            "ai_generation": "ai_config",
            "image": "image_config",
            "vision_ai": "vision_ai_config"
        }
        
        if source in config_map:
            config_key = config_map[source]
            if config_key not in var_meta:
                self.issues.append(ValidationIssue(
                    level="error",
                    category="metadata",
                    message=f"变量 {var_name} (source={source}) 缺少 '{config_key}'"
                ))
    
    def _validate_dependencies(self, metadata: Dict[str, Any]):
        """检查依赖关系（DAG无环）"""
        # 构建依赖图
        graph = {}
        for var_name, var_meta in metadata.items():
            deps = var_meta.get("dependencies", [])
            graph[var_name] = deps
        
        # 检查依赖是否存在
        for var_name, deps in graph.items():
            for dep in deps:
                if dep not in metadata:
                    self.issues.append(ValidationIssue(
                        level="error",
                        category="dependency",
                        message=f"变量 {var_name} 依赖的变量 {dep} 不存在"
                    ))
        
        # 检查循环依赖
        cycles = self._detect_cycles(graph)
        for cycle in cycles:
            self.issues.append(ValidationIssue(
                level="error",
                category="dependency",
                message=f"检测到循环依赖: {' -> '.join(cycle)}"
            ))
    
    def _detect_cycles(self, graph: Dict[str, List[str]]) -> List[List[str]]:
        """检测图中的环"""
        cycles = []
        visited = set()
        path = []
        
        def dfs(node):
            if node in path:
                # 找到环
                cycle_start = path.index(node)
                cycles.append(path[cycle_start:] + [node])
                return
            
            if node in visited:
                return
            
            visited.add(node)
            path.append(node)
            
            for neighbor in graph.get(node, []):
                dfs(neighbor)
            
            path.pop()
        
        for node in graph:
            if node not in visited:
                dfs(node)
        
        return cycles
    
    def _validate_schemas(self, metadata: Dict[str, Any]):
        """校验Schema格式（AI变量）"""
        for var_name, var_meta in metadata.items():
            if var_meta.get("source") in ["ai_generation", "vision_ai"]:
                schema = var_meta.get("schema")
                if schema:
                    # 基础Schema验证
                    if not isinstance(schema, dict):
                        self.issues.append(ValidationIssue(
                            level="error",
                            category="schema",
                            message=f"变量 {var_name} 的 schema 必须是对象"
                        ))
                    elif "type" not in schema:
                        self.issues.append(ValidationIssue(
                            level="warning",
                            category="schema",
                            message=f"变量 {var_name} 的 schema 缺少 'type' 字段"
                        ))


# 全局实例
template_validator = TemplateValidator()
```

```python
@router.post("/{template_id}/validate")
async def validate_template(
    template_id: str,
    db: Session = Depends(get_db)
):
    """
    校验模板
    """
    # 1. 获取模板
    template = db.query(Template).filter_by(id=template_id).first()
    if not template:
        raise HTTPException(404, "Template not found")
    
    # 2. 解析元数据
    try:
        if isinstance(template.metadata_json, str):
            metadata = yaml.safe_load(template.metadata_json)
        else:
            metadata = template.metadata_json
    except Exception as e:
        return {
            "success": True,
            "data": {
                "valid": False,
                "issues": [{
                    "level": "error",
                    "category": "metadata",
                    "message": f"元数据格式错误: {str(e)}"
                }]
            }
        }
    
    # 3. 执行校验
    result = template_validator.validate_template(
        template.template_content,
        metadata
    )
    
    return {
        "success": True,
        "data": result
    }
```
