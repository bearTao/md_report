## 🚀 常用命令

### 运行所有测试
```bash
conda activate test_md
pytest tests/
```

### 运行特定模块
```bash

pytest tests/api/

# 执行器测试
pytest tests/executors/

# 集成测试
pytest tests/integration/
```

### 运行单个文件
```bash

```

### 查看某个模块有哪些测试
```bash
pytest tests/api/ --collect-only
pytest tests/executors/ --collect-only
```

## 🔍 查找测试

不确定某个测试在哪？

1. **根据功能判断**：
   - API 相关 → `tests/api/`
   - 执行器相关 → `tests/executors/`
   - 模板相关 → `tests/template/`
   - SQL 相关 → `tests/sql/`
   - 集成测试 → `tests/integration/`
   - 核心功能 → `tests/core/`
   - Agent → `tests/agent/`

2. **搜索文件名**：
   ```bash
   # Windows PowerShell
   Get-ChildItem -Path tests -Recurse -Filter "*constant*"
   
   # 或使用 pytest
   pytest tests/ --collect-only | grep "constant"
   ```

## ❓ 常见问题

**Q: 为什么要重组？**  
A: 为了让测试结构更清晰，像 `tests/agent/` 那样容易维护和查找。

**Q: 旧的测试路径还能用吗？**  
A: 不能，文件已移动。需要使用新的路径，如 `tests/api/test_api_config.py`。

**Q: conftest.py 的 fixture 还能用吗？**  
A: 可以！所有模块都能使用根目录 conftest.py 中的 fixture。

**Q: 如何快速验证测试是否正常？**  
A: 运行 `pytest tests/ --collect-only`，应该显示 "290 tests collected"。
