# 错误处理优化说明

## 📋 概述

优化了报告生成过程中的错误处理机制，现在前端可以获取详细的错误信息，包括：
- 错误类型分类
- 具体错误位置（模板行号等）
- 用户友好的错误描述
- 修复建议

---

## 🔧 后端改进

### 1. 新增错误格式化函数

在 `/backend/app/api/reports.py` 中新增了 `_format_error_details()` 函数，将各种异常转换为结构化的错误信息。

### 2. 错误信息结构

所有错误现在返回统一的JSON结构：

```json
{
  "code": "错误类型代码",
  "message": "用户友好的错误消息",
  "details": {
    "line": 123,                    // 可选：模板行号
    "variable": "variable_name",     // 可选：出错的变量名
    "field": "field_name",           // 可选：出错的字段名
    "type": "error_type",            // 详细错误类型
    "description": "详细描述",
    "error": "原始错误信息",
    "traceback": ["...", "..."]      // 最后10行堆栈跟踪
  },
  "suggestion": "修复建议"           // 可选：如何修复这个错误
}
```

## 📊 错误类型分类

### 1. 模板渲染错误 (TEMPLATE_RENDER_ERROR)

#### 1.1 空值错误 (null_value_error)

**触发场景**：
```jinja2
{{ overview.problem_count + overview.gr_problem_count }}
{# 当 problem_count 或 gr_problem_count 为 None 时 #}
```

**返回信息**：
```json
{
  "code": "TEMPLATE_RENDER_ERROR",
  "message": "模板渲染失败（第123行）",
  "details": {
    "line": 123,
    "type": "null_value_error",
    "description": "尝试访问空值(None)的属性或进行运算",
    "error": "unsupported operand type(s) for +: 'int' and 'NoneType'"
  },
  "suggestion": "确保所有数值运算的变量都有默认值，如：{% set sum = (a or 0) + (b or 0) %}"
}
```

#### 1.2 字段不存在错误 (null_value_error with field)

**触发场景**：
```jinja2
{{ score.values }}
{# 字典中 'values' 字段不存在或与Python保留字冲突 #}
```

**返回信息**：
```json
{
  "code": "TEMPLATE_RENDER_ERROR",
  "message": "模板渲染失败（第80行）",
  "details": {
    "line": 80,
    "type": "null_value_error",
    "field": "values",
    "description": "尝试访问空值(None)的属性或进行运算"
  },
  "suggestion": "字段 'values' 不存在或为空。使用 .get('values', default_value) 安全访问"
}
```

#### 1.3 除以零错误 (division_by_zero)

**触发场景**：
```jinja2
{{ (numerator / denominator * 100) }}%
{# 当 denominator 为 0 时 #}
```

**返回信息**：
```json
{
  "code": "TEMPLATE_RENDER_ERROR",
  "message": "模板渲染失败（第150行）",
  "details": {
    "line": 150,
    "type": "division_by_zero",
    "description": "除数为0"
  },
  "suggestion": "添加除数检查：{% set ratio = (a / b) if b != 0 else 0 %}"
}
```

#### 1.4 类型错误 (type_error)

**触发场景**：
```jinja2
{% set total = value1 + value2 %}
{# value1 是整数，value2 是 None #}
```

**返回信息**：
```json
{
  "code": "TEMPLATE_RENDER_ERROR",
  "message": "模板渲染失败（第100行）",
  "details": {
    "line": 100,
    "type": "type_error",
    "description": "类型不匹配，通常是None参与了数值运算"
  },
  "suggestion": "确保所有数值运算的变量都有默认值，如：{% set sum = (a or 0) + (b or 0) %}"
}
```

#### 1.5 索引错误 (subscript_error)

**触发场景**：
```jinja2
{{ overview.starttime[:10] }}
{# starttime 是 datetime 对象而不是字符串 #}
```

**返回信息**：
```json
{
  "code": "TEMPLATE_RENDER_ERROR",
  "message": "模板渲染失败（第22行）",
  "details": {
    "line": 22,
    "type": "subscript_error",
    "description": "尝试对不支持索引的对象使用[]或[:n]"
  },
  "suggestion": "检查变量类型，字符串切片前确认是字符串，如：{{ date[:10] if date else '-' }}"
}
```

#### 1.6 字典访问错误 (dict_access_error)

**触发场景**：
```jinja2
{{ score.values }}
{# score 是字典，但 'values' 与字典的 values() 方法冲突 #}
```

**返回信息**：
```json
{
  "code": "TEMPLATE_RENDER_ERROR",
  "message": "模板渲染失败（第80行）",
  "details": {
    "line": 80,
    "type": "dict_access_error",
    "description": "尝试用点号访问字典的键，但该键可能是保留字"
  },
  "suggestion": "使用 .get() 方法或方括号访问字典：{{ dict.get('key', default) }} 或 {{ dict['key'] }}"
}
```

function formatErrorForUser(error) {
  let userMessage = error.message;
  
  // 添加上下文信息
  if (error.details.line) {
    userMessage += `\n位置：模板第 ${error.details.line} 行`;
  }
  
  if (error.details.variable) {
    userMessage += `\n变量：${error.details.variable}`;
  }
  
  if (error.details.field) {
    userMessage += `\n字段：${error.details.field}`;
  }
  
  // 添加修复建议
  if (error.suggestion) {
    userMessage += `\n\n💡 建议：${error.suggestion}`;
  }
  
  // 对于开发者，提供查看详情的选项
  if (isDevelopmentMode()) {
    userMessage += '\n\n[点击查看技术详情]';
  }
  
  return userMessage;
}

## 🔍 调试技巧

### 1. 使用堆栈跟踪定位问题

错误信息中的 `details.traceback` 包含最后10行堆栈：

```javascript
console.log(error.details.traceback.join('\n'));
```

输出：
```
File "/app/services/renderer.py", line 45, in render
  result = template.render(**variables)
File "jinja2/environment.py", line 1301, in render
  self.environment.handle_exception()
File "<template>", line 123, in top-level template code
jinja2.exceptions.UndefinedError: unsupported operand type(s) for +: 'int' and 'NoneType'
```

### 2. 错误分类统计

```javascript
// 统计错误类型分布
const errorStats = {
  'TEMPLATE_RENDER_ERROR': 0,
  'SQL_EXECUTION_ERROR': 0,
  'AI_GENERATION_ERROR': 0,
  // ...
};

function trackError(error) {
  errorStats[error.code] = (errorStats[error.code] || 0) + 1;
  
  // 发送到监控系统
  analytics.track('report_generation_error', {
    code: error.code,
    message: error.message,
    variable: error.details.variable,
    line: error.details.line
  });
}
```

### 3. 开发模式 vs 生产模式

```javascript
function shouldShowDetailedError() {
  return process.env.NODE_ENV === 'development' || 
         localStorage.getItem('show_debug_info') === 'true';
}

function displayError(error) {
  const basicMessage = error.message;
  
  if (shouldShowDetailedError()) {
    // 开发模式：显示完整信息
    return {
      message: basicMessage,
      details: error.details,
      suggestion: error.suggestion,
      traceback: error.details.traceback
    };
  } else {
    // 生产模式：只显示基本信息和建议
    return {
      message: basicMessage,
      suggestion: error.suggestion
    };
  }
}
```

from app.api.reports import _format_error_details
from app.core.exceptions import TemplateRenderError, SqlExecutionError

def test_template_render_error_formatting():

async def test_error_propagation_to_frontend():
    """测试错误能正确传递到前端"""
    # 创建一个会失败的报告生成任务
    task = await create_failing_task()
    
    # 监听WebSocket消息
    messages = []
    async with websocket_client(task.id) as ws:
        async for message in ws:
            messages.append(message)
            if message['type'] == 'task_failed':
                break
    
    # 验证错误信息结构
    error_message = next(m for m in messages if m['type'] == 'task_failed')
    assert 'error' in error_message
    assert 'code' in error_message['error']
    assert 'message' in error_message['error']
    assert 'details' in error_message['error']
    assert 'suggestion' in error_message['error']
