# 变量失败处理修复说明

## 🐛 问题描述

### 现象
用户报告：**后端日志显示变量执行失败，但前端仍然显示报告生成成功**

### 实际问题
从终端日志看到多个AI变量执行失败：
```
[ERROR] app.executors.ai - ❌ 提示词插值失败: Variable 'overview.micro_grid_name' not found for interpolation
[ERROR] app.executors.ai - ❌ 提示词插值失败: Variable 'overview.wgid_score' not found for interpolation
[ERROR] app.executors.ai - ❌ 提示词插值失败: Variable 'overview.problem_build_dj_count' not found for interpolation
```

但前端仍然收到"报告生成成功"的消息。

---

## 🔍 根本原因分析

### 原因1：调度器的失败处理逻辑不完善

**旧逻辑** (`app/services/scheduler.py`):
```python
if result.status == VariableStatus.FAILED:
    metadata = context.metadata[result.variable_name]
    if metadata.required and metadata.default is None:
        # 只有必需变量且无默认值时才中止
        raise Exception(f"Required variable '{result.variable_name}' failed")
    # 否则继续执行，忽略失败
```

**问题**：
- ✅ 必需变量失败 → 任务中止
- ❌ **可选变量失败 → 继续执行 → 最终标记为"成功"**
- ❌ 前端不知道有变量失败了
- ❌ 报告可能是不完整的，但用户以为成功了

### 原因2：字符串插值不支持嵌套属性访问

**旧逻辑** (`app/services/context.py`):
```python
def replace_var(match):
    var_name = match.group(1).strip()
    if not self.has_variable(var_name):
        raise DependencyError(f"Variable '{var_name}' not found")
    value = self.get_variable(var_name)
    return str(value)
```

**问题**：
- ✅ 支持简单变量：`{{wgid}}`
- ❌ **不支持嵌套属性：`{{overview.micro_grid_name}}`**
- ❌ AI提示词中常用嵌套访问，导致插值失败

---

## ✅ 修复方案

### 修复1：增强失败追踪和报告

**新逻辑** (`app/services/scheduler.py`):
```python
all_results = {}
failed_variables = []  # 追踪所有失败的变量

for batch_idx, batch in enumerate(batches):
    batch_results = await self.execute_batch(batch, context)
    
    for result in batch_results:
        all_results[result.variable_name] = result
        
        # 追踪失败的变量
        if result.status == VariableStatus.FAILED:
            failed_variables.append({
                'name': result.variable_name,
                'error': result.error,
                'required': context.metadata[result.variable_name].required
            })
            
            # 必需变量立即中止
            metadata = context.metadata[result.variable_name]
            if metadata.required and metadata.default is None:
                raise Exception(f"Required variable '{result.variable_name}' failed")

# 执行完所有批次后，检查是否有失败
if failed_variables:
    required_failures = [v for v in failed_variables if v['required']]
    optional_failures = [v for v in failed_variables if not v['required']]
    
    error_msg = f"报告生成部分失败: "
    if required_failures:
        error_msg += f"{len(required_failures)}个必需变量失败"
    if optional_failures:
        if required_failures:
            error_msg += f", "
        error_msg += f"{len(optional_failures)}个可选变量失败"
    
    # 添加详细信息
    error_details = []
    for v in failed_variables:
        error_details.append(f"  - {v['name']}: {v['error']}")
    error_msg += "\n" + "\n".join(error_details)
    
    # 抛出异常，触发任务失败
    raise Exception(error_msg)

return all_results
```

**改进**：
- ✅ 追踪所有失败的变量（必需和可选）
- ✅ 在最后检查是否有失败
- ✅ **即使只有可选变量失败，也标记任务为失败**
- ✅ 提供详细的失败列表
- ✅ 前端会收到失败通知

### 修复2：支持嵌套属性访问

**新逻辑** (`app/services/context.py`):
```python
def replace_var(match):
    var_name = match.group(1).strip()
    
    # 支持嵌套属性访问 (e.g., overview.micro_grid_name)
    if '.' in var_name:
        parts = var_name.split('.')
        base_var = parts[0]
        
        if not self.has_variable(base_var):
            raise DependencyError(f"Variable '{base_var}' not found")
        
        value = self.get_variable(base_var)
        
        # 导航到嵌套属性
        for attr in parts[1:]:
            if isinstance(value, dict):
                if attr not in value:
                    raise DependencyError(f"Variable '{var_name}' not found ('{attr}' not in {base_var})")
                value = value[attr]
            elif hasattr(value, attr):
                value = getattr(value, attr)
            else:
                raise DependencyError(f"Variable '{var_name}' not found ('{attr}' not accessible)")
    else:
        # 简单变量访问
        if not self.has_variable(var_name):
            raise DependencyError(f"Variable '{var_name}' not found")
        value = self.get_variable(var_name)
    
    # 转换为字符串
    if isinstance(value, (dict, list)):
        return json.dumps(value, ensure_ascii=False)
    elif value is None:
        return ''  # None 值返回空字符串
    return str(value)
```

**改进**：
- ✅ 支持嵌套属性：`{{overview.micro_grid_name}}`
- ✅ 支持多层嵌套：`{{user.profile.name}}`
- ✅ 同时支持字典和对象属性访问
- ✅ None值返回空字符串而不是报错

## 🎯 使用示例

### 示例1：AI提示词中使用嵌套属性

**旧方式（会失败）**：
```yaml
executive_summary:
  type: "string"
  source: "ai_generation"
  ai_config:
    prompt_template: |
      你是一位网络优化专家。
      
      微网格名称: {{overview.micro_grid_name}}  # ❌ 不支持
      网格评分: {{overview.wgid_score}}         # ❌ 不支持
      
      请生成执行摘要。
```

**新方式（支持）**：
```yaml
executive_summary:
  type: "string"
  source: "ai_generation"
  dependencies: ["overview"]  # 确保依赖overview
  ai_config:
    prompt_template: |
      你是一位网络优化专家。
      
      微网格名称: {{overview.micro_grid_name}}  # ✅ 支持
      网格评分: {{overview.wgid_score}}         # ✅ 支持
      问题数量: {{overview.problem_count}}       # ✅ 支持
      
      请根据以上信息生成执行摘要。
```

### 示例2：SQL查询中使用嵌套属性

```yaml
detailed_query:
  type: "array"
  source: "sql"
  dependencies: ["user", "config"]
  sql_config:
    connection: "main_db"
    query: |
      SELECT *
      FROM reports
      WHERE user_id = :user_id
        AND department = :department
        AND created_at >= :start_date
    parameters:
      - user_id
      - department
      - start_date
```

**参数绑定**：
```python
# 现在支持从嵌套对象中获取参数
user_id = user.id                    # ✅ 支持
department = user.profile.department # ✅ 支持
start_date = config.report.start     # ✅ 支持
```
