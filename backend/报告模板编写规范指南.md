## 1. SQL查询编写规范

### 1.1 基本原则

#### ✅ 必须遵守

1. **禁止使用保留字作为列名**，必须使用别名
2. **所有数据值必须使用参数化查询** (`:param_name`)
3. **SQL结构动态部分使用字符串插值** (`{{variable}}`)
4. **列名使用双引号**（PostgreSQL兼容）或不加引号（MySQL兼容）
5. **表名完全限定**：`schema.table_name`
6. **数值字段使用COALESCE处理NULL**
7. **使用标准SQL函数**，避免数据库特定语法

#### ❌ 禁止事项

- ❌ 禁止直接拼接用户输入到SQL中
- ❌ 禁止使用数据库特定的保留字作为字段名
  - 常见保留字：`values`, `index`, `key`, `order`, `limit`, `select`, `from`, `where`, `group`, `having`, `count`, `sum`, `avg`, `date`, `time`, `timestamp`, `user`, `role`, `type`, `status`, `level`, `rank`, `range`, `offset`, `fetch`
- ❌ 禁止使用数据库特定的数据类型而不提供兼容方案
  - PostgreSQL特定：`::geometry`, `::json`, `::uuid`, `ARRAY[]`
  - MySQL特定：反引号、`LIMIT offset, count`语法
- ❌ 禁止假设字段非NULL，必须使用COALESCE

### 1.2 SQL编写模板

#### 模板1：基础查询（带参数化）

```yaml
# ✅ 正确示例
query: |
  SELECT 
    wgid,
    micro_grid_name,
    city,
    area,
    COALESCE(problem_count, 0) AS problem_count,
    COALESCE(wgid_score, 0.0) AS wgid_score,
    starttime,
    endtime
  FROM microgrid.micro_grid_overview_w
  WHERE wgid = :wgid
    AND starttime >= :start_date
parameters:
  - wgid
  - start_date
```

```yaml
# ❌ 错误示例 - 直接拼接会导致SQL注入和引号问题
query: |
  SELECT * FROM table WHERE wgid = {{wgid}}
# 问题：当wgid='ZQGY0174'时会生成WHERE wgid = ZQGY0174（缺少引号）
```

**说明**：
- `:param_name` 会被数据库驱动自动转义和加引号
- `{{variable}}` 直接字符串替换，仅用于SQL结构（表名、列名等）

---

#### 模板2：动态表名 + 参数化条件

```yaml
# ✅ 正确示例：结构用插值，数据用参数化
query: |
  SELECT 
    wgid,
    grid_name,
    COALESCE(score, 0.0) AS score
  FROM {{schema_name}}.{{table_name}}
  WHERE wgid = :wgid
    AND status = :status
parameters:
  - wgid
  - status
```

**使用场景**：
- 表名需要动态切换（测试环境vs生产环境）
- 列名需要动态指定
- ORDER BY字段需要动态
- JOIN条件需要动态

---

#### 模板3：处理保留字字段名

```yaml
# ✅ 正确：使用AS别名避免保留字冲突
query: |
  SELECT 
    "index" AS index_name,          -- PostgreSQL用双引号
    "values" AS value_data,         -- 避免与Python字典的values()方法冲突
    "order" AS sort_order,
    "key" AS key_name,
    "count" AS record_count,
    "type" AS type_name,
    "level" AS level_value
  FROM microgrid.index_scores_w
  WHERE wgid = :wgid
parameters:
  - wgid
```

```yaml
# ❌ 错误：直接使用保留字
query: |
  SELECT "index", "values", "order"
  FROM table
# 会导致模板渲染时报错：'dict object' has no attribute 'values'
# 因为Python字典有values()方法，Jinja2会混淆
```

**重要保留字清单**：

| 类别 | 保留字 | 推荐别名 |
|------|--------|----------|
| SQL关键字 | `index`, `key`, `order`, `limit`, `offset` | `index_name`, `key_name`, `sort_order`, `limit_value`, `offset_value` |
| Python内置 | `values`, `keys`, `items`, `type`, `sum`, `count` | `value_data`, `key_list`, `item_list`, `type_name`, `sum_value`, `count_value` |
| 聚合函数 | `count`, `sum`, `avg`, `max`, `min` | `count_value`, `sum_value`, `avg_value`, `max_value`, `min_value` |
| 时间相关 | `date`, `time`, `timestamp`, `year`, `month` | `date_value`, `time_value`, `timestamp_value`, `year_value`, `month_value` |

#### 模板4：聚合查询

```yaml
# ✅ 正确：使用COALESCE处理NULL
query: |
  SELECT 
    wgid,
    COALESCE(SUM(problem_count), 0) AS total_problems,
    COALESCE(AVG(score), 0.0) AS avg_score,
    COALESCE(MAX(score), 0.0) AS max_score,
    COALESCE(MIN(score), 0.0) AS min_score,
    COUNT(*) AS total_records,
    COUNT(DISTINCT city) AS city_count
  FROM microgrid.overview
  WHERE wgid = :wgid
  GROUP BY wgid
parameters:
  - wgid
```

**关键点**：
- 所有聚合函数外层包裹COALESCE
- SUM可能返回NULL（没有数据时）
- AVG可能返回NULL
- COUNT(*)永远不为NULL，COUNT(column)可能为0

#### 模板5：跨数据库兼容SQL

```yaml
# ✅ 正确：避免数据库特定语法
query: |
  SELECT 
    wgid,
    micro_grid_name,
    -- 日期函数：PostgreSQL和MySQL都支持
    DATE(starttime) AS start_date,
    -- 类型转换：使用标准CAST
    CAST(longitude AS DECIMAL(10,6)) AS lng,
    CAST(latitude AS DECIMAL(10,6)) AS lat,
    -- 字符串截取：使用标准SUBSTRING
    SUBSTRING(wgid, 1, 4) AS wgid_prefix,
    -- 字符串拼接：使用标准CONCAT
    CONCAT(city, '-', area) AS location,
    -- 条件表达式：使用标准CASE
    CASE 
      WHEN score >= 90 THEN '优秀'
      WHEN score >= 80 THEN '良好'
      ELSE '需改进'
    END AS rating
  FROM microgrid.micro_grid_overview_w
  WHERE wgid = :wgid
    AND DATE(starttime) >= :start_date
parameters:
  - wgid
  - start_date
```

```yaml
# ❌ 错误：PostgreSQL特定语法
query: |
  SELECT 
    geom::geometry,           -- MySQL不支持::类型转换
    date_field::date,         -- 应该用CAST(date_field AS DATE)
    value::numeric,           -- 应该用CAST(value AS DECIMAL)
    ARRAY[1,2,3],            -- MySQL不支持数组类型
    data->>'key'             -- MySQL不支持JSON运算符
  FROM table
```

**跨数据库兼容函数对照表**：

| 功能 | PostgreSQL | MySQL | 标准SQL (推荐) |
|------|------------|-------|----------------|
| 类型转换 | `field::type` | `CAST(field AS type)` | `CAST(field AS type)` |
| 日期截取 | `field::date` | `DATE(field)` | `DATE(field)` 或 `CAST(field AS DATE)` |
| 字符串截取 | `SUBSTR(str, pos, len)` | `SUBSTRING(str, pos, len)` | `SUBSTRING(str, pos, len)` |
| 字符串拼接 | `str1 || str2` | `CONCAT(str1, str2)` | `CONCAT(str1, str2)` |
| 当前时间 | `NOW()` | `NOW()` | `CURRENT_TIMESTAMP` |
| 限制行数 | `LIMIT n OFFSET m` | `LIMIT m, n` | `LIMIT n OFFSET m` (推荐) |

#### 模板6：动态列和动态ORDER BY

```yaml
# ✅ 正确：动态列名和排序
query: |
  SELECT 
    wgid,
    {{select_columns}}  -- 动态列名：interpolation
  FROM {{schema_name}}.micro_grid_overview_w
  WHERE wgid = :wgid    -- 数据值：parameterization
  ORDER BY {{order_column}} {{order_direction}}
parameters:
  - wgid
```

**在元数据中设置变量**：
```yaml
select_columns:
  type: "string"
  source: "user_input"
  default: "micro_grid_name, city, area, score"

order_column:
  type: "string"
  source: "user_input"
  default: "score"
  validation:
    enum: ["score", "problem_count", "city"]  # 白名单限制

order_direction:
  type: "string"
  source: "user_input"
  default: "DESC"
  validation:
    enum: ["ASC", "DESC"]
```

### 1.3 SQL结果模式 (result_mode)

选择合适的结果模式：

```yaml
# 场景1：查询单条记录（如：微网格概况）
result_mode: "first_row"
# 返回：{col1: val1, col2: val2}
# 模板中使用：{{ overview.col1 }}

# 场景2：查询多条记录（如：问题楼宇列表）
result_mode: "all_rows"
# 返回：[{row1}, {row2}, ...]
# 模板中使用：{% for item in list %}

# 场景3：查询单个统计值（如：问题总数）
result_mode: "first_value"
# 返回：单个值（数字或字符串）
# 模板中使用：{{ count }}

# 场景4：查询一列数组（如：所有城市名）
result_mode: "first_column"
# 返回：[val1, val2, ...]
# 模板中使用：{% for city in cities %}

# 场景5：自动判断（不推荐，不明确）
result_mode: "auto"
```

**选择建议**：
- 明确指定result_mode，不要使用auto
- first_row用于单条详情
- all_rows用于列表、表格
- first_value用于计数、求和等标量
- first_column用于枚举值、标签列表

## 2. Jinja2模板编写规范

### 2.1 基本原则

#### ✅ 必须遵守

1. **假设所有字段可能为NULL/None**
2. **数值运算使用 `or 0` 提供默认值**
3. **除法运算检查分母不为0**
4. **字典访问优先使用 `.get()` 或方括号 `[]`**
5. **日期字段统一处理**（字符串切片或过滤器）
6. **格式化前检查 `is not none`**
7. **列表遍历前检查非空**
8. **提供降级显示**（用 `-` 或文字说明替代缺失数据）

#### ❌ 禁止事项

- ❌ 禁止直接运算不检查None：`{{ a + b }}`
- ❌ 禁止除法不检查分母：`{{ a / b }}`
- ❌ 禁止假设字段存在：`{{ obj.field }}`（字典应用`.get()`）
- ❌ 禁止假设日期类型：`.strftime()` 或 `[:10]` 需确认类型
- ❌ 禁止遍历前不检查：`{% for item in list %}`（list可能为None）

---

### 2.2 数据访问规范

#### 规范1：安全字段访问

```jinja2
{# ✅ 正确：字典使用get方法 #}
{{ score.get('index_name', '-') }}
{{ score.get('value', 0) }}

{# ✅ 正确：字典使用方括号（如果确定键存在） #}
{{ score['index_name'] }}

{# ✅ 正确：对象使用or提供默认值 #}
{{ overview.city or '-' }}
{{ overview.problem_count or 0 }}

{# ❌ 错误：直接访问字典可能不存在的字段 #}
{{ score.index_name }}  {# 如果score是字典，且index_name不存在会报错 #}

{# ❌ 错误：不提供默认值 #}
{{ overview.city }}  {# 如果为None会显示"None" #}
```

**访问规则**：
- **确定是对象**：用 `.field or default`
- **确定是字典**：用 `.get('field', default)`
- **不确定类型**：优先用 `.get()`，更安全
- **必须存在的字段**：可用 `.field` 或 `['field']`，但要有错误处理预案

---

#### 规范2：数值运算保护

```jinja2
{# ✅ 正确：所有数值提供默认值0 #}
{% set total = (value1 or 0) + (value2 or 0) + (value3 or 0) %}

{# ✅ 正确：复杂运算每个变量都保护 #}
{% set total_problems = (overview.problem_count or 0) + (overview.gr_problem_count or 0) + (overview.jg_problem_count or 0) + (overview.wh_problem_count or 0) %}

{# ✅ 正确：除法检查分母 #}
{% set ratio = ((numerator or 0) / (denominator or 1)) if (denominator or 0) != 0 else 0 %}

{# ✅ 正确：百分比计算完整保护 #}
{{ "%.1f"|format(((numerator or 0) / (denominator or 1) * 100) if (denominator or 0) > 0 else 0) }}%

{# ❌ 错误：直接运算 #}
{% set total = value1 + value2 %}  
{# 如果value1或value2是None会报错：unsupported operand type(s) for +: 'int' and 'NoneType' #}

{# ❌ 错误：除法不检查分母 #}
{{ (numerator / denominator * 100) }}%  
{# 分母为0会报错：ZeroDivisionError #}
```

**运算保护模板**：
```jinja2
{# 加法 #}
{% set sum = (a or 0) + (b or 0) %}

{# 减法 #}
{% set diff = (a or 0) - (b or 0) %}

{# 乘法 #}
{% set product = (a or 0) * (b or 1) %}

{# 除法（安全） #}
{% set quotient = ((a or 0) / (b or 1)) if (b or 0) != 0 else 0 %}

{# 百分比 #}
{% set percentage = (((a or 0) / (b or 1) * 100) if (b or 0) > 0 else 0) %}
```

---

#### 规范3：日期时间处理

```jinja2
{# ✅ 正确：统一处理字符串格式（ISO 8601） #}
{# 数据库返回的datetime已被序列化为 "2025-01-01T00:00:00" 格式 #}
{{ overview.starttime[:10] if overview.starttime else '-' }}

{# ✅ 正确：提取年月 #}
{{ overview.starttime[:7] if overview.starttime else '-' }}  {# 2025-01 #}

{# ✅ 正确：提取年份 #}
{{ overview.starttime[:4] if overview.starttime else '-' }}  {# 2025 #}

{# ✅ 推荐：如果配置了自定义过滤器 #}
{{ overview.starttime | to_date }}
{{ overview.starttime | to_date('%Y年%m月%d日') }}

{# ❌ 错误：假设是datetime对象 #}
{{ overview.starttime.strftime('%Y-%m-%d') }}  
{# 如果是字符串会报错：'str object' has no attribute 'strftime' #}

{# ❌ 错误：假设是字符串 #}
{{ overview.starttime[:10] }}  
{# 如果是datetime对象会报错：'datetime.datetime' object is not subscriptable #}
```

**当前系统行为**：
- 数据库connector会将datetime转换为ISO 8601字符串
- 因此模板中应该使用字符串切片 `[:10]`
- 如果未来改为保留datetime对象，需要使用 `.strftime()`

#### 规范4：格式化输出

```jinja2
{# ✅ 正确：检查None #}
{{ "%.2f"|format(value) if value is not none else '-' }}
{{ "%.1f"|format(value or 0) }}

{# ✅ 正确：多种格式 #}
{# 浮点数2位小数 #}
{{ "%.2f"|format(score) if score is not none else '-' }}

{# 浮点数1位小数 #}
{{ "%.1f"|format(ratio) if ratio is not none else '-' }}

{# 整数 #}
{{ "%d"|format(count) if count is not none else 0 }}

{# 百分比 #}
{{ "%.1f"|format(percentage) if percentage is not none else 0 }}%

{# ❌ 错误：不检查None #}
{{ "%.2f"|format(value) }}  
{# value为None时报错：unsupported format string passed to NoneType.__format__ #}
```

**格式化模板**：
```jinja2
{# 金额（2位小数） #}
{{ "%.2f"|format(amount) if amount is not none else '0.00' }}

{# 评分（1位小数） #}
{{ "%.1f"|format(score) if score is not none else '-' }}

{# 数量（整数） #}
{{ count or 0 }}

{# 百分比（1位小数） #}
{{ "%.1f"|format((value or 0)) }}%

{# 经纬度（6位小数） #}
{{ "%.6f"|format(longitude) if longitude is not none else '-' }}
```

#### 规范5：条件判断

```jinja2
{# ✅ 正确：明确检查 #}
{% if (value or 0) > 0 %}
{% if value is not none and value > 0 %}
{% if overview.problem_count and overview.problem_count > 0 %}

{# ✅ 正确：检查存在性 #}

{% if problem_list %}
```

```jinja2
{% if (score or 0) >= 90 %}
{% elif (score or 0) >= 80 %}
{% else %}

{# 检查字符串非空 #}

```

#### 规范6：聚合操作

```jinja2
{# ✅ 正确：提供start参数 #}
{% set total = items | sum(attribute='value', start=0) %}
{% set sorted_items = items | sort(attribute='score', reverse=true) %}

{# ✅ 正确：检查列表是否为空 #}
{% if items %}
  {% set total_count = items | length %}
  {% for item in items %}
    {{ item.name }}
  {% endfor %}
{% else %}
  无数据
{% endif %}

{# ✅ 正确：安全的平均值计算 #}
{% set sum_value = items | sum(attribute='score', start=0) %}
{% set avg_value = (sum_value / (items | length)) if items else 0 %}

{# ✅ 正确：取前N项 #}
{% for item in items[:20] %}  {# 前20个 #}

{# ✅ 正确：条件过滤 #}
{% set critical_items = items | selectattr('is_critical', 'equalto', true) | list %}

{# ❌ 错误：不提供start参数 #}
{% set total = items | sum(attribute='value') %}
{# items为空时返回None而不是0 #}

{# ❌ 错误：不检查列表是否为空 #}
{% for item in items %}
{# items为None时会报错 #}
```

**聚合操作模板**：
```jinja2
{# 求和 #}
{% set total = items | sum(attribute='count', start=0) %}

{# 平均值 #}
{% set avg = (total / (items | length)) if items else 0 %}

{# 最大值 #}
{% set max_item = items | max(attribute='score') if items else none %}

{# 最小值 #}
{% set min_item = items | min(attribute='score') if items else none %}

{# 排序 #}
{% set sorted_asc = items | sort(attribute='name') %}
{% set sorted_desc = items | sort(attribute='score', reverse=true) %}

{# 过滤 #}
{% set filtered = items | selectattr('status', 'equalto', 'active') | list %}
{% set high_scores = items | selectattr('score', 'greaterthan', 80) | list %}

{# 分组 #}
{% set grouped = items | groupby('category') %}
{% for category, group_items in grouped %}
  {{ category }}: {{ group_items | length }}
{% endfor %}
```

### 2.3 表格渲染规范

```jinja2
{# ✅ 正确：完整的表格模板 #}
{% if problem_buildings %}

### 问题楼宇清单

共发现 **{{ problem_buildings | length }}** 个问题楼宇。

| 序号 | 楼宇名称 | 楼宇类型 | 高度(m) | 面积(m²) | RSRP(dBm) | 弱覆盖比例 | 状态 |
|------|----------|----------|---------|----------|-----------|------------|------|
{% for building in problem_buildings[:20] %}
| {{ loop.index }} | {{ building.get('bldg_name', '-') }} | {{ building.get('build_type', '-') }} | {{ "%.1f"|format(building.get('height_m')) if building.get('height_m') is not none else '-' }} | {{ "%.1f"|format(building.get('area_m')) if building.get('area_m') is not none else '-' }} | {{ "%.2f"|format(building.get('rsrp')) if building.get('rsrp') is not none else '-' }} | {{ "%.1f"|format(building.get('weak_ratio', 0)) }}% | {% if (building.get('rsrp') or 0) < -110 %}🔴{% elif (building.get('rsrp') or 0) < -100 %}🟡{% else %}🟢{% endif %} |
{% endfor %}

**说明**：仅显示前20个问题楼宇，RSRP < -110dBm为弱覆盖。

{% else %}
未发现问题楼宇。✅
{% endif %}
```

## 4. 常见陷阱与解决方案

### 陷阱1：NULL值运算 ⚠️

**问题表现**：
```python
TypeError: unsupported operand type(s) for +: 'int' and 'NoneType'
```

**触发场景**：
```jinja2
{% set total = overview.problem_count + overview.gr_problem_count %}
```

**原因分析**：
- 数据库字段可能返回NULL
- NULL在Python中转换为None
- None参与数值运算会报错

**解决方案**：
```jinja2
{# ✅ 方案1：使用or提供默认值 #}
{% set total = (overview.problem_count or 0) + (overview.gr_problem_count or 0) %}

{# ✅ 方案2：在SQL中使用COALESCE #}
{# SQL: SELECT COALESCE(problem_count, 0) AS problem_count #}
{% set total = overview.problem_count + overview.gr_problem_count %}
```

---

### 陷阱2：保留字字段名冲突 ⚠️

**问题表现**：
```python
'dict object' has no attribute 'values'
jinja2.exceptions.UndefinedError: 'dict object' has no attribute 'values'
```

**触发场景**：
```sql
-- SQL中有保留字字段
SELECT "index", "values", "order" FROM table
```

```jinja2
-- 模板中访问
{{ score.values }}  {# 错误！Python字典有values()方法 #}
```

**原因分析**：
- SQL列名 `values` 与Python字典的 `values()` 方法冲突
- Jinja2尝试调用字典的 `values()` 方法而不是访问键

**解决方案**：
```sql
-- ✅ SQL中使用别名
SELECT 
  "index" AS index_name,
  "values" AS value_data,
  "order" AS sort_order
FROM table
```

```jinja2
{# ✅ 模板中访问别名 #}
{{ score.value_data }}

{# ✅ 或使用方括号访问 #}
{{ score['values'] }}

{# ✅ 或使用get方法 #}
{{ score.get('values') }}
```

---

### 陷阱3：日期类型不一致 ⚠️

**问题表现**：
```python
'str object' has no attribute 'strftime'
'datetime.datetime' object is not subscriptable
```

**触发场景**：
```jinja2
{# 假设是datetime对象 #}
{{ overview.starttime.strftime('%Y-%m-%d') }}

{# 假设是字符串 #}
{{ overview.starttime[:10] }}
```

**原因分析**：
- 数据库返回datetime对象
- DatabaseConnector序列化为ISO 8601字符串
- 模板中假设了错误的类型

**解决方案**：
```jinja2
{# ✅ 当前系统：数据库已序列化为字符串，使用字符串切片 #}
{{ overview.starttime[:10] if overview.starttime else '-' }}

{# ✅ 如果确定是datetime对象，使用strftime #}
{{ overview.starttime.strftime('%Y-%m-%d') if overview.starttime else '-' }}

{# ✅ 推荐：使用自定义过滤器（统一处理） #}
{{ overview.starttime | to_date }}
```

**长期方案**：在renderer.py中添加自定义过滤器
```python
@staticmethod
def _to_date(value, format='%Y-%m-%d'):
    """统一处理日期，支持datetime对象和字符串"""
    if isinstance(value, (datetime, date)):
        return value.strftime(format)
    elif isinstance(value, str):
        return value[:10] if len(value) >= 10 else value
    return '-'
```

### 陷阱4：除以零 ⚠️

**问题表现**：
```python
ZeroDivisionError: division by zero
```

**触发场景**：
```jinja2
{{ "%.1f"|format(numerator / denominator * 100) }}%
```

**原因分析**：
- 分母可能为0（没有数据时）
- 分母可能为None

**解决方案**：

```jinja2
{{ "%.1f"|format(((numerator or 0) / (denominator or 1) * 100) if (denominator or 0) > 0 else 0) }}%

{# ✅ 或分步处理 #}
{% set safe_numerator = numerator or 0 %}
{% set safe_denominator = denominator or 1 %}
{% set ratio = (safe_numerator / safe_denominator * 100) if safe_denominator > 0 else 0 %}
{{ "%.1f"|format(ratio) }}%
```

### 陷阱5：数据库特定语法 ⚠️

**问题表现**：
```
SQL在PostgreSQL正常运行，但在MySQL报错
```

**触发场景**：
```sql
-- PostgreSQL特定语法
SELECT 
  geom::geometry,
  date_field::date,
  data->>'key' AS json_key
FROM table
```

**原因分析**：
- `::` 类型转换是PostgreSQL特有
- `->>`JSON运算符是PostgreSQL特有
- MySQL不支持这些语法

**解决方案**：
```sql
-- ✅ 使用标准SQL
SELECT 
  ST_AsText(geom) AS geom_text,           -- 几何转文本
  CAST(date_field AS DATE) AS date_value, -- 标准类型转换
  JSON_EXTRACT(data, '$.key') AS json_key -- 标准JSON提取
FROM table
```

**跨数据库兼容检查清单**：
- [ ] 类型转换使用 `CAST(field AS type)`
- [ ] 字符串拼接使用 `CONCAT()`
- [ ] 字符串截取使用 `SUBSTRING()`
- [ ] 日期截取使用 `DATE()`
- [ ] 限制行数使用 `LIMIT n OFFSET m`
- [ ] 避免特定数据类型（geometry, uuid, json等）

### D. SQL查询模板

```sql
-- 模板1: 基础查询 + 参数化
SELECT 
  wgid,
  micro_grid_name,
  COALESCE(problem_count, 0) AS problem_count,
  COALESCE(score, 0.0) AS score,
  starttime,
  endtime
FROM microgrid.micro_grid_overview_w
WHERE wgid = :wgid
  AND DATE(starttime) >= :start_date

-- 模板2: 聚合统计
SELECT 
  COALESCE(COUNT(*), 0) AS total_count,
  COALESCE(SUM(problem_count), 0) AS total_problems,
  COALESCE(AVG(score), 0.0) AS avg_score,
  COALESCE(MAX(score), 0.0) AS max_score,
  COALESCE(MIN(score), 0.0) AS min_score
FROM microgrid.overview
WHERE wgid = :wgid

-- 模板3: 列表查询 + 排序限制
SELECT 
  bldg_id,
  bldg_name,
  build_type,
  CAST(height_m AS DECIMAL(10,1)) AS height_m,
  CAST(rsrp AS DECIMAL(10,2)) AS rsrp,
  is_bad_other_build
FROM microgrid.problem_buildings_w
WHERE wgid = :wgid
  AND (is_bad_cov_4g = true OR is_bad_cov_5g = true)
ORDER BY rsrp ASC
LIMIT 100

-- 模板4: 动态列 + 参数化条件
SELECT 
  wgid,
  {{select_columns}}
FROM {{schema_name}}.micro_grid_overview_w
WHERE wgid = :wgid
  AND status = :status
ORDER BY {{order_column}} {{order_direction}}
```
