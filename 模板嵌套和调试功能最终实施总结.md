## 🎯 实现功能

### 1. ✅ 模板嵌套/包含功能

**核心特性**:
- 支持 `{% include "template_id" %}` 语法嵌套其他模板
- 每个子模板独立执行其变量定义
- 子模板只能访问其自己元数据中定义的变量
- 自动检测并防止循环嵌套
- 支持多级嵌套（A包含B，B包含C）
- 错误处理优雅降级（显示错误信息而不中断）

**实现文件**:
- `backend/app/services/renderer.py`
  - 添加 `_resolve_includes()` 方法 - 递归解析include标签
  - 添加 `_execute_and_render_template()` 方法 - 独立执行子模板变量
- `backend/app/api/reports.py`
  - 在 `execute_report_generation()` 中集成include解析
- `frontend/src/pages/templates/TemplateEdit.tsx`
  - 添加模板嵌套功能说明Alert

**技术亮点**:
- 使用visited集合追踪已处理的template_id，防止循环嵌套
- 为每个子模板创建独立的ExecutionContext和Scheduler
- 正则表达式匹配：`{%\s*include\s+"([^"]+)"\s*%}`
- 反向处理matches避免索引位移问题

### 2. ✅ 集成调试功能

**核心特性**:
- 在模板编辑页面右侧直接显示调试面板
- 真实变量执行（SQL/API/AI）
- 左右分栏布局：编辑区（左14列） + 调试区（右10列）
- Monaco编辑器支持Jinja2和YAML语法高亮
- 动态生成用户输入表单（根据元数据自动创建）
- 实时Markdown渲染预览
- 变量执行详情展示（状态、耗时、错误）

**实现文件**:

**后端**:
- `backend/app/api/debug.py` - 调试API端点
- `backend/app/schemas/api_schemas.py` - 调试相关Schema
- `backend/app/main.py` - 注册调试路由

**前端**:
- `frontend/src/pages/templates/TemplateEdit.tsx` - 集成调试面板到右侧
- `frontend/src/api/debugApi.ts` - 调试API客户端（独立文件，不导出到index.ts）

**技术亮点**:
- 左右分栏响应式布局（Row + Col）
- 右侧面板position: sticky保持可见
- 实时YAML解析和动态表单生成
- Markdown实时预览（ReactMarkdown）
- 变量执行结果表格化展示
- 避免循环依赖（debugApi.ts不导出到api/index.ts）

## ⚙️ 技术架构

### 模板嵌套流程

```
主模板内容
  ↓
_resolve_includes() 解析{% include %}标签
  ↓
for each include:
  - 检查循环嵌套（visited set）
  - 从数据库加载子模板
  - 递归解析子模板的includes
  - 创建独立ExecutionContext
  - 执行子模板变量
  - 渲染子模板
  - 替换include标签
  ↓
返回完全解析的模板
  ↓
主模板渲染
```

### 调试功能流程

```
前端（TemplateEdit右侧面板）:
1. 监听metadataYaml变化 → 解析YAML → 生成用户输入表单
   ↓
2. 用户填写输入 → 点击"执行调试"
   ↓
3. POST /api/debug/render

后端:
4. 解析metadata YAML
   ↓
5. 创建临时ExecutionContext
   ↓
6. 执行所有变量（真实SQL/API/AI调用）
   ↓
7. 解析includes（如有）
   ↓
8. 渲染模板
   ↓
9. 返回结果 + 变量执行详情

前端:
10. 显示Markdown渲染预览
    ↓
11. 显示变量执行表格
```

## 🐛 故障排查过程

### 问题1：前端编译挂起
**现象**: 访问 `/templates` 等页面时，main.tsx 加载挂起，页面完全空白

**原因**: 初始版本的 `TemplatePlayground.tsx` 组件可能存在导入或语法问题，导致Vite编译时卡住

**解决方案**:
1. 临时移除所有调试相关文件和导入
2. 清理Vite缓存（`node_modules/.vite`）
3. 重新实现调试功能，集成到TemplateEdit而不是独立页面
4. 使用 `debugApi.ts` 而不是 `debug.ts`，且不导出到 `api/index.ts`

### 问题2：浏览器强缓存
**现象**: 代码已更新但浏览器仍显示旧版本页面

**原因**: 浏览器对静态资源（JS/CSS）有强缓存策略

**解决方案**:
- 使用无痕模式测试新功能
- 或硬刷新（Ctrl + Shift + F5）
- 或清除站点数据（F12 → Application → Clear site data）

### 问题3：Card标签缩进错误
**现象**: JSX语法错误，Babel解析失败

**原因**: 修改布局为Row/Col时，Card标签缩进没有正确调整

**解决方案**:
- 确保所有Card及其子元素都正确缩进
- Form内的元素多缩进一级

## 🚀 使用示例

### 示例1：测试简单模板

1. 打开模板编辑页面
2. 编辑模板内容：
   ```jinja2
   # {{title}}
   
   内容：{{content}}
   ```

3. 编辑元数据：
   ```yaml
   title:
     type: string
     source: user_input
     required: true
     description: 标题
   
   content:
     type: string
     source: user_input
     required: true
     description: 内容
   ```

4. 右侧面板自动生成两个输入框
5. 填写：
   - 标题：`测试报告`
   - 内容：`这是测试内容`

6. 点击"执行调试"
7. 查看右侧的渲染结果和变量详情

### 示例2：测试模板嵌套

**主模板**:
```jinja2
# {{report_title}}

## 销售总结
{% include "sales_summary" %}
```

**子模板** (ID: `sales_summary`):
```jinja2
- 总销售额：{{total_sales}}元
- 订单数量：{{order_count}}笔
```

执行调试时，系统会：
1. 加载子模板
2. 执行子模板的变量（total_sales, order_count）
3. 渲染子模板
4. 替换到主模板中
5. 渲染最终结果

## 📝 故障排查指南

### 问题1：调试面板不显示
**检查**:
- 浏览器是否清除了缓存
- 前端服务是否正常启动
- 浏览器控制台是否有错误

### 问题2：用户输入表单为空
**原因**: 元数据中没有 `source: user_input` 的变量

**解决**: 这是正常的，表示模板不需要用户输入

### 问题3：执行调试失败
**可能原因**:
- YAML格式错误
- 数据库连接未配置（SQL变量）
- AI API密钥未配置（AI变量）

**查看**: 变量执行详情表格中的错误信息

### 问题4：include标签未渲染
**可能原因**:
- 子模板ID不存在
- 子模板有循环嵌套

**解决**: 检查模板列表，确认子模板ID正确
