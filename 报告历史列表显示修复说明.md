# 报告历史列表显示修复说明

## 问题现象

用户发现报告历史列表中**只显示成功的报告**，没有显示失败、进行中、等待中或已取消状态的报告。

## 问题根因

### 数据流分析

```
任务创建 → 任务执行 → 任务完成
   ↓           ↓           ↓
PENDING → RUNNING → SUCCESS/FAILED/CANCELLED
```

**原有逻辑**：
- ❌ 任务创建时：只创建`GenerationTask`记录，**不创建**`Report`记录
- ❌ 任务执行中：更新`GenerationTask`状态，`Report`仍不存在
- ✅ 任务成功时：创建`Report`记录（状态=SUCCESS）
- ❌ 任务失败/取消时：更新`GenerationTask`状态，`Report`仍不存在

**结果**：报告列表查询`Report`表，只能查到成功的报告。

## 修复方案

### 核心思路

**Report记录应该与GenerationTask记录同生共存**：
- 任务创建时 → 立即创建Report记录（状态=PENDING）
- 任务状态变化时 → 同步更新Report状态
- 任务完成时 → 更新Report内容和状态

#### 3. 任务成功时更新Report（而非创建）

**位置**：`execute_report_generation`函数成功分支（约line 418-446）

**修改前**：
```python

    id=report_id,
    template_id=template_id,
    task_id=task_id,
    title=title,

```

**修改后**：
```python
# Update existing report with success status and content
report = db_session.query(Report).filter(Report.task_id == task_id).first()
if not report:
    # Fallback: create report if it doesn't exist (shouldn't happen normally)
    report_id = f"rpt_{uuid.uuid4().hex[:12]}"
    report = Report(
        id=report_id,
        template_id=template_id,
        task_id=task_id,
        title=title,
        status=ReportStatus.SUCCESS,
        markdown_content=markdown_content,
        duration_ms=duration_ms
    )
    db_session.add(report)
else:
    # Update existing report
    report_id = report.id
    report.title = title
    report.status = ReportStatus.SUCCESS
    report.markdown_content = markdown_content
    report.duration_ms = duration_ms
```

#### 4. 任务失败时更新Report状态

**位置**：`execute_report_generation`函数异常处理（约line 464-483）

```python
if report:
    report.status = ReportStatus.FAILED
    report.duration_ms = duration_ms

```

## 数据库一致性保证

### Report与GenerationTask的生命周期同步

| 事件 | GenerationTask状态 | Report状态 | Report内容 |
|------|-------------------|------------|----------|
| 任务创建 | PENDING | PENDING | NULL |
| 任务开始 | RUNNING | RUNNING | NULL |
| 任务成功 | SUCCESS | SUCCESS | markdown_content填充 |
| 任务失败 | FAILED | FAILED | NULL |
| 任务取消 | CANCELLED | CANCELLED | NULL |

### 关键原则

1. **1:1关系**：每个GenerationTask对应一个Report，通过`task_id`关联
2. **同步创建**：Task创建时立即创建Report
3. **同步状态**：Task状态变化时同步更新Report状态
4. **向后兼容**：如果Report不存在（历史数据），降级处理创建新的

## 历史数据迁移

### 问题

现有的成功报告有对应的Report记录，但历史上的失败/取消任务没有Report记录。

### 解决方案（可选）

如果需要迁移历史数据，可以运行以下SQL：

```sql
-- 为没有Report记录的GenerationTask创建对应的Report
INSERT INTO reports (id, template_id, task_id, title, status, markdown_content, created_at)
SELECT 
    CONCAT('rpt_', SUBSTR(MD5(RANDOM()::TEXT), 1, 12)),
    t.template_id,
    t.id,
    CONCAT('Report - ', t.id),
    t.status,
    NULL,
    t.created_at
FROM generation_tasks t
LEFT JOIN reports r ON r.task_id = t.id
WHERE r.id IS NULL;
```

**注意**：通常不需要执行此迁移，因为：
1. 新任务会自动创建Report
2. 历史失败任务通常不需要在列表中显示
