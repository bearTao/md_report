## 📈 功能效果

### 配置简化

**修改前**（需要声明依赖）:
```yaml
api_base_url:
  source: constant
  value: "http://10.10.20.10:5000"

var1: { dependencies: [api_base_url], ... }
var2: { dependencies: [api_base_url], ... }
var3: { dependencies: [api_base_url], ... }
# ... 20 个变量 = 20 行 dependencies
```

**修改后**（自动可用）:
```yaml
api_base_url:
  source: constant
  value: "http://10.10.20.10:5000"

var1: { ... }  # 无需 dependencies
var2: { ... }  # 无需 dependencies
var3: { ... }  # 无需 dependencies
# ... 20 个变量 = 0 行 dependencies
```

**减少**: 20 行代码（减少 95%）

### 类型保持

```yaml
min_salary:
  source: constant
  value: 15000  # 数字

api_body:
  min_salary: "{{min_salary}}"  # → 15000 (数字) ✅
                                 # 不是 "15000" (字符串) ❌
```

## 🔄 执行流程

### 三阶段执行模型

```
阶段 1: 预执行所有 CONSTANT 变量
   ├─ 并行执行
   ├─ 自动注入到 context
   └─ 失败不影响其他变量

阶段 2: 构建非常量变量的 DAG
   ├─ 只包含非常量变量
   └─ 分析依赖关系

阶段 3: 执行非常量变量
   └─ 按依赖顺序批量执行
```

### 执行顺序示例

```
变量定义:
  api_base_url: constant
  min_salary: constant
  department: user_input
  user_query: api (使用 api_base_url 和 min_salary)

实际执行顺序:
  1. api_base_url (阶段1)
  2. min_salary (阶段1)
  3. department (阶段3)
  4. user_query (阶段3)
```

## 🚨 注意事项

### 1. 必须重启后端服务

数据库迁移后，**必须重启后端服务**，以清除 SQLAlchemy 缓存。

### 2. 三层同步

添加新的 source 类型时，必须同步更新：
- Python 枚举（`app/core/models.py` + `app/models/db_models.py`）
- 数据库 Schema（执行 SQL 迁移）
- 前端类型（`frontend/src/types/index.ts`）

### 3. 常量 vs 用户输入

- **常量**: 固定不变的配置值
- **用户输入**: 每次报告生成都可能变化的值
