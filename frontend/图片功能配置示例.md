# 图片功能配置示例

本文档提供图片功能（P1.1）的前端配置示例。

---

## 1. 图片变量配置示例

在模板编辑器的"变量元数据 (YAML)"中，按以下格式配置图片变量：

### 示例1：获取单张产品图片

```yaml
# 用户输入
product_id:
  type: string
  source: user_input
  required: true
  description: 产品ID
  ui_config:
    input_type: text
    placeholder: 请输入产品ID

# 获取产品图片
product_photo:
  type: image
  source: image
  description: 产品照片
  dependencies:
    - product_id
  image_config:
    endpoint: "https://picsum.photos/400/300"  # 示例图片API
    method: GET
    output_format: url  # base64 | url | bytes
    timeout: 30
```

### 示例2：带认证的图片获取

```yaml
api_token:
  type: string
  source: user_input
  required: true
  description: API Token
  ui_config:
    input_type: text
    placeholder: 请输入API Token

secure_image:
  type: image
  source: image
  description: 需要认证的图片
  dependencies:
    - api_token
  image_config:
    endpoint: "https://api.example.com/secure/image.jpg"
    method: GET
    headers:
      Authorization: "Bearer {{api_token}}"
    output_format: base64
```

### 示例3：批量图片获取

```yaml
batch_images:
  type: array
  source: image
  description: 批量图片
  image_config:
    endpoint: "https://picsum.photos/300/200"
    multiple: true
    output_format: url
```

## 2. 视觉AI配置示例

### 示例1：产品质量检测

```yaml
# 先获取图片
product_photo:
  type: image
  source: image
  description: 产品照片
  image_config:
    endpoint: "https://picsum.photos/400/300"
    output_format: url

# AI质量分析
quality_check:
  type: string
  source: vision_ai
  description: 质量检测报告
  dependencies:
    - product_photo
  vision_ai_config:
    model: gpt-4o
    image_source: product_photo
    prompt_template: |
      请对这个产品进行质量检测：
      1. 外观评价
      2. 可见缺陷
      3. 质量评分（1-10分）
    temperature: 0.3
    max_tokens: 500
```

### 示例2：带系统提示的分析

```yaml
product_category:
  type: string
  source: user_input
  description: 产品类别
  ui_config:
    input_type: text

product_image:
  type: image
  source: image
  image_config:
    endpoint: "https://picsum.photos/400/300"
    output_format: url

expert_analysis:
  type: string
  source: vision_ai
  dependencies:
    - product_image
    - product_category
  vision_ai_config:
    model: gpt-4o
    image_source: product_image
    system_prompt: "你是专业的{{product_category}}质检专家"
    prompt_template: |
      请以专业角度分析这个{{product_category}}产品：
      - 整体印象
      - 细节评价
      - 改进建议
    temperature: 0.5
    max_tokens: 800
```

## 3. 完整报告模板示例

### 模板内容 (Jinja2)

```markdown
# 产品质量检测报告

## 基本信息
- 产品ID: {{product_id}}
- 检测时间: {{current_time}}

## 产品照片
{{product_photo.markdown}}

## AI质量分析
{{quality_check}}

---
*本报告由AI自动生成*
```

### 变量元数据 (YAML)

```yaml
current_time:
  type: string
  source: system
  description: 当前时间
  system_config:
    fields:
      current_time:
        generator: datetime
        format: "%Y-%m-%d %H:%M:%S"

product_id:
  type: string
  source: user_input
  required: true
  description: 产品ID
  ui_config:
    input_type: text
    placeholder: 输入产品ID

product_photo:
  type: image
  source: image
  description: 产品照片
  dependencies:
    - product_id
  image_config:
    endpoint: "https://picsum.photos/400/300"
    method: GET
    output_format: base64
    timeout: 30

quality_check:
  type: string
  source: vision_ai
  description: AI质量分析
  dependencies:
    - product_photo
  vision_ai_config:
    model: gpt-4o
    image_source: product_photo
    prompt_template: |
      请对这个产品进行全面的质量检测，包括：
      1. 整体外观评价
      2. 是否有明显缺陷（划痕、污渍、变形等）
      3. 颜色是否均匀
      4. 质量评分（1-10分，10分为完美）
      
      请以专业、客观的语气描述。
    temperature: 0.3
    max_tokens: 500
```

## 4. 使用步骤

### 步骤1：创建模板
1. 进入"模板管理"
2. 点击"新建模板"
3. 填写模板名称和描述

### 步骤2：编辑模板内容
在"模板内容 (Jinja2)"区域输入上述Markdown模板

### 步骤3：配置变量元数据
在"变量元数据 (YAML)"区域粘贴上述YAML配置

### 步骤4：保存模板
点击"保存模板"按钮

### 步骤5：生成报告
1. 进入"生成报告"页面
2. 选择刚创建的模板
3. 填写用户输入（如product_id）
4. 点击"生成报告"
5. 查看实时进度和最终报告

## 5. 配置参数说明

### image_config参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| endpoint | string | ✅ | 图片URL或API端点，支持变量插值 |
| method | string | ❌ | HTTP方法，默认GET |
| headers | object | ❌ | 请求头，支持变量插值 |
| parameters | object | ❌ | URL参数，支持变量插值 |
| output_format | string | ❌ | 输出格式：base64/url/bytes，默认base64 |
| multiple | boolean | ❌ | 是否批量获取，默认false |
| timeout | number | ❌ | 超时时间（秒），默认30 |

### vision_ai_config参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| model | string | ✅ | 模型名称：gpt-4o 或 gpt-4-vision-preview |
| image_source | string | ✅ | 图片变量名 |
| prompt_template | string | ✅ | 分析提示词，支持变量插值 |
| system_prompt | string | ❌ | 系统角色定义，支持变量插值 |
| temperature | number | ❌ | 温度参数(0.0-2.0)，默认0.7 |
| max_tokens | number | ❌ | 最大输出Token数，默认500 |

## 6. 常见问题

### Q: 图片显示不出来？
A: 检查：
1. 图片URL是否正确
2. output_format是否设置正确
3. 查看执行日志中的错误信息

### Q: Vision AI分析失败？
A: 检查：
1. OpenAI API Key是否已配置（系统设置 → AI配置）
2. image_source变量名是否正确
3. 图片是否已成功获取
4. 查看变量执行状态

### Q: 如何使用多张图片？
A: 设置 `multiple: true` 并在模板中使用循环：
```jinja2
{% for img in batch_images %}
![图片{{loop.index}}]({{img.url}})
{% endfor %}
```

   - 查看result_preview了解变量值

3. **检查YAML语法**
   - 使用在线YAML校验工具
   - 注意缩进（使用空格，不用Tab）

4. **测试图片URL**
   - 先在浏览器中访问图片URL
   - 确保URL可访问
