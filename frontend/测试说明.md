# 前端P0核心模块测试说明

## 测试环境

- 后端服务器：http://localhost:8000
- 前端服务器：http://localhost:5173

## 测试流程

### 1. AI配置测试

1. 访问 http://localhost:5173/settings/ai
2. 输入有效的 OpenAI API Key（格式：sk-...）
3. 点击"保存配置"
4. 验证显示"AI配置已完成"的成功提示

**预期结果**：
- API Key 保存成功
- 页面显示配置状态为已完成

---

### 2. 模板管理测试

#### 2.1 创建模板

1. 访问 http://localhost:5173/templates
2. 点击"创建模板"按钮
3. 填写模板信息：
   - 模板名称：简单报告测试
   - 模板描述：用于测试的简单报告模板
   - 模板内容：
     ```jinja2
     # {{title}}

     生成时间: {{generation_time}}

     ## 内容

     {{content}}
     ```
   - 变量元数据：
     ```yaml
     title:
       type: string
       source: user_input
       required: true
       description: 报告标题
       ui_config:
         input_type: text
         placeholder: 请输入报告标题

     content:
       type: string
       source: user_input
       required: true
       description: 报告内容
       ui_config:
         input_type: textarea
         placeholder: 请输入报告内容

     generation_time:
       type: string
       source: system
       required: true
       description: 生成时间
       system_config:
         fields:
           timestamp:
             generator: datetime
             format: "%Y-%m-%d %H:%M:%S"
     ```
4. 点击"保存模板"

**预期结果**：
- 模板创建成功
- 跳转到模板列表页面
- 列表中显示新创建的模板

#### 2.2 编辑模板

1. 在模板列表中找到刚创建的模板
2. 点击"编辑"按钮
3. 修改模板描述为：更新后的描述
4. 点击"保存模板"

**预期结果**：
- 模板更新成功
- 描述已更新

#### 2.3 查看模板列表

1. 访问 http://localhost:5173/templates
2. 验证模板列表显示正确
3. 测试搜索功能

**预期结果**：
- 显示所有模板
- 搜索功能正常

### 5. AI生成测试（可选）

如果配置了 OpenAI API Key，可以测试带 AI 生成的模板：

#### 5.1 创建AI模板

创建包含 AI 生成变量的模板：

```yaml
report_title:
  type: string
  source: user_input
  required: true
  description: 报告主题

ai_summary:
  type: string
  source: ai_generation
  required: true
  description: AI生成的总结
  dependencies: [report_title]
  ai_config:
    model: gpt-3.5-turbo
    prompt_template: |
      请为以下主题生成一段简短的总结（不超过100字）：
      主题：{{report_title}}
    temperature: 0.7
  schema:
    type: string
```

#### 5.2 生成报告

1. 选择AI模板
2. 填写报告主题
3. 开始生成
4. 观察AI变量的执行进度

**预期结果**：
- AI 变量显示正在执行
- 执行完成后显示结果预览
- 最终报告包含AI生成的内容

## 常见问题

### 1. 前端无法连接后端

**解决方案**：
- 检查后端服务器是否运行在 http://localhost:8000
- 检查 CORS 配置
- 查看浏览器控制台错误信息

### 2. 模板创建失败

**解决方案**：
- 检查 YAML 格式是否正确
- 验证模板内容的 Jinja2 语法
- 查看后端日志

### 3. 报告生成失败

**解决方案**：
- 检查 AI API Key 是否正确配置（如果使用 AI 变量）
- 查看进度页面的错误信息
- 检查后端日志

### 4. 进度不更新

**解决方案**：
- 刷新页面
- 检查网络连接
- 查看浏览器控制台
