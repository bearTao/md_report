# 报告历史功能增强 - 前端实现

## 修改日期
2025-10-20

## 功能说明

在报告历史列表中添加"查看生成过程"按钮，支持：
1. ✅ 显示所有状态的报告（成功、失败、进行中、排队中）
2. ✅ 一键跳转到生成进度页面查看详细过程
3. ✅ 失败的报告可以重新生成

---

#### b) 修改操作列

{
  title: '操作',
  key: 'action',
  width: 100,
  render: (_: any, record: ReportListItem) => (
    <Button
      type="link"

**修改后**:
```typescript
{
  title: '操作',
  key: 'action',
  width: 200,  // 增加宽度以容纳更多按钮
  render: (_: any, record: ReportListItem) => (
    <Space size="small">
      {/* 成功状态：显示查看报告按钮 */}
      {record.status === 'success' && (
        <Button
          type="link"
          size="small"
          icon={<EyeOutlined />}
          onClick={() => navigate(`/reports/${record.id}`)}
        >
          查看报告
        </Button>
      )}
      
      {/* 所有状态：显示查看生成过程按钮（只要有task_id） */}
      {record.task_id && (
        <Button
          type="link"
          size="small"
          icon={<ClockCircleOutlined />}
          onClick={() => navigate(`/generate/${record.task_id}`)}
        >
          查看生成过程
        </Button>
      )}
      
      {/* 失败状态：显示重新生成按钮 */}
      {record.status === 'failed' && record.task_id && (
        <Button
          type="link"
          size="small"
          icon={<RedoOutlined />}
          onClick={() => {
            message.info('跳转到生成页面，您可以重新生成报告');
            navigate(`/generate/${record.task_id}`);
          }}
        >
          重新生成
        </Button>
      )}
    </Space>
  ),
}
```

## 按钮显示逻辑

根据报告状态显示不同的操作按钮：

### 成功状态 (success)
```
[查看报告] [查看生成过程]
```

### 失败状态 (failed)
```
[查看生成过程] [重新生成]
```

### 进行中状态 (running)
```
[查看生成过程]
```

### 排队中状态 (pending)
```
[查看生成过程]
```

## 用户交互流程

### 场景1: 查看成功的报告
1. 用户在报告历史列表看到成功的报告
2. 点击 **[查看报告]** → 跳转到报告详情页
3. 或点击 **[查看生成过程]** → 跳转到生成进度页面（查看历史）

### 场景2: 重试失败的报告
1. 用户在报告历史列表看到失败的报告（红色标签）
2. 点击 **[查看生成过程]** → 查看失败原因和详细日志
3. 或点击 **[重新生成]** → 跳转到生成进度页面并开始新的生成

### 场景3: 查看进行中的报告
1. 用户在报告历史列表看到进行中的报告（蓝色标签）
2. 点击 **[查看生成过程]** → 跳转到生成进度页面实时查看

## URL跳转

### 查看报告详情
```
/reports/{report_id}
```
例如: `/reports/rpt_abc123`

### 查看生成过程
```
/generate/{task_id}
```
例如: `/generate/task_258373f1d1d7`

**说明**: 
- `report_id` 来自 `record.id`
- `task_id` 来自 `record.task_id`（后端新增字段）
