## 技术实现要点

### 1. 使用主task_id而非临时ID
**关键**: 所有子模板的变量执行都使用主任务的task_id，这样才能在查询时获取到所有变量记录。

```python
# 旧实现（问题）
task_id = f"include_{template_id}_{uuid.uuid4().hex[:8]}"  # 临时ID

# 新实现（正确）
# 使用传入的main_task_id，所有子模板共享同一个task_id
```

### 2. 模板路径构建
采用递归方式构建完整路径：
- 主模板: "主模板名称"
- 一级子模板: "主模板 > 子模板1"
- 二级子模板: "主模板 > 子模板1 > 子子模板"

```python
if parent_path:
    sub_template_path = f"{parent_path} > {sub_template.name or template_id}"
else:
    sub_template_path = sub_template.name or template_id
```

### 3. Context属性动态设置
由于Python支持动态属性，在不修改`ExecutionContext.__init__`的情况下，直接设置属性：

```python
context.template_id = template_id
context.template_path = template_path
```

在executor中使用`getattr`安全获取：
```python
template_id = getattr(self.context, 'template_id', None)
template_path = getattr(self.context, 'template_path', None)
```

### 4. 向后兼容
所有新增字段都设为可选（`nullable=True` 或 `Optional`），确保：
- 已有数据不受影响
- 不使用嵌套模板的场景正常工作
- 逐步迁移而非强制升级
