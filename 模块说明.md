## 模块说明（P0/P1优先级版）

说明：本文件整理了前端路由、后端接口、WebSocket 事件模型和最小数据表结构（PostgreSQL），并标注 P0（MVP 必需）/P1（增强稳定与易用）优先级。依据《需求文档.md》《字段说明文档.md》。

---

## 前端路由（React Router 示例）

- P0
  - `/templates`：模板列表（名称、描述、创建时间，搜索可延后至 P1）
  - `/templates/:templateId/edit`：模板编辑（Jinja2 代码 + 元数据 YAML/JSON 基础校验）
  - `/generate`：选择模板 + 动态表单（基于 user_input 变量）
  - `/generate/:taskId`：报告生成执行页（总进度 + 变量任务列表 + 下拉详情）
  - `/reports/:reportId`：报告预览（Markdown 渲染，下载 .md）
  - `/settings/ai`：AI Key 配置（OpenAI Key）

- P1
  - `/reports`：报告历史列表（时间、模板、标题、状态）
  - `/templates/:templateId/preview`：模板结构预览（示例数据渲染）
  - `/logs/:taskId`：执行日志详情（任务级/变量级）
  - `/settings/db`：数据库连接配置（新增/编辑/测试）

## 后端接口清单（FastAPI 示例）

约定：所有响应采用统一结构 `{ "success": boolean, "data": any, "error": { code, message } | null }`

### 1) 模板管理 /api/templates/*

- P0: GET `/api/templates`
  - 描述：分页获取模板列表
  - Query：`page`(int, 默认1), `page_size`(int, 默认20), `q`(string, 可选)
  - 响应（示例）：
    ```json
    {
      "success": true,
      "data": {
        "items": [
          {"id": "tpl_123", "name": "项目周报", "description": "...", "created_at": "2025-10-13T10:00:00Z"}
        ],
        "total": 1
      },
      "error": null
    }
    ```

- P0: POST `/api/templates`
  - 描述：创建模板（支持 multipart 上传或 JSON 内联）
  - 请求（JSON 内联示例）：
    ```json
    {
      "name": "项目周报",
      "description": "周报模板",
      "template_content": "# {{report_title}}\n...",  
      "metadata": { "variables": { /* YAML/JSON 解析后对象 */ } }
    }
    ```
  - 响应：`{ id, name, created_at }`

- P0: GET `/api/templates/{template_id}`
  - 描述：获取模板详情（含模板内容与元数据）

- P0: PUT `/api/templates/{template_id}`
  - 描述：更新模板（模板内容与元数据）

- P0: DELETE `/api/templates/{template_id}`
  - 描述：删除模板（MVP 可硬删，P1 可改为软删）

- P1: POST `/api/templates/{template_id}/validate`
  - 描述：校验模板与元数据完整性（语法/依赖/Schema）
  - 响应：`{ valid: boolean, issues: [ { level: "error|warn", message } ] }`

### 2) 报告生成与查询 /api/reports/*

- P0: POST `/api/reports/generate`
  - 描述：启动生成任务
  - 请求：
    ```json
    {
      "template_id": "tpl_123",
      "inputs": { "report_title": "Q3项目总结", "project_id": "PRJ-2025-001" }
    }
    ```
  - 响应：`{ task_id: "task_abc", status: "pending" }`

- P0: GET `/api/reports/{report_id}`
  - 描述：获取报告详情（含 Markdown 内容）
  - 响应：
    ```json
    {
      "success": true,
      "data": {
        "id": "rpt_001",
        "template_id": "tpl_123",
        "title": "Q3项目总结",
        "status": "success",
        "markdown_content": "# Q3项目总结\n...",
        "cost_usd": 0.12,
        "duration_ms": 25300,
        "created_at": "2025-10-13T10:05:00Z"
      },
      "error": null
    }
    ```

- P0: GET `/api/reports/{report_id}/download`
  - 描述：下载报告（`format=md`）

- P1: GET `/api/reports`
  - 描述：历史列表（支持按时间/模板/状态筛选）

- P1: POST `/api/reports/{task_id}/cancel`
  - 描述：取消执行中的任务

- P1: POST `/api/reports/{task_id}/variables/{variable_name}/retry`
  - 描述：单变量重试

- P1: GET `/api/reports/{task_id}/logs`
  - 描述：获取执行日志（任务级/变量级明细）

### 3) 系统配置 /api/config/*

- P0: GET `/api/config/ai`
  - 描述：获取 AI Key 配置（仅状态/是否已配置等）
  - 响应：`{ configured: true, provider: "openai" }`

- P0: PUT `/api/config/ai`
  - 描述：设置/更新 AI Key
  - 请求：`{ provider: "openai", api_key: "sk-***" }`

- P1: GET `/api/config/db-connections`
  - 描述：数据库连接列表

- P1: POST `/api/config/db-connections`
  - 描述：新增数据库连接

- P1: PUT `/api/config/db-connections/{connection_id}` / DELETE 同理

- P1: POST `/api/config/db-connections/{connection_id}/test`
  - 描述：连接测试

## WebSocket 事件模型

- 路径：`/ws/report-generation/{task_id}`
- 建议：连接后即推送 `task_started`；其后按变量执行生命周期推送事件；任务结束推送 `task_completed` 或 `task_failed`。

### 事件类型与负载（P0）

```json
// task_started
{
  "type": "task_started",
  "task_id": "task_abc",
  "template_id": "tpl_123",
  "queued_at": "2025-10-13T10:00:01Z",
  "started_at": "2025-10-13T10:00:03Z"
}

// variable_started
{
  "type": "variable_started",
  "task_id": "task_abc",
  "variable_name": "user_profile",
  "source": "sql",
  "dependencies": ["user_id"],
  "started_at": "2025-10-13T10:00:05Z"
}

// variable_progress（AI 可选进度）
{
  "type": "variable_progress",
  "task_id": "task_abc",
  "variable_name": "progress_summary",
  "progress": 65,
  "info": {"tokens_used": 1250}
}

// variable_completed
{
  "type": "variable_completed",
  "task_id": "task_abc",
  "variable_name": "user_profile",
  "duration_ms": 1200,
  "result_preview": {"name": "张三", "department": "研发部"}
}

// variable_failed
{
  "type": "variable_failed",
  "task_id": "task_abc",
  "variable_name": "weather_data",
  "error": {"code": "HTTP_500", "message": "upstream error"},
  "duration_ms": 900
}

// task_completed
{
  "type": "task_completed",
  "task_id": "task_abc",
  "report_id": "rpt_001",
  "summary": {"duration_ms": 25300, "ai_cost_usd": 0.12}
}

// task_failed
{
  "type": "task_failed",
  "task_id": "task_abc",
  "summary": {"duration_ms": 8400},
  "error": {"code": "AI_SCHEMA_ERROR", "message": "output parse failed"}
}
```

### 可选扩展（P1）
- `heartbeat` 心跳/健康
- `variable_log` 变量级日志片段（长日志分片）

## 最小数据表结构（PostgreSQL）

说明：以 P0 为主，P1 表做准备但不阻塞 MVP。

### 公共枚举（或以 CHECK 代替）

```sql
-- P0
CREATE TYPE report_status AS ENUM ('pending','running','success','failed','cancelled');
CREATE TYPE variable_source AS ENUM ('user_input','sql','api','ai_generation','system');
CREATE TYPE variable_status AS ENUM ('pending','running','success','failed','skipped');
```

### P0 表

```sql
-- 模板表
CREATE TABLE templates (
  id              UUID PRIMARY KEY,
  name            TEXT NOT NULL,
  description     TEXT,
  template_content TEXT NOT NULL,          -- Jinja2 内容
  metadata_json   JSONB NOT NULL,          -- 变量元数据（解析后的 JSON）
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 报告生成任务（一次生成过程，可能对应一个输出报告）
CREATE TABLE generation_tasks (
  id              UUID PRIMARY KEY,
  template_id     UUID NOT NULL REFERENCES templates(id) ON DELETE CASCADE,
  status          report_status NOT NULL DEFAULT 'pending',
  inputs_json     JSONB NOT NULL,          -- 用户输入
  started_at      TIMESTAMPTZ,
  finished_at     TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 变量级执行明细（便于前端下拉查看）
CREATE TABLE generation_task_variables (
  id              BIGSERIAL PRIMARY KEY,
  task_id         UUID NOT NULL REFERENCES generation_tasks(id) ON DELETE CASCADE,
  variable_name   TEXT NOT NULL,
  source          variable_source NOT NULL,
  status          variable_status NOT NULL DEFAULT 'pending',
  started_at      TIMESTAMPTZ,
  finished_at     TIMESTAMPTZ,
  duration_ms     INTEGER,
  error_code      TEXT,
  error_message   TEXT,
  result_preview  JSONB                     -- 结果预览（截断后的片段）
);

-- 报告成品
CREATE TABLE reports (
  id              UUID PRIMARY KEY,
  template_id     UUID NOT NULL REFERENCES templates(id) ON DELETE SET NULL,
  task_id         UUID UNIQUE REFERENCES generation_tasks(id) ON DELETE SET NULL,
  title           TEXT,
  status          report_status NOT NULL,
  markdown_content TEXT NOT NULL,
  cost_usd        NUMERIC(10,4),
  duration_ms     INTEGER,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- AI Key（最小可用：只支持 OpenAI）
CREATE TABLE ai_provider_keys (
  id              BIGSERIAL PRIMARY KEY,
  provider        TEXT NOT NULL,           -- 'openai'
  api_key_ciphertext TEXT NOT NULL,        -- 加密存储（或由 KMS 托管）
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### P1 表

```sql
-- 数据库连接配置（P1）
CREATE TYPE db_engine AS ENUM ('postgresql','mysql','sqlserver','oracle');

CREATE TABLE db_connections (
  id              UUID PRIMARY KEY,
  name            TEXT NOT NULL UNIQUE,
  engine          db_engine NOT NULL,
  host            TEXT NOT NULL,
  port            INTEGER NOT NULL,
  database        TEXT NOT NULL,
  username        TEXT NOT NULL,
  password_ciphertext TEXT NOT NULL,
  options_json    JSONB,                   -- 连接参数
  is_active       BOOLEAN NOT NULL DEFAULT true,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 可选：执行日志明细（文本/长日志）
CREATE TABLE execution_logs (
  id              BIGSERIAL PRIMARY KEY,
  task_id         UUID NOT NULL REFERENCES generation_tasks(id) ON DELETE CASCADE,
  variable_name   TEXT,
  level           TEXT NOT NULL,           -- info|warn|error
  message         TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## 接口与表的优先级说明与取舍

- 坚持 P0 聚焦：模板 CRUD、生成任务、AI 结构化输出、Jinja2 渲染、过程可视化（WS）、报告可读可下。
- P1 以稳定性/易用性为主：历史/日志/取消重试/连接管理/校验增强；均不阻塞 MVP。
- 延后（P2/P3）示例：模板版本/市场、DAG 交互编辑、时间线/Gantt、成本预算与告警、LangSmith 追踪、多模型适配、全文检索与归档、复杂 RBAC 等。

## 后端非接口服务模块（纯服务组件）

说明：以下为不直接对外暴露 HTTP/WS 的服务能力，供 API 层编排或由任务队列触发。标注 P0（MVP 必需）/P1（增强）。

### 1) 执行调度器 Scheduler（P0）
- 职责：基于模板变量依赖图（DAG）生成执行计划；管理并发；控制状态迁移（pending→running→success/failed）。
- 关键点：
  - DAG 拓扑排序，批次并行（同层无依赖可并行）。
  - 超时控制（变量级与任务级）。
  - 状态写入：`generation_tasks` 与 `generation_task_variables`。
  - 与进度推送器协作（事件入队 → WS 层消费）。

### 2) 变量执行器 Variable Runners（P0）
- 职责：按 `source` 类型执行单变量逻辑，向上下文写入结果。
- 子执行器：
  - UserInputRunner（从请求的 inputs_json 读取并校验，默认值填充）。
  - SystemRunner（timestamp/uuid/constant 生成与格式化）。
  - SqlRunner（SQL 渲染→参数化执行→结果映射→Schema 验证）。
  - ApiRunner（HTTP 调用→响应映射→错误码处理→可选重试）。
  - AiRunner（LangChain 调用→Parser→Schema 验证→重试）。
- 关键点：幂等性、错误分类、失败降级（default 值）。

### 3) 上下文与缓存 Context/Cache（P0/P1）
- P0：进程内上下文（字典），存放变量结果；可序列化快照用于日志或任务恢复。
- P1：Redis 缓存（api/ai 参数哈希→TTL），跨进程共享上下文片段（大对象按需存）。

### 4) 模板解析与渲染 Renderer（P0）
- 职责：Jinja2 `SandboxedEnvironment` 加载模板，注入上下文，输出 Markdown。
- 关键点：禁用危险操作、自定义过滤器（P1）、渲染错误捕获与定位。

### 5) 数据源连接器 Connectors（P0/P1）
- DBConnector（P0）：初始化 SQLAlchemy 引擎、参数化查询、超时；P1 扩展连接池与健康检查。
- ApiConnector（P0）：httpx 客户端、超时与简单重试；P1 支持 JSONPath 映射器与重试退避策略。

### 6) 日志与可观测性 Logging/Observability（P1）
- 变量级与任务级结构化日志；错误堆栈；可选分布式追踪埋点接口（后期接入）。
- 写入 `execution_logs`（P1）与外部日志系统适配层（可延后）。

### 7) 配置与密钥管理 Config/Secrets（P0/P1）
- P0：AI Provider Key 加密存取（KMS 或本地加密）。
- P1：DB 连接配置存取与校验；对象存储/S3 凭证管理。

### 8) 对象存储 Storage（P1）
- 说明：模板与报告文本可先落 PostgreSQL；P1 可引入 S3/MinIO 以优化存储与大文件管理。

### 9) 任务队列与异步执行 Task Queue（P1）
- 说明：MVP 可由进程内线程池/协程并发；P1 引入 Celery/RQ + Redis，支持大规模并发与重试回放。

### 10) 重试/超时/幂等策略（P1）
- 变量级策略配置（AI/HTTP 重试次数、退避、幂等键）；任务级超时；失败回滚与默认值替代。

### 11) 成本计量与配额 Cost/Quota（P1）
- 统计 AI Token、调用次数与 USD 成本；可设置任务/日预算阈值（仅统计，告警可后置）。

### 12) 运行链路（示意）
```
请求入队 → 调度器生成批次 → 并行 VariableRunners 执行
  → 写入变量结果与状态 → 进度事件推送 → 所有变量完成
  → 渲染器生成 Markdown → 报告持久化 → 结束事件/摘要
```

### P0/P1 取舍速览
- P0：Scheduler、VariableRunners、Renderer、基础 Connectors、上下文（内存）、AI Key 存取。
- P1：Redis 缓存、DB 连接管理、对象存储、任务队列、日志明细、重试幂等、成本统计。
