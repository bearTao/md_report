# 项目概述

本系统旨在为用户提供一个可配置、可扩展的 Markdown 报告自动生成平台。用户可上传 Jinja2 格式的报告模板，并通过表单输入或系统自动获取所需变量数据（包括 AI 生成内容），系统将自动渲染生成结构完整、内容准确的 Markdown 报告。

系统核心能力包括：

- 模板管理（上传、编辑、删除）
- 多源变量注入（用户输入、SQL、API、系统、AI）
- 基于依赖关系的变量执行调度
- 利用 LangChain 集成大语言模型（LLM）生成结构化内容
- 安全可控的 Jinja2 渲染

# 功能需求

## 用户层功能

### 模板选择

用户可从已有模板列表中选择一个用于生成报告

### 数据输入

系统根据所选模板中 `user_input` 类型变量，动态生成表单供用户填写

### 报告预览与下载

生成后支持在线预览 Markdown 内容，并提供下载 `.md` 文件

## 应用层功能（基于 LangChain 架构）

### 模板管理模块

- 支持上传、编辑、删除 .jinja2 或 .md.j2 格式模板文件
- 模板存储于文件系统或对象存储（如 S3/MinIO）
- 每个模板关联一个 变量元数据配置文件（YAML/JSON）

### 变量解析与执行引擎

- 解析模板对应的变量元数据（依据《字段说明文档》）
- 构建变量依赖图（DAG），按拓扑序执行变量加载
- 支持五类数据源：
  - `user_input`：从前端表单获取
  - `sql`：连接数据库执行查询
  - `api`：调用外部 HTTP 接口
  - `system`：生成时间戳、UUID 等
  - `ai_generation`：通过 LangChain 调用 LLM

### AI 对话层（LangChain 集成）

**目标**：安全、可靠地生成符合 schema 要求的结构化内容

**实现方式**：

- 使用 `langchain_core.runnables` 封装每个 `ai_generation` 变量为一个 Runnable
- 使用 `ChatPromptTemplate.from_template(ai_config.prompt_template)` 构建提示
- 使用 `JsonOutputParser` 或 `PydanticOutputParser` 强制 LLM 输出符合 schema 的 JSON
- 支持模型配置（model, temperature）
- 可选：集成 `langchain.retrievers` 或 `langgraph` 用于复杂多轮生成（当前阶段非必需）
- 输出校验：若解析失败，记录日志并使用 default 值（或触发重试）

### 报告渲染模块

- 使用 `jinja2.SandboxedEnvironment` 加载模板
- 注入所有已解析变量（包括 AI 生成结果）
- 渲染输出标准 Markdown 字符串
- ~~禁用模板中的危险操作（如 `__import__`, `getattr`）~~

## 数据层功能

- 模板存储：模板文件 + 元数据配置（YAML）
- 报告存储：生成的 Markdown 报告（可选持久化，用于审计或历史追溯）
- ~~缓存机制：对 api 和 ai_generation 类型变量支持 TTL 缓存（基于输入参数哈希）~~

# LangChain 集成设计要点

- `ChatPromptTemplate`
  - 构建动态提示
  - 支持变量插值（如 `{{project_info}}`）
- `ChatOpenAI` 
  - LLM 调用
  - 从 `ai_config.model` 动态选择
- `JsonOutputParser`
  - 强制结构化输出
  - 基于 `schema` 自动生成 parser
- `RunnableSequence`
  - 编排生成流程
  - Prompt → LLM → Parser → Validate
- `@traceable` (LangSmith)
  - 可选：链路追踪
  - 便于调试 AI 生成过程

✅ **优势**：LangChain 的 Runnable 抽象天然适合将每个 AI 变量封装为独立、可测试、可缓存的单元。

# 交付范围（MVP）

- 模板上传与管理（含元数据 YAML）
- 用户输入表单动态生成
- 变量依赖解析与顺序执行
- LangChain 集成 AI 生成（含 schema 校验）
- 安全 Jinja2 渲染与 Markdown 输出
- 基础错误处理与日志

⚠️ **不包含**：用户权限管理、多租户、高并发队列、Web UI（可另起前端项目）

# 附录：变量元数据示例（YAML）

```yaml
risk_analysis:
  type: "array"
  source: "ai_generation"
  required: false
  default: []
  description: "风险分析结果"
  dependencies: ["project_tasks", "project_info"]
  ai_config:
    model: "gpt-4"
    prompt_template: |
      基于项目任务和项目信息进行风险分析：
      项目信息：{{project_info}}
      任务列表：{{project_tasks}}
  
      请识别3-5个主要风险...（略）
    temperature: 0.3
  schema:
    type: "array"
    items:
      type: "object"
      properties:
        risk_description: {type: "string"}
        impact_level: {type: "string", enum: ["高", "中", "低"]}
        probability: {type: "string", enum: ["高", "中", "低"]}
        suggestion: {type: "string"}
```
