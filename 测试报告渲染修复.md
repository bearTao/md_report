## 常见问题排查

### 问题1: 前端没有显示日志

**检查**:
1. 后端 API 是否正常：`curl http://localhost:8000/api/reports/tasks/{task_id}/logs`
2. 浏览器控制台是否有错误
3. 检查 TaskLogs 组件是否正确导入

### 问题2: 渲染日志没有写入数据库

**检查**:
1. 查看后端日志文件：`tail -f backend/logs/app.log | grep render`
2. 检查数据库表 `execution_logs` 的权限
3. 确认 `_log_to_db` 方法没有异常

### 问题3: 模板仍然报错

**检查**:
1. 确认模板已更新：查询数据库中的模板内容
2. 清除浏览器缓存
3. 重启后端服务

## 性能监控

可以通过日志监控渲染性能：

```sql
-- 查询渲染耗时统计
SELECT 
    task_id,
    SUBSTRING_INDEX(SUBSTRING_INDEX(message, '耗时 ', -1), 'ms', 1) as duration_ms,
    created_at
FROM execution_logs
WHERE variable_name = '[渲染引擎]' 
  AND message LIKE '%渲染成功%'
ORDER BY created_at DESC
LIMIT 10;
```

**预期**: 大多数报告渲染时间应该在 10-50ms 之间
